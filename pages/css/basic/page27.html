CSS Selectors

Understanding CSS Selectors

CSS selectors are patterns used to select and style HTML elements. They are the bridge between your HTML document and your style rules, allowing you to target specific elements or groups of elements for styling.

Mastering CSS selectors is crucial for efficient and effective styling. With the right selectors, you can:

  • Target elements precisely without adding extra classes or IDs to your HTML
  • Create more maintainable code by reducing redundancy
  • Apply styles conditionally based on element states or positions
  • Implement complex styling patterns with minimal code

In this lesson, we'll explore the full range of CSS selectors, from basic to advanced, and learn how to use them effectively.

Selector Specificity

Before diving into the different types of selectors, it's important to understand specificity, which determines which styles take precedence when multiple rules target the same element.

Specificity is calculated as a four-part value: inline styles, IDs, classes/attributes, elements

  • Inline styles: style="..." (highest specificity)
  • ID selectors: #header
  • Class selectors, attribute selectors, and pseudo-classes: .button, [type="submit"], :hover
  • Element selectors and pseudo-elements: div, ::before (lowest specificity)

When conflicts arise, the selector with the highest specificity wins. If specificity is equal, the last rule defined takes precedence.

Basic Selectors

Let's start with the most fundamental CSS selectors that form the building blocks of more complex selection patterns.

Universal Selector

The universal selector (*) selects all elements on the page.

/* Selects all elements */
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

Use cases:

  • CSS resets
  • Setting global box-sizing
  • Debugging (e.g., adding borders to all elements)

Performance Note

While convenient, the universal selector can impact performance on large pages because it applies to every element. Use it judiciously, especially for properties that trigger layout recalculations.

Type Selector (Element Selector)

Type selectors target elements based on their HTML tag name.

/* Selects all paragraph elements */
p {
  line-height: 1.6;
  margin-bottom: 1em;
}

/* Selects all heading level 2 elements */
h2 {
  font-size: 1.5em;
  margin-top: 1em;
}

Use cases:

  • Setting base styles for common elements
  • Establishing typographic hierarchy
  • Resetting browser defaults for specific elements

Class Selector

Class selectors target elements based on the value of their class attribute. They are preceded by a dot (.).

/* Selects all elements with class="button" */
.button {
  display: inline-block;
  padding: 10px 20px;
  background-color: #0066cc;
  color: white;
  border-radius: 4px;
}

/* Selects all elements with class="error" */
.error {
  color: red;
  border: 1px solid red;
  padding: 10px;
}

HTML usage:

<a href="#" class="button">Click Me</a>
<button class="button">Submit</button>
<div class="error">Invalid input</div>

Use cases:

  • Styling reusable components
  • Applying the same styles to different types of elements
  • Creating utility classes for common styling patterns

ID Selector

ID selectors target elements based on the value of their id attribute. They are preceded by a hash symbol (#).

/* Selects the element with id="header" */
#header {
  background-color: #333;
  color: white;
  padding: 20px;
}

/* Selects the element with id="main-content" */
#main-content {
  max-width: 1200px;
  margin: 0 auto;
}

HTML usage:

<header id="header">
  <h1>Website Title</h1>
</header>
<main id="main-content">
  <p>Main content goes here...</p>
</main>

Use cases:

  • Styling unique page elements that appear only once
  • Creating CSS hooks for JavaScript functionality
  • Creating anchor links within a page

Classes vs. IDs

While both classes and IDs can be used as CSS selectors, they have important differences:

  • Uniqueness: IDs must be unique within a document, while multiple elements can share the same class
  • Specificity: ID selectors have higher specificity than class selectors
  • JavaScript: IDs can be used with getElementById(), while classes require getElementsByClassName() or querySelectorAll()
  • URL Fragments: IDs can be targeted in URLs with hash fragments (#section-name)

Best practice: Use classes for styling and IDs for JavaScript hooks or anchor links. Avoid using IDs for styling when possible to keep specificity manageable.

Attribute Selector

Attribute selectors target elements based on the presence or value of their attributes. They are enclosed in square brackets.

/* Selects all elements with a title attribute */
[title] {
  cursor: help;
}

/* Selects all input elements with type="text" */
input[type="text"] {
  border: 1px solid #ccc;
  padding: 8px;
}

/* Selects all links with target="_blank" */
a[target="_blank"] {
  padding-right: 20px;
  background: url('external-link-icon.svg') no-repeat right center;
}

HTML usage:

<p title="Additional information">Hover over me for more info.</p>
<input type="text" placeholder="Enter your name">
<a href="https://example.com" target="_blank">External Link</a>

Attribute selectors can also match patterns within attribute values:

/* Selects elements with an attribute value that begins with specified text */
[class^="icon-"] {
  padding-left: 20px;
}

/* Selects elements with an attribute value that ends with specified text */
[href$=".pdf"] {
  background-image: url('pdf-icon.svg');
}

/* Selects elements with an attribute value that contains specified text */
[href*="example"] {
  color: purple;
}

/* Selects elements with an attribute value that contains a word (space-separated) */
[class~="primary"] {
  font-weight: bold;
}

Use cases:

  • Styling form elements based on their type
  • Adding icons to links based on their URL or target
  • Styling elements without adding extra classes
  • Creating icon systems with prefixed class names

Combinators

Combinators allow you to select elements based on their relationship to other elements in the document tree.

Descendant Combinator (Space)

The descendant combinator (represented by a space) selects elements that are descendants of another element, regardless of how deeply nested they are.

/* Selects all p elements that are descendants of article elements */
article p {
  font-size: 16px;
  line-height: 1.6;
}

/* Selects all li elements that are descendants of nav elements */
nav li {
  display: inline-block;
  margin-right: 10px;
}

HTML structure:

<article>
  <h2>Article Title</h2>
  <p>This paragraph will be selected.</p>
  <div>
    <p>This nested paragraph will also be selected.</p>
  </div>
</article>

<p>This paragraph outside the article will NOT be selected.</p>

Child Combinator (>)

The child combinator selects elements that are direct children of another element.

/* Selects all li elements that are direct children of ul elements */
ul > li {
  list-style-type: square;
}

/* Selects all p elements that are direct children of div elements */
div > p {
  margin-bottom: 10px;
}

HTML structure:

<div>
  <p>This paragraph is a direct child of the div and will be selected.</p>
  <section>
    <p>This paragraph is NOT a direct child of the div and will NOT be selected.</p>
  </section>
</div>

Adjacent Sibling Combinator (+)

The adjacent sibling combinator selects an element that immediately follows another element and shares the same parent.

/* Selects p elements that immediately follow h2 elements */
h2 + p {
  font-weight: bold;
}

/* Selects div elements that immediately follow form elements */
form + div {
  margin-top: 20px;
}

HTML structure:

<section>
  <h2>Section Title</h2>
  <p>This paragraph immediately follows the h2 and will be selected.</p>
  <p>This paragraph does NOT immediately follow the h2 and will NOT be selected.</p>
</section>

General Sibling Combinator (~)

The general sibling combinator selects elements that follow another element and share the same parent, regardless of whether they immediately follow it.

/* Selects all p elements that follow an h2 element and share the same parent */
h2 ~ p {
  color: #666;
}

/* Selects all li elements that follow an li with class="active" and share the same parent */
li.active ~ li {
  opacity: 0.7;
}

HTML structure:

<section>
  <p>This paragraph appears before the h2 and will NOT be selected.</p>
  <h2>Section Title</h2>
  <p>This paragraph follows the h2 and will be selected.</p>
  <div>Some content</div>
  <p>This paragraph also follows the h2 and will be selected, even though there's a div in between.</p>
</section>
Combinator Symbol Description Example
Descendant Space ( ) Selects all descendants, regardless of nesting level div p selects all <p> inside any <div>
Child > Selects only direct children div > p selects only <p> that are direct children of <div>
Adjacent Sibling + Selects the element immediately following h2 + p selects the <p> immediately after an <h2>
General Sibling ~ Selects all following siblings h2 ~ p selects all <p> that follow an <h2>

Pseudo-Classes

Pseudo-classes select elements based on their state or position in the document. They are preceded by a colon (:).

User Action Pseudo-Classes

These pseudo-classes respond to user interactions:

/* Styles applied when the mouse hovers over a link */
a:hover {
  text-decoration: underline;
  color: #0077cc;
}

/* Styles applied when a link is being clicked/activated */
a:active {
  color: #ff0000;
}

/* Styles applied when an element receives focus */
input:focus {
  outline: 2px solid #4d90fe;
  box-shadow: 0 0 5px rgba(77, 144, 254, 0.5);
}

Link State Pseudo-Classes

These pseudo-classes target links in different states:

/* Unvisited links */
a:link {
  color: #0066cc;
}

/* Visited links */
a:visited {
  color: #800080;
}

/* Note: For privacy reasons, browsers limit the styles you can apply to :visited links */

Order Matters: LVHA

When using link pseudo-classes, the order matters due to specificity rules. Remember the mnemonic "LoVe HAte" for the correct order:

  1. :link
  2. :visited
  3. :hover
  4. :active

Form Pseudo-Classes

These pseudo-classes target form elements in different states:

/* Checked checkboxes and radio buttons */
input:checked {
  box-shadow: 0 0 0 2px #4d90fe;
}

/* Disabled form elements */
input:disabled {
  background-color: #f2f2f2;
  cursor: not-allowed;
}

/* Required form elements */
input:required {
  border-left: 4px solid #ff0000;
}

/* Valid form elements */
input:valid {
  border-color: green;
}

/* Invalid form elements */
input:invalid {
  border-color: red;
}

Structural Pseudo-Classes

These pseudo-classes select elements based on their position in the document structure:

/* First child of its parent */
li:first-child {
  font-weight: bold;
}

/* Last child of its parent */
li:last-child {
  margin-bottom: 0;
}

/* Only child of its parent */
p:only-child {
  font-style: italic;
}

/* First element of its type among siblings */
p:first-of-type {
  font-size: 1.2em;
}

/* Last element of its type among siblings */
p:last-of-type {
  margin-bottom: 2em;
}

/* Only element of its type among siblings */
img:only-of-type {
  display: block;
  margin: 0 auto;
}

/* Every odd-numbered child (1st, 3rd, 5th, etc.) */
tr:nth-child(odd) {
  background-color: #f9f9f9;
}

/* Every even-numbered child (2nd, 4th, 6th, etc.) */
tr:nth-child(even) {
  background-color: #ffffff;
}

/* Specific numbered child (e.g., 3rd child) */
li:nth-child(3) {
  color: red;
}

/* Every 3rd child starting from the 2nd */
li:nth-child(3n+2) {
  text-decoration: underline;
}

/* Every odd-numbered element of its type */
p:nth-of-type(odd) {
  background-color: #f0f0f0;
}

Other Useful Pseudo-Classes

/* Elements that don't match the selector */
:not(.featured) {
  opacity: 0.8;
}

/* Empty elements (no children or text) */
div:empty {
  display: none;
}

/* The root element (usually html) */
:root {
  --main-color: #0066cc;
  font-size: 16px;
}

/* Target element (when URL has a fragment matching the element's ID) */
:target {
  background-color: #ffffd0;
  animation: highlight 2s ease-out;
}

Pseudo-Elements

Pseudo-elements create "virtual" elements that don't exist in the HTML but can be styled with CSS. They are preceded by a double colon (::).

Single vs. Double Colon

Historically, both pseudo-classes and pseudo-elements used a single colon. CSS3 introduced the double colon (::) to distinguish pseudo-elements from pseudo-classes. For backward compatibility, many browsers still support the single-colon syntax for pseudo-elements, but the double-colon is preferred for new code.

Common Pseudo-Elements

/* First line of text */
p::first-line {
  font-weight: bold;
  text-transform: uppercase;
}

/* First letter of text */
p::first-letter {
  font-size: 2em;
  float: left;
  margin-right: 0.1em;
}

/* Content before an element */
.info::before {
  content: "ℹ️ ";
}

/* Content after an element */
a[href^="http"]::after {
  content: " ↗";
  font-size: 0.8em;
}

/* Selected text */
::selection {
  background-color: #ffb7b7;
  color: #000;
}

/* Placeholder text in form inputs */
input::placeholder {
  color: #999;
  font-style: italic;
}

Creating Decorative Elements with ::before and ::after

The ::before and ::after pseudo-elements are particularly powerful because they allow you to create decorative elements without adding extra HTML.

/* Creating a fancy quote style */
blockquote {
  position: relative;
  padding: 20px 40px;
  font-style: italic;
}

blockquote::before {
  content: """;
  position: absolute;
  left: 0;
  top: 0;
  font-size: 4em;
  line-height: 1;
  color: #ccc;
}

blockquote::after {
  content: """;
  position: absolute;
  right: 0;
  bottom: -20px;
  font-size: 4em;
  line-height: 1;
  color: #ccc;
}

/* Creating a tooltip */
.tooltip {
  position: relative;
  border-bottom: 1px dotted #000;
  cursor: help;
}

.tooltip::after {
  content: attr(data-tooltip);
  position: absolute;
  bottom: 125%;
  left: 50%;
  transform: translateX(-50%);
  background-color: #333;
  color: white;
  padding: 5px 10px;
  border-radius: 4px;
  font-size: 0.9em;
  white-space: nowrap;
  opacity: 0;
  visibility: hidden;
  transition: opacity 0.3s;
}

.tooltip:hover::after {
  opacity: 1;
  visibility: visible;
}

HTML usage:

<blockquote>
  The only way to learn a new programming language is by writing programs in it.
  <cite>Dennis Ritchie</cite>
</blockquote>

<p>Hover over <span class="tooltip" data-tooltip="This is a helpful tip!">this text</span> to see a tooltip.</p>

Important Rules for ::before and ::after

  • The content property is required, even if it's empty (content: "";)
  • They are inline elements by default but can be changed with the display property
  • They are not actual DOM elements, so they can't be selected with JavaScript or receive user interactions directly
  • They are not accessible to screen readers by default, so don't use them for important content
  • They can use the attr() function to access HTML attributes of their parent element

Complex Selectors and Combinations

CSS selectors can be combined to create powerful and precise targeting patterns.

Multiple Selectors

You can apply the same styles to multiple selectors by separating them with commas:

/* Apply the same styles to h1, h2, and h3 elements */
h1, h2, h3 {
  font-family: 'Georgia', serif;
  color: #333;
}

/* Apply the same styles to multiple classes */
.error, .warning, .notice {
  padding: 10px;
  border-radius: 4px;
  margin-bottom: 15px;
}

Compound Selectors

You can combine different types of selectors without spaces to target elements that match all conditions:

/* Target div elements with class="container" */
div.container {
  max-width: 1200px;
  margin: 0 auto;
}

/* Target elements with both "btn" and "primary" classes */
.btn.primary {
  background-color: #0066cc;
  color: white;
}

/* Target input elements with type="submit" and class="button" */
input[type="submit"].button {
  cursor: pointer;
  font-weight: bold;
}

Complex Selector Chains

You can create complex selector chains by combining various selectors and combinators:

/* Target paragraphs that are direct children of articles and have class="intro" */
article > p.intro {
  font-size: 1.2em;
  font-weight: bold;
}

/* Target list items that are descendants of nav, but only when they're hovered */
nav li:hover {
  background-color: #f0f0f0;
}

/* Target the first paragraph after a heading inside a section */
section h2 + p {
  margin-top: 0;
}

/* Target all paragraphs that follow a div with class="note" and share the same parent */
div.note ~ p {
  font-style: italic;
}

/* Target the first list item of an unordered list inside the main element */
main ul li:first-child {
  font-weight: bold;
}

/* Target all links that are being hovered and have an href attribute starting with "https" */
a[href^="https"]:hover {
  color: green;
}

:is() and :where() Pseudo-Classes

Modern CSS includes the :is() and :where() pseudo-classes, which can simplify complex selectors:

/* Without :is() */
header h1, header h2, header h3,
footer h1, footer h2, footer h3,
main h1, main h2, main h3 {
  font-family: 'Arial', sans-serif;
}

/* With :is() */
:is(header, footer, main) :is(h1, h2, h3) {
  font-family: 'Arial', sans-serif;
}

/* The difference between :is() and :where() is specificity:
   - :is() takes the specificity of its most specific argument
   - :where() has zero specificity */

/* This has the specificity of an ID selector */
:is(#header, .container, p) {
  margin-bottom: 20px;
}

/* This has zero specificity, regardless of what's inside */
:where(#header, .container, p) {
  margin-bottom: 20px;
}

Browser Support Note

The :is() and :where() pseudo-classes are relatively new and may not be supported in older browsers. Check browser compatibility before using them in production.

Selector Specificity in Detail

Understanding specificity is crucial for writing maintainable CSS and debugging style conflicts.

How Specificity Is Calculated

Specificity is calculated as a four-part value, often represented as four numbers separated by commas:

  1. Inline styles: 1,0,0,0
  2. IDs: 0,1,0,0 per ID
  3. Classes, attributes, and pseudo-classes: 0,0,1,0 per instance
  4. Elements and pseudo-elements: 0,0,0,1 per instance

The selector with the highest specificity wins. If two selectors have the same specificity, the one that comes later in the CSS takes precedence.

Specificity Examples

Selector Specificity Value Explanation
p 0,0,0,1 One element
p.intro 0,0,1,1 One class, one element
.container p 0,0,1,1 One class, one element
div.container p.intro 0,0,2,2 Two classes, two elements
#header 0,1,0,0 One ID
nav ul li a:hover 0,0,1,4 One pseudo-class, four elements
div p::first-line 0,0,0,3 Two elements, one pseudo-element
[type="text"] 0,0,1,0 One attribute
p[class="intro"] 0,0,1,1 One attribute, one element
style="color: red;" 1,0,0,0 Inline style

The !important Exception

The !important declaration overrides normal specificity rules and takes precedence over all other declarations. If multiple conflicting declarations use !important, the normal specificity rules apply among them.

p {
  color: red !important; /* This will override any other color declarations for p elements */
}

#content p {
  color: blue; /* This would normally win due to higher specificity, but !important above takes precedence */
}

Avoiding Specificity Problems

To maintain manageable CSS, follow these best practices:

  • Avoid using IDs for styling when possible
  • Keep selectors as simple as possible
  • Use classes for most styling needs
  • Avoid using !important except as a last resort
  • Use a consistent naming convention (like BEM) to reduce the need for complex selectors
  • Consider using CSS custom properties (variables) for values that need to be overridden

Summary

In this lesson, we've explored the wide range of CSS selectors available for targeting HTML elements:

  • Basic Selectors: Universal (*), type (element), class (.class), ID (#id), and attribute ([attr]) selectors
  • Combinators: Descendant (space), child (>), adjacent sibling (+), and general sibling (~) combinators
  • Pseudo-Classes: Selectors that target elements based on their state or position, like :hover, :first-child, and :checked
  • Pseudo-Elements: Selectors that create virtual elements for styling, like ::before, ::after, and ::first-line
  • Complex Selectors: Combinations of multiple selectors to create precise targeting patterns
  • Specificity: The system that determines which styles take precedence when multiple rules target the same element

Understanding and effectively using CSS selectors is a fundamental skill for web development. With the right selectors, you can write more efficient, maintainable, and powerful CSS.
