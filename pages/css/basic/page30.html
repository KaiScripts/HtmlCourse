Colors & Backgrounds

The Power of Color in Web Design

Color is one of the most powerful tools in web design. It sets the mood, creates visual hierarchy, reinforces branding, and guides users through your interface. Understanding how to effectively use color in CSS is essential for creating visually appealing and user-friendly websites.

Similarly, backgrounds provide context, create contrast for content, and add visual interest to your designs. CSS offers a wide range of options for creating rich, layered backgrounds that enhance your web pages.

In this lesson, we'll explore the various ways to define and apply colors and backgrounds in CSS, from simple solid colors to complex gradients and patterns.

Color Psychology

Colors evoke emotional responses and have cultural associations that can affect how users perceive your website:

  • Red: Energy, passion, urgency, attention
  • Blue: Trust, calm, professionalism, security
  • Green: Growth, nature, health, prosperity
  • Yellow: Optimism, clarity, warmth, caution
  • Purple: Creativity, luxury, wisdom, spirituality
  • Orange: Enthusiasm, playfulness, affordability
  • Black: Sophistication, power, elegance, formality
  • White: Simplicity, cleanliness, purity, space

Consider these associations when choosing colors for your website to ensure they align with your brand and message.

CSS Color Values

CSS provides several ways to specify colors, each with its own advantages and use cases.

Named Colors

CSS includes 140+ named colors that you can use by their keyword names:

h1 {
  color: red;
}

.alert {
  background-color: yellow;
}

.button {
  border-color: navy;
}

Named colors are easy to remember and use, but they offer limited options and can vary slightly between browsers.

Hexadecimal Colors

Hexadecimal (hex) color values are six-digit codes preceded by a hash (#) that represent RGB values in base-16:

h1 {
  color: #ff0000; /* Red */
}

.alert {
  background-color: #ffff00; /* Yellow */
}

.button {
  border-color: #000080; /* Navy */
}

Each pair of digits represents a color channel (red, green, blue) with values from 00 to FF (0-255 in decimal).

You can also use a shorthand three-digit hex code when each channel has repeated digits:

/* #ff0000 can be shortened to #f00 */
h1 {
  color: #f00; /* Red */
}

/* #00ff00 can be shortened to #0f0 */
.success {
  color: #0f0; /* Green */
}

Hex codes with an additional pair of digits for alpha (transparency) are also supported:

/* #rrggbbaa format */
.overlay {
  background-color: #00000080; /* Black with 50% opacity */
}

/* Shorthand #rgba format */
.highlight {
  background-color: #f003; /* Red with 30% opacity */
}

RGB and RGBA Colors

RGB colors specify red, green, and blue channel values directly:

h1 {
  color: rgb(255, 0, 0); /* Red */
}

.alert {
  background-color: rgb(255, 255, 0); /* Yellow */
}

/* With alpha channel for transparency */
.overlay {
  background-color: rgba(0, 0, 0, 0.5); /* Black with 50% opacity */
}

Each channel takes a value from 0 to 255, and the alpha channel (in RGBA) takes a value from 0 (fully transparent) to 1 (fully opaque).

Modern CSS also supports a space-separated format with an optional alpha value:

/* Modern RGB syntax */
.modern-rgb {
  color: rgb(255 0 0); /* Red */
}

/* Modern RGBA syntax */
.modern-rgba {
  color: rgb(255 0 0 / 0.5); /* Red with 50% opacity */
}

HSL and HSLA Colors

HSL (Hue, Saturation, Lightness) is an intuitive way to specify colors:

h1 {
  color: hsl(0, 100%, 50%); /* Red */
}

.alert {
  background-color: hsl(60, 100%, 50%); /* Yellow */
}

/* With alpha channel for transparency */
.overlay {
  background-color: hsla(0, 0%, 0%, 0.5); /* Black with 50% opacity */
}
  • Hue: A degree on the color wheel (0-360), where 0/360 is red, 120 is green, and 240 is blue
  • Saturation: A percentage where 0% is grayscale and 100% is full color
  • Lightness: A percentage where 0% is black, 100% is white, and 50% is the "normal" color

HSL makes it easy to create variations of a color by adjusting saturation or lightness:

/* Different shades of blue */
.primary {
  color: hsl(210, 100%, 50%); /* Vibrant blue */
}

.primary-light {
  color: hsl(210, 100%, 70%); /* Lighter blue */
}

.primary-dark {
  color: hsl(210, 100%, 30%); /* Darker blue */
}

.primary-muted {
  color: hsl(210, 50%, 50%); /* Less saturated blue */
}

Modern CSS also supports a space-separated format with an optional alpha value:

/* Modern HSL syntax */
.modern-hsl {
  color: hsl(0 100% 50%); /* Red */
}

/* Modern HSLA syntax */
.modern-hsla {
  color: hsl(0 100% 50% / 0.5); /* Red with 50% opacity */
}

HWB Colors

HWB (Hue, Whiteness, Blackness) is a newer color model that some find more intuitive than HSL:

.hwb-example {
  color: hwb(0 0% 0%); /* Red */
}

.hwb-with-white {
  color: hwb(0 20% 0%); /* Red with 20% white added */
}

.hwb-with-black {
  color: hwb(0 0% 20%); /* Red with 20% black added */
}

.hwb-with-both {
  color: hwb(0 20% 20%); /* Red with both white and black added */
}

/* With alpha */
.hwb-alpha {
  color: hwb(0 0% 0% / 0.5); /* Red with 50% opacity */
}
  • Hue: Same as HSL, a degree on the color wheel (0-360)
  • Whiteness: Percentage of white to mix in (0-100%)
  • Blackness: Percentage of black to mix in (0-100%)

Lab and LCH Colors

CSS now supports Lab and LCH color spaces, which can represent a wider range of colors than RGB:

/* Lab color */
.lab-example {
  color: lab(50% 50 0); /* Reddish color */
}

/* LCH color */
.lch-example {
  color: lch(50% 50 0); /* Reddish color */
}

/* With alpha */
.lab-alpha {
  color: lab(50% 50 0 / 0.5); /* With 50% opacity */
}

These color spaces are designed to be perceptually uniform and can represent colors outside the sRGB gamut, though browser support is still evolving.

currentColor Keyword

The currentColor keyword refers to the current value of the color property:

.button {
  color: blue;
  border-color: currentColor; /* Will be blue */
  box-shadow: 0 0 5px currentColor; /* Will also be blue */
}

.button:hover {
  color: darkblue; /* When color changes, currentColor references also change */
}

This is useful for maintaining color consistency across different properties.

transparent Keyword

The transparent keyword represents a fully transparent color:

.invisible-border {
  border: 1px solid transparent;
}

/* Equivalent to */
.also-invisible {
  border: 1px solid rgba(0, 0, 0, 0);
}

Color Properties

CSS provides several properties for applying colors to different aspects of elements.

Text Color

The color property sets the color of text content:

p {
  color: #333; /* Dark gray text */
}

a {
  color: #0066cc; /* Blue links */
}

a:hover {
  color: #004499; /* Darker blue on hover */
}

Background Color

The background-color property sets the background color of an element:

body {
  background-color: #f5f5f5; /* Light gray background */
}

.card {
  background-color: white;
}

.highlight {
  background-color: rgba(255, 255, 0, 0.3); /* Semi-transparent yellow */
}

Border Color

The border-color property sets the color of an element's border:

.box {
  border: 1px solid #ccc; /* Gray border */
}

/* Individual border sides */
.custom-borders {
  border-top-color: red;
  border-right-color: green;
  border-bottom-color: blue;
  border-left-color: yellow;
}

Outline Color

The outline-color property sets the color of an element's outline:

input:focus {
  outline-color: #4d90fe; /* Blue outline for focused inputs */
}

.highlight {
  outline: 2px dashed #ff9900;
  outline-offset: 2px; /* Space between the element and its outline */
}

Text Decoration Color

The text-decoration-color property sets the color of text decorations:

a {
  text-decoration: underline;
  text-decoration-color: #ff6600; /* Orange underline */
}

.strikethrough {
  text-decoration: line-through;
  text-decoration-color: red;
}

Caret Color

The caret-color property sets the color of the cursor (caret) in inputs:

input {
  caret-color: #0066cc; /* Blue cursor in text inputs */
}

Accent Color

The accent-color property sets the accent color for form controls:

/* Changes the color of checkboxes, radio buttons, and other controls */
form {
  accent-color: #0066cc;
}

Column Rule Color

The column-rule-color property sets the color of the rule between columns:

.multi-column {
  columns: 3;
  column-rule: 1px solid;
  column-rule-color: #ccc; /* Gray rule between columns */
}

Background Properties

CSS provides a rich set of properties for controlling element backgrounds.

Background Image

The background-image property sets one or more background images:

.hero {
  background-image: url('hero.jpg');
}

/* Multiple background images (listed from top to bottom) */
.layered {
  background-image: 
    url('overlay.png'),
    url('texture.png'),
    url('background.jpg');
}

Background Repeat

The background-repeat property controls how background images repeat:

.repeat {
  background-image: url('pattern.png');
  background-repeat: repeat; /* Default: repeats in both directions */
}

.repeat-x {
  background-image: url('pattern.png');
  background-repeat: repeat-x; /* Repeats horizontally only */
}

.repeat-y {
  background-image: url('pattern.png');
  background-repeat: repeat-y; /* Repeats vertically only */
}

.no-repeat {
  background-image: url('icon.png');
  background-repeat: no-repeat; /* No repetition */
}

.space {
  background-image: url('pattern.png');
  background-repeat: space; /* Repeats with space between */
}

.round {
  background-image: url('pattern.png');
  background-repeat: round; /* Stretches or squeezes to avoid partial tiles */
}

Background Position

The background-position property sets the starting position of a background image:

/* Keywords */
.top-left {
  background-position: top left;
}

.center {
  background-position: center;
}

/* Percentages */
.custom-position {
  background-position: 25% 75%;
}

/* Length values */
.pixel-position {
  background-position: 20px 50px;
}

/* Mixed values */
.mixed-position {
  background-position: right 20px bottom 10px;
}

Background Size

The background-size property controls the size of background images:

/* Keywords */
.cover {
  background-size: cover; /* Scales to cover the entire element */
}

.contain {
  background-size: contain; /* Scales to fit inside the element */
}

/* Length values */
.pixel-size {
  background-size: 200px 100px;
}

/* Percentages */
.percentage-size {
  background-size: 50% 75%;
}

/* Auto for one dimension */
.auto-width {
  background-size: auto 300px;
}

Background Attachment

The background-attachment property determines whether a background image scrolls with the content:

.scroll {
  background-attachment: scroll; /* Default: scrolls with the element */
}

.fixed {
  background-attachment: fixed; /* Fixed relative to the viewport */
}

.local {
  background-attachment: local; /* Scrolls with the element's contents */
}

Background Origin

The background-origin property specifies where the background positioning area begins:

.border-box {
  background-origin: border-box; /* Background starts at the border edge */
}

.padding-box {
  background-origin: padding-box; /* Default: Background starts at the padding edge */
}

.content-box {
  background-origin: content-box; /* Background starts at the content edge */
}

Background Clip

The background-clip property determines how far the background extends:

.border-box {
  background-clip: border-box; /* Default: Background extends to the border edge */
}

.padding-box {
  background-clip: padding-box; /* Background extends to the padding edge */
}

.content-box {
  background-clip: content-box; /* Background extends to the content edge */
}

/* Text clipping (non-standard but widely supported) */
.text-clip {
  background-clip: text;
  -webkit-background-clip: text;
  color: transparent; /* Makes text transparent to show background */
  background-image: linear-gradient(to right, #ff8a00, #da1b60);
}

Background Shorthand

The background shorthand property sets all background properties at once:

/* background: color image position/size repeat attachment origin clip */
.hero {
  background: #f5f5f5 url('hero.jpg') center/cover no-repeat fixed;
}

/* Multiple backgrounds */
.complex {
  background: 
    url('overlay.png') top left/auto no-repeat,
    url('pattern.png') center/200px repeat,
    linear-gradient(to bottom, #f5f5f5, #e5e5e5);
}

Background Shorthand Order

When using the background shorthand, the order is flexible, but some rules apply:

  • If both background-position and background-size are specified, they must be separated by a slash (/)
  • background-color can only be specified once and should be the last layer
  • Omitted values are set to their initial values

Gradients

CSS gradients are special types of background images that show a smooth transition between two or more colors.

Linear Gradients

Linear gradients transition colors along a straight line:

/* Default (top to bottom) */
.gradient-default {
  background-image: linear-gradient(red, blue);
}

/* With direction angle */
.gradient-angle {
  background-image: linear-gradient(45deg, red, blue);
}

/* With direction keyword */
.gradient-direction {
  background-image: linear-gradient(to right, red, blue);
}

/* Multiple color stops */
.gradient-multi {
  background-image: linear-gradient(to right, red, yellow, green, blue);
}

/* With specific color stop positions */
.gradient-positions {
  background-image: linear-gradient(to right, red 0%, yellow 25%, green 50%, blue 100%);
}

/* Hard color stops (creates stripes) */
.gradient-stripes {
  background-image: linear-gradient(to right, 
    red 0%, red 25%, 
    yellow 25%, yellow 50%, 
    green 50%, green 75%, 
    blue 75%, blue 100%);
}

Radial Gradients

Radial gradients transition colors from a central point outward:

/* Default (center, ellipse) */
.radial-default {
  background-image: radial-gradient(red, blue);
}

/* Circle shape */
.radial-circle {
  background-image: radial-gradient(circle, red, blue);
}

/* With position */
.radial-position {
  background-image: radial-gradient(at top right, red, blue);
}

/* With size */
.radial-size {
  background-image: radial-gradient(circle closest-side, red, blue);
}

/* Full syntax */
.radial-full {
  background-image: radial-gradient(circle closest-corner at 20% 30%, red, yellow, blue);
}

Size keywords for radial gradients:

  • closest-side: Gradient ends at the closest side of the box
  • closest-corner: Gradient ends at the closest corner of the box
  • farthest-side: Gradient ends at the farthest side of the box
  • farthest-corner: Gradient ends at the farthest corner of the box (default)

Conic Gradients

Conic gradients transition colors around a center point (like a color wheel):

/* Default (center, starting at top) */
.conic-default {
  background-image: conic-gradient(red, yellow, green, blue, purple, red);
}

/* With position */
.conic-position {
  background-image: conic-gradient(at 70% 30%, red, yellow, green, blue);
}

/* With starting angle */
.conic-angle {
  background-image: conic-gradient(from 45deg, red, yellow, green, blue);
}

/* Full syntax */
.conic-full {
  background-image: conic-gradient(from 90deg at 60% 40%, red, yellow, green, blue);
}

/* Pie chart effect */
.pie-chart {
  background-image: conic-gradient(
    red 0deg, red 90deg,
    yellow 90deg, yellow 180deg,
    green 180deg, green 270deg,
    blue 270deg, blue 360deg
  );
  border-radius: 50%;
}

Repeating Gradients

Repeating gradients create patterns by repeating color stops:

/* Repeating linear gradient */
.repeating-linear {
  background-image: repeating-linear-gradient(
    45deg,
    #f5f5f5,
    #f5f5f5 10px,
    #e5e5e5 10px,
    #e5e5e5 20px
  );
}

/* Repeating radial gradient */
.repeating-radial {
  background-image: repeating-radial-gradient(
    circle,
    #f5f5f5,
    #f5f5f5 10px,
    #e5e5e5 10px,
    #e5e5e5 20px
  );
}

/* Repeating conic gradient */
.repeating-conic {
  background-image: repeating-conic-gradient(
    #f5f5f5 0deg,
    #f5f5f5 15deg,
    #e5e5e5 15deg,
    #e5e5e5 30deg
  );
}

Background Patterns and Effects

CSS backgrounds can be combined to create complex patterns and effects.

Striped Backgrounds

Create striped patterns using linear gradients:

/* Horizontal stripes */
.horizontal-stripes {
  background-image: linear-gradient(
    #f5f5f5 50%,
    #e5e5e5 50%
  );
  background-size: 100% 20px;
}

/* Vertical stripes */
.vertical-stripes {
  background-image: linear-gradient(
    to right,
    #f5f5f5 50%,
    #e5e5e5 50%
  );
  background-size: 20px 100%;
}

/* Diagonal stripes */
.diagonal-stripes {
  background-image: linear-gradient(
    45deg,
    #f5f5f5 25%,
    #e5e5e5 25%,
    #e5e5e5 50%,
    #f5f5f5 50%,
    #f5f5f5 75%,
    #e5e5e5 75%
  );
  background-size: 40px 40px;
}

Checkerboard Pattern

Create a checkerboard pattern using multiple gradients:

.checkerboard {
  background-image:
    linear-gradient(45deg, #ccc 25%, transparent 25%),
    linear-gradient(-45deg, #ccc 25%, transparent 25%),
    linear-gradient(45deg, transparent 75%, #ccc 75%),
    linear-gradient(-45deg, transparent 75%, #ccc 75%);
  background-size: 20px 20px;
  background-position: 0 0, 0 10px, 10px 0, 10px 10px;
}

Polka Dot Pattern

Create a polka dot pattern using radial gradients:

.polka-dots {
  background-image: radial-gradient(#ccc 20%, transparent 20%);
  background-size: 20px 20px;
}

/* Offset pattern for more dots */
.more-dots {
  background-image: 
    radial-gradient(#ccc 20%, transparent 20%),
    radial-gradient(#ccc 20%, transparent 20%);
  background-size: 20px 20px;
  background-position: 0 0, 10px 10px;
}

Grid Pattern

Create a grid pattern using linear gradients:

.grid {
  background-image:
    linear-gradient(#ccc 1px, transparent 1px),
    linear-gradient(to right, #ccc 1px, transparent 1px);
  background-size: 20px 20px;
}

Layered Backgrounds

Combine multiple backgrounds for complex effects:

.layered {
  background:
    linear-gradient(rgba(0, 0, 0, 0.5), rgba(0, 0, 0, 0.5)), /* Dark overlay */
    url('texture.png'), /* Texture */
    linear-gradient(to right, #3494e6, #ec6ead); /* Gradient base */
}

Frosted Glass Effect

Create a frosted glass effect using backdrop-filter:

.frosted-glass {
  background-color: rgba(255, 255, 255, 0.2);
  backdrop-filter: blur(10px);
  -webkit-backdrop-filter: blur(10px);
}

Performance Considerations

Complex background patterns and effects can impact performance, especially on mobile devices. Consider these tips:

  • Use CSS patterns instead of image files when possible for better performance
  • Limit the number of background layers
  • Be cautious with backdrop-filter, as it can be GPU-intensive
  • Test on lower-end devices to ensure good performance

Color Theory and Accessibility

Understanding color theory and accessibility considerations is essential for effective web design.

Color Schemes

Common color schemes used in web design:

  • Monochromatic: Different shades and tints of a single hue
  • Analogous: Colors that are adjacent on the color wheel
  • Complementary: Colors opposite each other on the color wheel
  • Split Complementary: A base color and two colors adjacent to its complement
  • Triadic: Three colors equally spaced around the color wheel
  • Tetradic: Four colors arranged in two complementary pairs

Color Contrast

Sufficient color contrast is crucial for readability and accessibility:

  • WCAG 2.1 Level AA requires a contrast ratio of at least 4.5:1 for normal text
  • Large text (18pt or 14pt bold) requires a contrast ratio of at least 3:1
  • UI components and graphical objects require a contrast ratio of at least 3:1
/* Good contrast for body text */
body {
  color: #333; /* Dark gray text */
  background-color: #fff; /* White background */
}

/* Improved contrast for links */
a {
  color: #0056b3; /* Darker blue for better contrast */
}

Color and User Preferences

Respect user preferences for color schemes:

/* Detect user's preference for dark mode */
@media (prefers-color-scheme: dark) {
  body {
    color: #f0f0f0;
    background-color: #121212;
  }
  
  a {
    color: #80b3ff;
  }
}

/* Detect user's preference for reduced motion */
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}

Color Blindness Considerations

About 8% of men and 0.5% of women have some form of color blindness. Design with this in mind:

  • Don't rely solely on color to convey information
  • Avoid problematic color combinations (red/green, blue/purple, green/brown)
  • Use patterns, shapes, or labels in addition to color
  • Test your designs with color blindness simulators
/* Using patterns with color */
.success {
  background-color: #4caf50; /* Green */
  background-image: linear-gradient(45deg, rgba(255, 255, 255, 0.15) 25%, transparent 25%, transparent 50%, rgba(255, 255, 255, 0.15) 50%, rgba(255, 255, 255, 0.15) 75%, transparent 75%, transparent);
  background-size: 20px 20px;
}

.error {
  background-color: #f44336; /* Red */
  background-image: linear-gradient(-45deg, rgba(255, 255, 255, 0.15) 25%, transparent 25%, transparent 50%, rgba(255, 255, 255, 0.15) 50%, rgba(255, 255, 255, 0.15) 75%, transparent 75%, transparent);
  background-size: 20px 20px;
}
