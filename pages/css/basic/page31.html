Layout Basics

Understanding CSS Layout

Layout is one of the most important aspects of web design. It determines how elements are positioned and arranged on the page, affecting both the visual appearance and user experience of your website.

CSS offers several layout methods, from traditional techniques like normal flow, floats, and positioning to modern approaches like Flexbox and Grid. In this lesson, we'll focus on the fundamental layout concepts that form the basis for all CSS layouts.

Understanding these basics will help you create well-structured, responsive layouts and prepare you for the more advanced layout techniques we'll cover in subsequent lessons.

Layout Evolution

CSS layout techniques have evolved significantly over the years:

  • Early days (1990s): Table-based layouts, limited CSS support
  • CSS2 era (2000s): Float-based layouts, positioning, clearfix hacks
  • Modern era (2010s+): Flexbox, Grid, and other purpose-built layout tools

While modern techniques like Flexbox and Grid are preferred for new projects, understanding the fundamentals is still important for maintaining legacy code and building a solid foundation.

The CSS Visual Formatting Model

The CSS Visual Formatting Model is the algorithm that browsers use to process and display documents. It determines how elements are laid out based on several factors:

  • The box model (content, padding, border, margin)
  • The display property (block, inline, inline-block, etc.)
  • The position property (static, relative, absolute, fixed)
  • The float property
  • The containing block (parent element)
  • The document flow

Normal Flow

By default, elements are laid out according to normal flow, where:

  • Block-level elements (like div, p, h1-h6) create a new line and take up the full width available
  • Inline elements (like span, a, strong) flow within the text and only take up as much width as necessary
/* Block-level elements */
div, p, h1, h2, h3, h4, h5, h6, ul, ol, li, section, article, header, footer, nav {
  display: block;
}

/* Inline elements */
span, a, strong, em, img, br, input, button {
  display: inline;
}

The Display Property

The display property determines how an element is treated in the layout flow:

/* Block: Creates a new line, takes full width available */
.block {
  display: block;
}

/* Inline: Flows within text, takes only necessary width */
.inline {
  display: inline;
}

/* Inline-block: Flows within text but can have width/height */
.inline-block {
  display: inline-block;
  width: 100px;
  height: 100px;
}

/* None: Removes the element from the document flow */
.hidden {
  display: none;
}

/* Table-related display values */
.table {
  display: table;
}
.table-row {
  display: table-row;
}
.table-cell {
  display: table-cell;
}

/* Flex and Grid (covered in detail in later lessons) */
.flex-container {
  display: flex;
}
.grid-container {
  display: grid;
}
Display Value Description Common Use Cases
block Creates a block-level element Page sections, paragraphs, headings
inline Creates an inline element Text-level elements, links, spans
inline-block Creates an inline-level block container Buttons, input fields, navigation items
none Removes the element from the layout Hiding elements, toggling visibility
flex Creates a flex container One-dimensional layouts (rows or columns)
grid Creates a grid container Two-dimensional layouts (rows and columns)

Visibility vs. Display: None

There's an important difference between display: none and visibility: hidden:

  • display: none removes the element from the document flow, and it takes up no space
  • visibility: hidden hides the element but preserves its space in the layout
.removed {
  display: none; /* Element disappears completely */
}

.invisible {
  visibility: hidden; /* Element is invisible but still takes up space */
}

CSS Positioning

The position property determines how an element is positioned in the document. It works together with the top, right, bottom, and left properties to place elements precisely.

Static Positioning

The default positioning mode is static, where elements follow the normal document flow:

.static {
  position: static; /* Default value */
}

/* The top, right, bottom, and left properties have no effect on static elements */
.static {
  position: static;
  top: 20px; /* This will be ignored */
  left: 20px; /* This will be ignored */
}

Relative Positioning

With relative positioning, an element is positioned relative to its normal position:

.relative {
  position: relative;
  top: 20px;   /* Moves down 20px from normal position */
  left: 20px;  /* Moves right 20px from normal position */
}

/* Negative values move in the opposite direction */
.relative-negative {
  position: relative;
  top: -20px;  /* Moves up 20px from normal position */
  right: 20px; /* Moves left 20px from normal position */
}

Key points about relative positioning:

  • The element still occupies its original space in the document flow
  • Other elements are not affected by the repositioning
  • The element is offset relative to itself
  • Relative positioning also creates a new containing block for absolutely positioned children

Absolute Positioning

With absolute positioning, an element is positioned relative to its nearest positioned ancestor (or the document body if none exists):

.container {
  position: relative; /* Creates a positioning context */
  height: 200px;
  width: 200px;
}

.absolute {
  position: absolute;
  top: 20px;    /* 20px from the top of the container */
  right: 20px;  /* 20px from the right of the container */
}

/* Centering with absolute positioning */
.centered {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%); /* Adjusts for the element's dimensions */
}

Key points about absolute positioning:

  • The element is removed from the normal document flow
  • Other elements act as if the absolutely positioned element doesn't exist
  • The element is positioned relative to its nearest positioned ancestor (an element with position set to anything other than static)
  • If no positioned ancestor exists, it's positioned relative to the initial containing block (usually the viewport)

Fixed Positioning

With fixed positioning, an element is positioned relative to the viewport and doesn't move when the page is scrolled:

.fixed-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  background-color: white;
  box-shadow: 0 2px 5px rgba(0, 0, 0, 0.1);
}

.fixed-button {
  position: fixed;
  bottom: 20px;
  right: 20px;
}

Key points about fixed positioning:

  • The element is removed from the normal document flow
  • It's positioned relative to the viewport
  • It stays in the same position even when the page is scrolled
  • Common uses include fixed headers, footers, and "back to top" buttons

Sticky Positioning

With sticky positioning, an element is treated as relative until it crosses a specified threshold, then it's treated as fixed:

.sticky-header {
  position: sticky;
  top: 0; /* Sticks to the top when scrolled to this position */
  background-color: white;
  z-index: 100;
}

.sticky-sidebar {
  position: sticky;
  top: 20px; /* Starts sticking when the top is 20px from the viewport top */
}

Key points about sticky positioning:

  • The element is treated as relatively positioned until it crosses a specified threshold
  • Then it's treated as fixed until its parent is out of view
  • It requires at least one threshold value (top, right, bottom, or left)
  • Common uses include sticky headers, navigation, and section headings
Position Value Positioned Relative To In Document Flow Common Use Cases
static Normal flow Yes Default positioning for most elements
relative Its normal position Yes Small adjustments, creating positioning context
absolute Nearest positioned ancestor No Tooltips, modals, badges, precise positioning
fixed Viewport No Fixed headers/footers, floating buttons
sticky Normal flow, then viewport Yes, until "stuck" Sticky headers, persistent navigation

Z-Index and Stacking Context

The z-index property controls the vertical stacking order of elements that overlap. It only works on positioned elements (those with a position value other than static).

.behind {
  position: relative;
  z-index: 1;
}

.middle {
  position: relative;
  z-index: 2; /* Higher z-index appears on top */
}

.front {
  position: relative;
  z-index: 3;
}

Elements with higher z-index values appear in front of elements with lower values. If elements have the same z-index, the one that appears later in the HTML code is displayed on top.

Stacking Context

A stacking context is formed by certain CSS properties, creating a hierarchy that affects how z-index works:

/* These properties create a new stacking context */
.new-context {
  position: relative; /* or absolute, fixed, sticky */
  z-index: 1;
}

.also-new-context {
  opacity: 0.9; /* Values less than 1 create a stacking context */
}

.another-context {
  transform: scale(1); /* Any transform creates a stacking context */
}

Other properties that create a stacking context include:

  • filter
  • mix-blend-mode (other than normal)
  • isolation: isolate
  • will-change (for certain properties)
  • contain: layout, paint, or strict

Z-index only works within the same stacking context. An element with a lower z-index in a higher stacking context will appear above an element with a higher z-index in a lower stacking context.

.parent1 {
  position: relative;
  z-index: 1;
}

.child1 {
  position: relative;
  z-index: 100; /* High value, but in a lower stacking context */
}

.parent2 {
  position: relative;
  z-index: 2; /* Higher than parent1 */
}

.child2 {
  position: relative;
  z-index: 1; /* Low value, but in a higher stacking context */
  /* This will appear above child1 despite having a lower z-index */
}

Z-Index Best Practices

To maintain manageable z-index values:

  • Use small, incremental values (1, 2, 3) instead of large numbers (999, 9999)
  • Create a z-index scale in your CSS variables or comments
  • Be aware of stacking contexts when debugging z-index issues
  • Use the minimum z-index necessary to achieve the desired stacking

Float-Based Layouts

The float property was one of the primary layout tools before Flexbox and Grid. It moves an element to the left or right, allowing text and inline elements to wrap around it.

Basic Floating

.float-left {
  float: left;
  margin-right: 15px; /* Space between the floated element and the text */
}

.float-right {
  float: right;
  margin-left: 15px;
}

/* Clear floats */
.clear {
  clear: both; /* Can also be 'left' or 'right' */
}

Floated elements are removed from the normal flow, but still affect the flow of text and inline elements around them.

Clearing Floats

When elements are floated, their containers may collapse if they contain only floated elements. There are several ways to clear floats:

/* Method 1: Clear element */
.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

/* Method 2: Overflow */
.container {
  overflow: auto; /* or hidden */
}

/* Method 3: Display flow-root */
.modern-clearfix {
  display: flow-root;
}

Float-Based Grid Systems

Before Flexbox and Grid, float-based grid systems were common for creating multi-column layouts:

/* Simple float-based grid */
.row::after {
  content: "";
  display: table;
  clear: both;
}

.col {
  float: left;
  padding: 0 15px;
}

.col-1 { width: 8.33%; }
.col-2 { width: 16.66%; }
.col-3 { width: 25%; }
.col-4 { width: 33.33%; }
.col-5 { width: 41.66%; }
.col-6 { width: 50%; }
.col-7 { width: 58.33%; }
.col-8 { width: 66.66%; }
.col-9 { width: 75%; }
.col-10 { width: 83.33%; }
.col-11 { width: 91.66%; }
.col-12 { width: 100%; }

Modern Alternatives to Floats

While floats are still useful for specific cases (like wrapping text around images), modern layout techniques are preferred for overall page layout:

  • Use display: flex for one-dimensional layouts (rows or columns)
  • Use display: grid for two-dimensional layouts (rows and columns)
  • Use position: absolute for precise positioning of elements

Multi-Column Layout

CSS Multi-column Layout is a module that allows you to create multiple columns of text, similar to newspaper layouts.

/* Specify the number of columns */
.multi-column {
  column-count: 3;
}

/* Or specify the width of columns */
.multi-column-width {
  column-width: 200px; /* Browser determines the number of columns */
}

/* Both together (browser will use the maximum number of columns that fit) */
.multi-column-both {
  column-count: 4;
  column-width: 200px;
}

/* Add space between columns */
.multi-column-gap {
  column-gap: 40px;
}

/* Add a line between columns */
.multi-column-rule {
  column-rule: 1px solid #ccc;
}

Spanning Columns

You can make elements span across all columns:

.column-span {
  column-span: all;
}

Controlling Breaks

You can control how content breaks across columns:

/* Prevent an element from breaking across columns */
.no-break {
  break-inside: avoid-column;
}

/* Force a column break before an element */
.new-column {
  break-before: column;
}

/* Force a column break after an element */
.end-column {
  break-after: column;
}

Box Sizing and Layout

The box-sizing property affects how element dimensions are calculated, which is crucial for layout:

/* Default box sizing */
.content-box {
  box-sizing: content-box;
  width: 300px;
  padding: 20px;
  border: 1px solid #ccc;
  /* Total width: 300px + 20px*2 + 1px*2 = 342px */
}

/* Alternative box sizing */
.border-box {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 1px solid #ccc;
  /* Total width: 300px (includes content, padding, and border) */
}

For consistent and predictable layouts, it's common to set box-sizing: border-box for all elements:

*, *::before, *::after {
  box-sizing: border-box;
}

Centering Elements

Centering elements is a common layout task. Here are various techniques for different scenarios:

Horizontal Centering

/* Center a block element */
.center-block {
  margin-left: auto;
  margin-right: auto;
  width: 50%; /* Must have a width */
}

/* Center text */
.center-text {
  text-align: center;
}

/* Center an inline or inline-block element */
.center-inline-parent {
  text-align: center;
}
.center-inline-child {
  display: inline-block;
}

Vertical Centering

/* Center with line-height (for single-line text) */
.center-line {
  line-height: 100px; /* Same as the height of the container */
}

/* Center with padding */
.center-padding {
  padding-top: 20px;
  padding-bottom: 20px;
}

/* Center with absolute positioning and transform */
.center-absolute-parent {
  position: relative;
}
.center-absolute {
  position: absolute;
  top: 50%;
  transform: translateY(-50%);
}

Both Horizontal and Vertical Centering

/* Center with absolute positioning and transform */
.center-both-parent {
  position: relative;
}
.center-both {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* Center with flexbox (modern approach) */
.center-flex {
  display: flex;
  justify-content: center; /* Horizontal centering */
  align-items: center;     /* Vertical centering */
}

/* Center with grid (modern approach) */
.center-grid {
  display: grid;
  place-items: center; /* Shorthand for align-items and justify-items */
}

Modern Centering

While all these techniques work, modern approaches using Flexbox or Grid are generally preferred:

  • They require less code
  • They're more intuitive
  • They work for multiple elements
  • They handle dynamic content better

We'll cover these in detail in the Flexbox and Grid lessons.

Viewport Units and Layout

Viewport units are relative to the browser viewport dimensions, making them useful for responsive layouts:

/* Full viewport height */
.full-height {
  height: 100vh; /* 100% of viewport height */
}

/* Full viewport width */
.full-width {
  width: 100vw; /* 100% of viewport width */
}

/* Half viewport */
.half {
  width: 50vw;
  height: 50vh;
}

/* Minimum of viewport width or height */
.square {
  width: 50vmin; /* 50% of the smaller viewport dimension */
  height: 50vmin;
}

/* Maximum of viewport width or height */
.responsive-size {
  width: 50vmax; /* 50% of the larger viewport dimension */
}

Viewport units are particularly useful for:

  • Full-screen sections
  • Hero areas
  • Maintaining aspect ratios
  • Responsive typography

Mobile Viewport Considerations

On mobile devices, viewport units can behave differently due to the dynamic viewport size when the address bar shows/hides or when the keyboard appears. Modern browsers now support additional viewport units to address these issues:

  • svh: Small Viewport Height (when UI elements are shown)
  • lvh: Large Viewport Height (when UI elements are hidden)
  • dvh: Dynamic Viewport Height (automatically adjusts)
.modern-full-height {
  height: 100dvh; /* Adjusts dynamically as viewport changes */
}

Aspect Ratio

The aspect-ratio property allows you to maintain a specific width-to-height ratio for elements:

/* Square (1:1 ratio) */
.square {
  aspect-ratio: 1 / 1;
  width: 100%;
}

/* 16:9 ratio (common for videos) */
.widescreen {
  aspect-ratio: 16 / 9;
  width: 100%;
}

/* 4:3 ratio */
.standard {
  aspect-ratio: 4 / 3;
  width: 100%;
}

This property is particularly useful for:

  • Responsive images and videos
  • Cards and UI elements that need to maintain proportions
  • Creating consistent grid items

Legacy Aspect Ratio Technique

Before aspect-ratio was widely supported, developers used the padding trick:

/* 16:9 aspect ratio using padding trick */
.aspect-ratio-container {
  position: relative;
  width: 100%;
  padding-top: 56.25%; /* 9/16 = 0.5625 = 56.25% */
}

.aspect-ratio-content {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}

Container Queries

Container queries allow you to style elements based on the size of their container rather than the viewport:

/* Define a container */
.card-container {
  container-type: inline-size;
  container-name: card;
}

/* Apply styles based on container width */
@container card (min-width: 400px) {
  .card-title {
    font-size: 1.5rem;
  }
}

@container card (max-width: 399px) {
  .card-title {
    font-size: 1.2rem;
  }
}

Container queries are particularly useful for:

  • Component-based design
  • Reusable UI elements that appear in different contexts
  • Creating truly responsive components regardless of viewport size

Browser Support

Container queries are a newer feature and may not be supported in all browsers. Check current browser support before using in production, and consider providing fallbacks for older browsers.

Common Layout Patterns

Here are some common layout patterns and how to implement them using the techniques we've covered:

Holy Grail Layout

A classic layout with header, footer, and three columns (left sidebar, main content, right sidebar):

/* Traditional approach with floats */
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
}

.header, .footer {
  clear: both;
}

.left-sidebar {
  float: left;
  width: 20%;
}

.main-content {
  float: left;
  width: 60%;
}

.right-sidebar {
  float: left;
  width: 20%;
}

/* Clear the floats */
.container::after {
  content: "";
  display: table;
  clear: both;
}

Card Layout

A grid of cards, commonly used for products, blog posts, etc.:

/* Using floats */
.card-container {
  margin: 0 -15px; /* Compensate for card padding */
}

.card-container::after {
  content: "";
  display: table;
  clear: both;
}

.card {
  float: left;
  width: 33.33%;
  padding: 0 15px;
  margin-bottom: 30px;
  box-sizing: border-box;
}

/* Responsive adjustments */
@media (max-width: 768px) {
  .card {
    width: 50%;
  }
}

@media (max-width: 480px) {
  .card {
    width: 100%;
  }
}

Sticky Footer

A footer that sticks to the bottom of the viewport when content is short:

/* Using minimum height and flexbox */
html, body {
  height: 100%;
  margin: 0;
}

body {
  display: flex;
  flex-direction: column;
}

.content {
  flex: 1 0 auto; /* Grow to fill available space */
}

.footer {
  flex-shrink: 0; /* Don't shrink */
}

Split Screen

A layout that divides the screen into two equal parts:

/* Using floats */
.split-container::after {
  content: "";
  display: table;
  clear: both;
}

.split-left, .split-right {
  float: left;
  width: 50%;
  height: 100vh;
}

/* Responsive adjustment */
@media (max-width: 768px) {
  .split-left, .split-right {
    float: none;
    width: 100%;
    height: auto;
    min-height: 50vh;
  }
}

In this lesson, we've explored the fundamental concepts of CSS layout:

  • Visual Formatting Model: How browsers process and display elements

  • Z-Index and Stacking Context: Managing the vertical stacking order of elements
  • Float-Based Layouts: Using floats for text wrapping and basic layouts
  • Multi-Column Layout: Creating newspaper-style columns of text
  • Centering Techniques: Various methods for centering elements horizontally and vertically
  • Viewport Units: Using viewport-relative measurements for responsive layouts
  • Aspect Ratio: Maintaining proportional dimensions for elements
  • Container Queries: Styling elements based on their container's size
  • Common Layout Patterns: Implementing frequently used layout structures

These layout fundamentals form the foundation for more advanced techniques. While some of these methods (like floats for layout) are considered legacy approaches, understanding them is valuable for maintaining existing code and appreciating the evolution of CSS layout.
