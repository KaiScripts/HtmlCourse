Introduction to CSS

What is CSS?

CSS (Cascading Style Sheets) is a style sheet language used to describe the presentation of a document written in HTML. While HTML provides the structure and content of a web page, CSS controls how that content looks and is laid out on the screen.

Think of it this way: if HTML is the skeleton of a web page, CSS is the skin, clothing, and makeup that gives it visual appeal and personality. CSS allows you to control colors, fonts, spacing, layout, animations, and much more.

CSS Evolution

CSS has evolved significantly since its introduction:

  • CSS1 (1996): Basic styling capabilities like fonts, colors, and margins
  • CSS2 (1998): Added positioning, z-index, media types, and bidirectional text
  • CSS3 (2001-present): Modular approach with features like animations, flexbox, grid, variables, and more

Unlike its predecessors, CSS3 is divided into modules that can be updated independently, allowing for more rapid evolution of the language.

Why CSS is Important

CSS is a fundamental technology of the web for several reasons:

  • Separation of Concerns: CSS allows you to separate content (HTML) from presentation (CSS), making your code more maintainable and easier to update
  • Consistency: With CSS, you can define styles once and apply them across your entire website, ensuring a consistent look and feel
  • Efficiency: CSS reduces redundancy by allowing you to define styles in one place instead of repeating them throughout your HTML
  • Responsive Design: CSS enables you to create websites that adapt to different screen sizes and devices
  • Accessibility: Proper CSS implementation can improve the accessibility of your website for users with disabilities
  • User Experience: CSS allows you to create visually appealing and intuitive interfaces that enhance the user experience

Before CSS: Styling in HTML

Before CSS was widely adopted, styling was done directly in HTML using attributes and tags that are now deprecated:

<!-- Styling before CSS (don't use these methods!) -->
<font face="Arial" color="blue" size="4">This text is blue Arial.</font>

<table bgcolor="#f0f0f0" width="100%" border="1">
  <tr>
    <td align="center">Centered cell content</td>
  </tr>
</table>

<body bgcolor="lightgray">
  <h1 align="center">Centered Heading</h1>
</body>

This approach had several problems:

  • Mixing content and presentation made HTML files larger and harder to maintain
  • Changing the design required updating every HTML file
  • Limited styling options compared to CSS
  • Inconsistent implementation across browsers

CSS vs. HTML: Separation of Concerns

Let's compare the old HTML styling approach with modern CSS:

HTML with Inline Styling (Bad Practice) HTML with External CSS (Good Practice) 
<h1 style="color: blue; font-family: Arial; text-align: center;">
  Welcome to My Website
</h1>
<p style="color: #333; font-size: 16px; line-height: 1.5;">
  This is a paragraph with inline styles.
</p>
 
<!-- HTML file -->
<h1 class="page-title">Welcome to My Website</h1>
<p class="content">This is a paragraph with external styles.</p>

<!-- CSS file (style.css) -->
.page-title {
  color: blue;
  font-family: Arial;
  text-align: center;
}
.content {
  color: #333;
  font-size: 16px;
  line-height: 1.5;
}

The CSS approach offers several advantages:

  • HTML is cleaner and focuses on content structure
  • Styles can be changed in one place and affect the entire website
  • Styles can be cached by browsers, improving load times
  • Different style sheets can be applied for different devices or user preferences

How CSS Works

CSS works by selecting HTML elements and applying styles to them. The browser reads the CSS and renders the HTML accordingly.

Basic CSS Syntax

CSS consists of rules that have two main parts: a selector and a declaration block.

selector {
  property: value;
  another-property: another-value;
}
  • Selector: Targets the HTML element(s) to style (e.g., h1, .class-name, #id-name)
  • Declaration Block: Contains one or more declarations enclosed in curly braces
  • Declaration: A property-value pair that specifies which style property to change and what value to assign to it
  • Property: The style attribute you want to change (e.g., color, font-size, margin)
  • Value: The setting you specify for the property (e.g., blue, 16px, 20px)

Here's a simple example:

h1 {
  color: blue;
  font-size: 24px;
  text-align: center;
}

This rule selects all <h1> elements and applies three style declarations: it sets the text color to blue, the font size to 24 pixels, and centers the text.

CSS Comments

Comments in CSS are enclosed between /* and */. They are ignored by browsers but can help you document your code:

/* This is a CSS comment */

/* 
 * This is a multi-line comment
 * that explains what the following CSS does
 */
 
body {
  font-family: Arial, sans-serif; /* Use Arial font if available, otherwise use any sans-serif font */
  line-height: 1.6; /* Set line height to 1.6 times the font size */
}

Ways to Include CSS in HTML

There are three ways to include CSS in an HTML document:

1. External CSS (Recommended)

CSS is written in a separate file with a .css extension and linked to the HTML document using the <link> element in the <head> section:

<!-- In the HTML file -->
<head>
  <link rel="stylesheet" href="styles.css">
</head>
/* In styles.css */
body {
  font-family: Arial, sans-serif;
  line-height: 1.6;
  color: #333;
}

h1 {
  color: #0066cc;
}

Advantages:

  • Separates HTML and CSS completely
  • Makes maintenance easier as styles are in one place
  • Reduces file size and improves load times when multiple pages use the same styles
  • Browser caching improves performance for returning visitors

2. Internal CSS (Style Element)

CSS is included within a <style> element in the <head> section of the HTML document:

<head>
  <style>
    body {
      font-family: Arial, sans-serif;
      line-height: 1.6;
      color: #333;
    }
    
    h1 {
      color: #0066cc;
    }
  </style>
</head>

Advantages:

  • No need for additional files
  • Useful for single-page websites or pages with unique styles
  • No extra HTTP requests to fetch external CSS files

3. Inline CSS (Style Attribute)

CSS is applied directly to individual HTML elements using the style attribute:

<h1 style="color: #0066cc; font-size: 28px;">Welcome to My Website</h1>
<p style="font-family: Arial, sans-serif; line-height: 1.6;">This is a paragraph with inline styles.</p>

Advantages:

  • Useful for applying unique styles to a single element
  • Can override other styles for quick testing or fixes
  • Necessary in some email templates where external CSS isn't fully supported

Best Practice: Use External CSS

While all three methods work, external CSS is generally considered the best practice for most websites because:

  • It maintains a clean separation between content and presentation
  • It makes your website more maintainable and scalable
  • It improves page load times through caching
  • It allows for consistent styling across multiple pages

Use internal CSS for page-specific styles that aren't used elsewhere, and reserve inline CSS for exceptional cases or dynamic styling with JavaScript.

CSS Terminology

Before diving deeper into CSS, let's familiarize ourselves with some key terminology:

Selectors

Selectors are patterns that match HTML elements. They tell the browser which elements should receive the specified styles.

/* Type selector (element name) */
p {
  color: navy;
}

/* Class selector (preceded by a dot) */
.highlight {
  background-color: yellow;
}

/* ID selector (preceded by a hash) */
#header {
  background-color: black;
  color: white;
}

/* Attribute selector */
input[type="text"] {
  border: 1px solid gray;
}

/* Pseudo-class selector */
a:hover {
  text-decoration: underline;
}

We'll explore selectors in much more detail in the next lesson.

Properties and Values

Properties are the style attributes you want to change, and values are the settings for those properties.

h1 {
  color: blue;          /* Property: color, Value: blue */
  font-size: 24px;      /* Property: font-size, Value: 24px */
  margin-bottom: 20px;  /* Property: margin-bottom, Value: 20px */
}

CSS has hundreds of properties, each controlling a different aspect of an element's appearance.

Units

CSS uses various units to specify values for properties like size, spacing, and positioning:

Unit Type Unit Description Example
Absolute Units px Pixels (1/96th of an inch) font-size: 16px;
pt Points (1/72nd of an inch) font-size: 12pt;
in Inches margin: 0.5in;
cm Centimeters padding: 1cm;
Relative Units % Percentage (relative to parent element) width: 50%;
em Relative to the font-size of the element margin: 2em;
rem Relative to the font-size of the root element padding: 1.5rem;
vw Viewport width (1% of viewport width) width: 50vw;
vh Viewport height (1% of viewport height) height: 100vh;
vmin 1% of the smaller dimension of the viewport font-size: 5vmin;
vmax 1% of the larger dimension of the viewport width: 80vmax;

For responsive design, relative units are generally preferred over absolute units because they scale better across different devices and screen sizes.

Colors

CSS provides several ways to specify colors:

/* Color names */
h1 {
  color: blue;
}

/* Hexadecimal values */
p {
  color: #ff0000; /* Red */
}

/* RGB values */
div {
  color: rgb(0, 255, 0); /* Green */
}

/* RGBA values (with alpha/transparency) */
button {
  background-color: rgba(0, 0, 255, 0.5); /* Semi-transparent blue */
}

/* HSL values (Hue, Saturation, Lightness) */
span {
  color: hsl(120, 100%, 50%); /* Green */
}

/* HSLA values (with alpha/transparency) */
a {
  color: hsla(240, 100%, 50%, 0.7); /* Semi-transparent blue */
}

We'll explore colors in more detail in a later lesson.

The Cascade

The "C" in CSS stands for "Cascading," which refers to how styles are applied when multiple rules target the same element. The cascade determines which styles take precedence.

Styles are applied in this order of importance (from highest to lowest):

  1. Importance: Styles with !important override other styles
  2. Specificity: More specific selectors override less specific ones
  3. Source Order: Later styles override earlier ones
/* Less specific selector */
p {
  color: blue;
}

/* More specific selector (overrides the above) */
.content p {
  color: red;
}

/* Using !important (overrides everything else) */
p {
  color: green !important; /* Avoid using !important when possible */
}

Avoid !important

While !important can be useful in certain situations, it's generally considered a bad practice because:

  • It breaks the natural cascade of CSS
  • It makes debugging more difficult
  • It can lead to specificity wars where you need increasingly powerful selectors

Instead of using !important, try to write more specific selectors or restructure your CSS to avoid conflicts.

Inheritance

Some CSS properties are inherited from parent elements to their children, while others are not.

body {
  font-family: Arial, sans-serif; /* Inherited by all elements inside body */
  color: #333; /* Inherited by all elements inside body */
}

div {
  border: 1px solid black; /* Not inherited by child elements */
  padding: 20px; /* Not inherited by child elements */
}

Properties that are typically inherited include:

  • Text properties (color, font-family, font-size, line-height, etc.)
  • List properties (list-style, list-style-type, etc.)
  • Some table properties (border-collapse, border-spacing)

Properties that are typically not inherited include:

  • Box model properties (width, height, margin, padding, border)
  • Positioning properties (position, top, right, bottom, left)
  • Background properties (background, background-color, etc.)

You can force inheritance using the inherit keyword:

div {
  border: 1px solid black;
}

p {
  border: inherit; /* Inherits the border from its parent div */
}

Browser Developer Tools

Modern web browsers include powerful developer tools that are invaluable for working with CSS. These tools allow you to inspect and modify CSS in real-time, making them essential for learning and debugging.

Accessing Developer Tools

You can access developer tools in most browsers using these methods:

  • Chrome, Edge, Firefox, Safari: Right-click on any element and select "Inspect" or "Inspect Element"
  • Keyboard shortcuts:
    • Windows/Linux: F12 or Ctrl+Shift+I
    • macOS: Cmd+Option+I

Key Features for CSS Development

Browser developer tools offer several features that are particularly useful for CSS:

1. Element Inspector

The Element Inspector shows the HTML structure of the page and allows you to select elements to view and edit their CSS properties.

2. Styles Panel

The Styles panel shows all CSS rules that apply to the selected element, including:

  • Applied styles (active)
  • Overridden styles (crossed out)
  • Inherited styles
  • User agent (browser default) styles

3. Computed Panel

The Computed panel shows the final computed values of all CSS properties for the selected element after the cascade and inheritance have been applied.

4. Box Model Visualization

Most developer tools include a visual representation of the box model for the selected element, showing content, padding, border, and margin dimensions.

5. CSS Changes in Real-Time

You can edit CSS properties directly in the developer tools and see the changes applied to the page in real-time. This is incredibly useful for experimenting with different styles.

Developer Tools Tip

Changes made in developer tools are temporary and will be lost when you refresh the page. To save your changes, you need to copy them to your actual CSS files.

Most developer tools have a feature to copy all changes you've made to CSS, which you can then paste into your stylesheet.

CSS Resets and Normalizers

Different browsers have different default styles for HTML elements. These inconsistencies can make it challenging to create a consistent look across all browsers.

To address this issue, web developers often use CSS resets or normalizers:

CSS Resets

CSS resets strip away all browser default styles, essentially giving you a blank slate to work with. This approach removes margins, paddings, font sizes, and other default styles.

A simple CSS reset might look like this:

/* Simple CSS Reset */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

html, body, div, span, h1, h2, h3, h4, h5, h6, p, blockquote, pre,
a, abbr, address, cite, code, del, em, img, ins, kbd, q, s, samp,
small, strong, sub, sup, var, b, i, dl, dt, dd, ol, ul, li,
fieldset, form, label, legend, table, caption, tbody, tfoot, thead, tr, th, td,
article, aside, canvas, details, figcaption, figure, footer, header, hgroup,
menu, nav, section, summary, time, mark, audio, video {
  margin: 0;
  padding: 0;
  border: 0;
  font-size: 100%;
  font: inherit;
  vertical-align: baseline;
}

/* HTML5 display-role reset for older browsers */
article, aside, details, figcaption, figure, footer, header, hgroup, menu, nav, section {
  display: block;
}

body {
  line-height: 1;
}

ol, ul {
  list-style: none;
}

blockquote, q {
  quotes: none;
}

table {
  border-collapse: collapse;
  border-spacing: 0;
}

Popular CSS reset libraries include:

  • Eric Meyer's Reset CSS
  • Reset CSS

CSS Normalizers

Unlike resets, normalizers preserve useful browser defaults while still ensuring consistency across browsers. They fix bugs and common inconsistencies without completely stripping styles.

The most popular CSS normalizer is Normalize.css, which:

  • Preserves useful browser defaults rather than erasing them
  • Normalizes styles for a wide range of HTML elements
  • Corrects bugs and common browser inconsistencies
  • Improves usability with subtle modifications
  • Explains the code with detailed comments

Modern Approaches

In recent years, more minimal approaches have gained popularity:

1. Modern Normalize

A smaller, more modern version of Normalize.css that focuses on newer browsers.

2. CSS Reset

A very minimal reset that includes just the essentials:

/* Modern minimal CSS reset */
*, *::before, *::after {
  box-sizing: border-box;
}

body, h1, h2, h3, h4, p, figure, blockquote, dl, dd {
  margin: 0;
}

ul, ol {
  list-style: none;
  padding: 0;
  margin: 0;
}

html:focus-within {
  scroll-behavior: smooth;
}

body {
  min-height: 100vh;
  text-rendering: optimizeSpeed;
  line-height: 1.5;
}

a {
  text-decoration: none;
  color: inherit;
}

img, picture {
  max-width: 100%;
  display: block;
}

input, button, textarea, select {
  font: inherit;
}

Which Approach Should You Use?

The choice between a reset, normalizer, or minimal approach depends on your project needs:

  • CSS Reset: Use when you want complete control and plan to define all styles from scratch
  • Normalize.css: Use when you want consistency across browsers while preserving useful defaults
  • Minimal Reset: Use for modern projects where you need just the essentials and don't need to support older browsers

For beginners, Normalize.css is often a good starting point as it provides consistency without requiring you to redefine every style from scratch.

Practice Exercise: Your First CSS

Let's put what we've learned into practice by creating a simple HTML page with CSS styling.

Exercise: Style a Simple Web Page

Create an HTML file with the following content:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First CSS Page</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <header>
    <h1>Welcome to My Website</h1>
    <nav>
      <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Services</a></li>
        <li><a href="#">Contact</a></li>
      </ul>
    </nav>
  </header>
  
  <main>
    <section class="hero">
      <h2>Learn Web Development</h2>
      <p>Welcome to my website where you can learn HTML and CSS from scratch.</p>
      <button class="cta-button">Get Started</button>
    </section>
    
    <section class="features">
      <div class="feature">
        <h3>HTML Basics</h3>
        <p>Learn the fundamentals of HTML, the backbone of the web.</p>
      </div>
      <div class="feature">
        <h3>CSS Styling</h3>
        <p>Master CSS to create beautiful and responsive websites.</p>
      </div>
      <div class="feature">
        <h3>Web Projects</h3>
        <p>Apply your knowledge by building real-world projects.</p>
      </div>
    </section>
  </main>
  
  <footer>
    <p>© 2025 My Website. All rights reserved.</p>
  </footer>
</body>
</html>

Now, create a CSS file named styles.css and add styles to make the page look attractive. Here's a starting point:

/* Basic reset */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

/* Base styles */
body {
  font-family: 'Arial', sans-serif;
  line-height: 1.6;
  color: #333;
}

/* Add your styles here */
/* Try styling the header, navigation, hero section, features, and footer */
/* Experiment with colors, fonts, spacing, and layout */

Challenge Yourself

Try to implement the following styles:

  1. Set a background color for the body
  2. Style the header with a different background color and some padding
  3. Make the navigation links display horizontally instead of vertically
  4. Add hover effects to the navigation links
  5. Center the hero section text and add some margin
  6. Style the CTA button with a background color, rounded corners, and hover effect
  7. Make the features section display the three features side by side
  8. Add some spacing between elements using margin and padding
  9. Style the footer with a different background color and centered text

Don't worry if you're not sure how to implement all of these yet—we'll cover these techniques in detail in the upcoming lessons. The goal is to experiment and get comfortable with CSS syntax.

  • CSS Syntax: Selectors, declaration blocks, properties, and values
  • Ways to Include CSS: External, internal, and inline CSS, with external being the recommended approach
