Flexbox

Introduction to Flexbox

CSS Flexbox (Flexible Box Layout) is a one-dimensional layout method designed for laying out items in rows or columns. It provides a more efficient way to distribute space and align items, even when their size is unknown or dynamic.

Flexbox solves many common layout problems that were difficult to achieve with traditional CSS techniques like floats and positioning. It's particularly well-suited for:

  • Navigation menus and toolbars
  • Card layouts
  • Form controls
  • Centering elements
  • Distributing space among items
  • Aligning items on one axis

In this lesson, we'll explore the core concepts of Flexbox and how to use it to create powerful, flexible layouts.

Flexbox vs. Traditional Layout Methods

Compared to traditional layout methods like floats and positioning, Flexbox offers several advantages:

  • Simplicity: Fewer lines of code for complex layouts
  • Flexibility: Items can grow or shrink based on available space
  • Direction independence: Works with both rows and columns
  • Alignment control: Easy vertical and horizontal alignment
  • Order control: Items can be visually reordered without changing HTML
  • Space distribution: Intelligent distribution of extra space

Flexbox Terminology

Before diving into Flexbox properties, it's important to understand the key terminology:

Flex Container and Flex Items

A flex container is an element with display: flex or display: inline-flex. Its direct children automatically become flex items.

.container {
  display: flex; /* Creates a block-level flex container */
}

.inline-container {
  display: inline-flex; /* Creates an inline-level flex container */
}

Main Axis and Cross Axis

Flexbox operates on two axes:

  • The main axis is the primary axis along which flex items are laid out. It's defined by the flex-direction property.
  • The cross axis is perpendicular to the main axis.

For example, if flex-direction is row, the main axis runs horizontally (left to right) and the cross axis runs vertically (top to bottom).

Main Start/End and Cross Start/End

These terms refer to the starting and ending points of the main and cross axes:

  • main start/main end: The start and end points of the main axis
  • cross start/cross end: The start and end points of the cross axis

The direction of these points depends on the flex-direction and writing mode (e.g., left-to-right or right-to-left).

Main Size and Cross Size

These terms refer to the dimensions of flex items:

  • main size: The width or height of a flex item along the main axis
  • cross size: The width or height of a flex item along the cross axis

Flexbox Axes Visualization

When flex-direction: row:

  • Main axis: horizontal (left to right)
  • Cross axis: vertical (top to bottom)
  • Main size: width
  • Cross size: height

When flex-direction: column:

  • Main axis: vertical (top to bottom)
  • Cross axis: horizontal (left to right)
  • Main size: height
  • Cross size: width

Flex Container Properties

These properties are applied to the flex container to control the overall layout behavior.

display

Defines an element as a flex container:

.container {
  display: flex; /* Creates a block-level flex container */
}

.inline-container {
  display: inline-flex; /* Creates an inline-level flex container */
}

flex-direction

Establishes the main axis, determining the direction in which flex items are placed:

.container {
  display: flex;
  
  /* Default: left to right in ltr, right to left in rtl */
  flex-direction: row;
  
  /* Right to left in ltr, left to right in rtl */
  flex-direction: row-reverse;
  
  /* Top to bottom */
  flex-direction: column;
  
  /* Bottom to top */
  flex-direction: column-reverse;
}

flex-wrap

Controls whether flex items are forced onto a single line or can wrap onto multiple lines:

.container {
  display: flex;
  
  /* Default: all items on one line */
  flex-wrap: nowrap;
  
  /* Items wrap onto multiple lines if needed */
  flex-wrap: wrap;
  
  /* Items wrap onto multiple lines in reverse order */
  flex-wrap: wrap-reverse;
}

flex-flow

A shorthand property for flex-direction and flex-wrap:

.container {
  display: flex;
  
  /* flex-flow:   */
  flex-flow: row nowrap; /* Default */
  flex-flow: column wrap;
  flex-flow: row-reverse wrap-reverse;
}

justify-content

Aligns flex items along the main axis, distributing extra space:

.container {
  display: flex;
  
  /* Default: items are packed toward the start */
  justify-content: flex-start;
  
  /* Items are packed toward the end */
  justify-content: flex-end;
  
  /* Items are centered along the line */
  justify-content: center;
  
  /* Items are evenly distributed with equal space between them */
  justify-content: space-between;
  
  /* Items are evenly distributed with equal space around them */
  justify-content: space-around;
  
  /* Items are evenly distributed with equal space between and around them */
  justify-content: space-evenly;
}

align-items

Aligns flex items along the cross axis:

.container {
  display: flex;
  
  /* Default: items are stretched to fill the container */
  align-items: stretch;
  
  /* Items are aligned to the start of the cross axis */
  align-items: flex-start;
  
  /* Items are aligned to the end of the cross axis */
  align-items: flex-end;
  
  /* Items are centered along the cross axis */
  align-items: center;
  
  /* Items are aligned such that their baselines align */
  align-items: baseline;
}

align-content

Aligns flex lines (when there are multiple lines) within the flex container along the cross axis:

.container {
  display: flex;
  flex-wrap: wrap; /* Must have multiple lines for align-content to work */
  
  /* Default: lines are stretched to fill the container */
  align-content: stretch;
  
  /* Lines are packed toward the start of the container */
  align-content: flex-start;
  
  /* Lines are packed toward the end of the container */
  align-content: flex-end;
  
  /* Lines are centered in the container */
  align-content: center;
  
  /* Lines are evenly distributed with equal space between them */
  align-content: space-between;
  
  /* Lines are evenly distributed with equal space around them */
  align-content: space-around;
  
  /* Lines are evenly distributed with equal space between and around them */
  align-content: space-evenly;
}
Property Purpose Default Value Applies To
display Defines a flex container N/A Container
flex-direction Sets the main axis direction row Container
flex-wrap Controls whether items wrap nowrap Container
flex-flow Shorthand for direction and wrap row nowrap Container
justify-content Aligns items on main axis flex-start Container
align-items Aligns items on cross axis stretch Container
align-content Aligns multiple lines stretch Container (multi-line)

Flex Item Properties

These properties are applied to individual flex items to control their behavior within the flex container.

order

Controls the order in which flex items appear, without changing the HTML structure:

.item {
  /* Default: items are laid out in source order */
  order: 0;
}

.first {
  /* Lower values appear earlier */
  order: -1;
}

.last {
  /* Higher values appear later */
  order: 1;
}

All flex items have a default order value of 0. Items with the same order value are laid out in source order.

flex-grow

Determines how much a flex item can grow relative to other items when there's extra space:

.item {
  /* Default: item doesn't grow */
  flex-grow: 0;
}

.grow {
  /* Item can grow */
  flex-grow: 1;
}

.grow-more {
  /* Item grows twice as much as .grow */
  flex-grow: 2;
}

The flex-grow value is a proportion. If all items have flex-grow: 1, they will share the extra space equally. If one item has flex-grow: 2 and others have flex-grow: 1, the first item will get twice as much extra space as the others.

flex-shrink

Determines how much a flex item can shrink relative to other items when there's not enough space:

.item {
  /* Default: item can shrink */
  flex-shrink: 1;
}

.no-shrink {
  /* Item doesn't shrink */
  flex-shrink: 0;
}

.shrink-more {
  /* Item shrinks twice as much as default */
  flex-shrink: 2;
}

Like flex-grow, the flex-shrink value is a proportion. Higher values mean the item will shrink more compared to other items when there's not enough space.

flex-basis

Sets the initial main size of a flex item before growing or shrinking:

.item {
  /* Default: size based on content */
  flex-basis: auto;
}

.basis-specific {
  /* Specific size */
  flex-basis: 200px;
}

.basis-percentage {
  /* Percentage of container */
  flex-basis: 25%;
}

.basis-content {
  /* Size based on content */
  flex-basis: content;
}

.basis-zero {
  /* No initial size, rely on flex-grow/shrink */
  flex-basis: 0;
}

The flex-basis property determines the initial size of an item before any growing or shrinking occurs. It can be a length, percentage, or keyword.

flex

A shorthand property for flex-grow, flex-shrink, and flex-basis:

.item {
  /* flex:    */
  flex: 0 1 auto; /* Default */
}

.flexible {
  /* Can grow and shrink, starting from auto */
  flex: 1 1 auto;
}

.fixed-width {
  /* Doesn't grow or shrink, fixed width */
  flex: 0 0 200px;
}

.grow-only {
  /* Can grow but not shrink, starting from auto */
  flex: 1 0 auto;
}

.equal-width {
  /* All items with this class will have equal width */
  flex: 1 1 0;
}

The flex shorthand is recommended over the individual properties. It also has special keyword values:

.item {
  /* Same as flex: 0 1 auto */
  flex: initial;
  
  /* Same as flex: 1 1 auto */
  flex: auto;
  
  /* Same as flex: 0 0 auto */
  flex: none;
  
  /* Same as flex: 1 1 0 */
  flex: 1;
}

align-self

Overrides the align-items value for individual flex items:

.container {
  display: flex;
  align-items: center; /* Centers all items by default */
}

.item {
  /* Default: inherits from container's align-items */
  align-self: auto;
}

.align-start {
  /* Overrides to align at the start */
  align-self: flex-start;
}

.align-end {
  /* Overrides to align at the end */
  align-self: flex-end;
}

.align-stretch {
  /* Overrides to stretch */
  align-self: stretch;
}

.align-baseline {
  /* Overrides to align at the baseline */
  align-self: baseline;
}
Property Purpose Default Value Applies To
order Controls item order 0 Item
flex-grow Controls item growth 0 Item
flex-shrink Controls item shrinking 1 Item
flex-basis Sets initial item size auto Item
flex Shorthand for grow, shrink, basis 0 1 auto Item
align-self Overrides cross-axis alignment auto Item

Common Flexbox Patterns

Let's explore some common layout patterns that are easy to achieve with Flexbox.

Horizontal Navigation

A simple horizontal navigation menu:

<nav class="main-nav">
  <ul>
    <li><a href="#">Home</a></li>
    <li><a href="#">About</a></li>
    <li><a href="#">Services</a></li>
    <li><a href="#">Contact</a></li>
  </ul>
</nav>
.main-nav ul {
  display: flex;
  list-style: none;
  padding: 0;
  margin: 0;
}

.main-nav li {
  margin-right: 20px;
}

.main-nav li:last-child {
  margin-right: 0;
}

/* Centered navigation */
.centered-nav ul {
  justify-content: center;
}

/* Right-aligned navigation */
.right-nav ul {
  justify-content: flex-end;
}

/* Evenly spaced navigation */
.spaced-nav ul {
  justify-content: space-between;
}

Card Layout

A flexible card layout that adjusts to available space:

<div class="card-container">
  <div class="card">Card 1</div>
  <div class="card">Card 2</div>
  <div class="card">Card 3</div>
  <div class="card">Card 4</div>
</div>
.card-container {
  display: flex;
  flex-wrap: wrap;
  gap: 20px; /* Modern spacing between items */
}

.card {
  flex: 1 1 300px; /* Grow, shrink, and min-width of 300px */
  padding: 20px;
  border: 1px solid #ccc;
  border-radius: 4px;
}

/* Fixed-width cards */
.fixed-cards .card {
  flex: 0 0 300px; /* No grow, no shrink, fixed width */
}

/* Equal-width cards */
.equal-cards .card {
  flex: 1; /* Equal growth */
}

Holy Grail Layout

The classic "holy grail" layout with header, footer, and three columns:

<div class="holy-grail">
  <header>Header</header>
  <div class="holy-grail-body">
    <nav class="holy-grail-nav">Navigation</nav>
    <main class="holy-grail-content">Main Content</main>
    <aside class="holy-grail-aside">Sidebar</aside>
  </div>
  <footer>Footer</footer>
</div>
.holy-grail {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

.holy-grail-body {
  display: flex;
  flex: 1;
}

.holy-grail-content {
  flex: 1;
}

.holy-grail-nav, .holy-grail-aside {
  flex: 0 0 200px;
}

/* Responsive adjustment */
@media (max-width: 768px) {
  .holy-grail-body {
    flex-direction: column;
  }
  
  .holy-grail-nav, .holy-grail-aside {
    flex: 0 0 auto;
  }
}

Sticky Footer

A footer that sticks to the bottom of the viewport when content is short:

<div class="page-container">
  <div class="content">
    <!-- Page content here -->
  </div>
  <footer>Footer</footer>
</div>
.page-container {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

.content {
  flex: 1; /* Grows to fill available space */
}

Centering Content

Perfect centering (both horizontally and vertically):

<div class="center-container">
  <div class="centered-content">Centered Content</div>
</div>
.center-container {
  display: flex;
  justify-content: center; /* Horizontal centering */
  align-items: center;     /* Vertical centering */
  height: 100vh;           /* Full viewport height */
}

Split Screen

A layout that divides the screen into two equal parts:

<div class="split-container">
  <div class="split-left">Left Content</div>
  <div class="split-right">Right Content</div>
</div>
.split-container {
  display: flex;
  height: 100vh;
}

.split-left, .split-right {
  flex: 1; /* Equal width */
}

/* Unequal split */
.unequal-split .split-left {
  flex: 2; /* Takes up 2/3 of the space */
}

.unequal-split .split-right {
  flex: 1; /* Takes up 1/3 of the space */
}

/* Responsive adjustment */
@media (max-width: 768px) {
  .split-container {
    flex-direction: column;
  }
}

Advanced Flexbox Techniques

Let's explore some more advanced Flexbox techniques and patterns.

Nested Flexbox

You can nest flex containers to create complex layouts:

<div class="outer-flex">
  <div class="left-section">
    <div class="inner-flex">
      <div class="inner-item">Item 1</div>
      <div class="inner-item">Item 2</div>
    </div>
  </div>
  <div class="right-section">
    <div class="inner-flex">
      <div class="inner-item">Item 3</div>
      <div class="inner-item">Item 4</div>
    </div>
  </div>
</div>
.outer-flex {
  display: flex;
}

.left-section, .right-section {
  flex: 1;
}

.inner-flex {
  display: flex;
  flex-direction: column;
}

.inner-item {
  margin: 10px;
}

Flexbox and Auto Margins

Auto margins in Flexbox can create powerful alignment effects:

<div class="toolbar">
  <div class="left-tools">
    <button>New</button>
    <button>Open</button>
    <button>Save</button>
  </div>
  <div class="right-tools">
    <button>Settings</button>
    <button>Profile</button>
  </div>
</div>
.toolbar {
  display: flex;
  padding: 10px;
  background-color: #f5f5f5;
}

.right-tools {
  margin-left: auto; /* Pushes right-tools to the right */
}

/* Alternative approach */
.toolbar {
  display: flex;
  justify-content: space-between; /* Spreads items to edges */
}

Flexbox and Absolute Positioning

Combining Flexbox with absolute positioning can create interesting layouts:

<div class="card">
  <div class="card-content">
    <h3>Card Title</h3>
    <p>Card description text...</p>
  </div>
  <div class="card-badge">New</div>
</div>
.card {
  position: relative;
  display: flex;
  flex-direction: column;
  padding: 20px;
  border: 1px solid #ccc;
}

.card-badge {
  position: absolute;
  top: -10px;
  right: -10px;
  padding: 5px 10px;
  background-color: red;
  color: white;
  border-radius: 20px;
}

Flexbox and CSS Grid Together

Flexbox and Grid can be used together for powerful layouts:

<div class="grid-container">
  <header>Header</header>
  <nav>Navigation</nav>
  <main>
    <div class="flex-container">
      <div class="flex-item">Item 1</div>
      <div class="flex-item">Item 2</div>
      <div class="flex-item">Item 3</div>
    </div>
  </main>
  <footer>Footer</footer>
</div>
.grid-container {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header"
    "nav main"
    "footer footer";
  min-height: 100vh;
}

header { grid-area: header; }
nav { grid-area: nav; }
main { grid-area: main; }
footer { grid-area: footer; }

.flex-container {
  display: flex;
  flex-wrap: wrap;
  gap: 20px;
}

.flex-item {
  flex: 1 1 300px;
}

Responsive Flexbox Without Media Queries

Flexbox can create responsive layouts without media queries:

<div class="responsive-container">
  <div class="responsive-item">Item 1</div>
  <div class="responsive-item">Item 2</div>
  <div class="responsive-item">Item 3</div>
  <div class="responsive-item">Item 4</div>
</div>
.responsive-container {
  display: flex;
  flex-wrap: wrap;
}

.responsive-item {
  flex: 1 1 300px; /* Min width of 300px, but can grow and shrink */
  margin: 10px;
  padding: 20px;
  border: 1px solid #ccc;
}

/* Items will automatically wrap to new lines when container width is less than 
   (300px * number of items) + (margins and borders) */

Flexbox Browser Support and Fallbacks

Flexbox is well-supported in modern browsers, but you may need fallbacks for older browsers.

Browser Support

Flexbox is supported in all modern browsers, including:

  • Chrome 29+
  • Firefox 22+
  • Safari 6.1+
  • Edge (all versions)
  • Opera 12.1+
  • iOS Safari 7.1+
  • Android Browser 4.4+

Providing Fallbacks

For older browsers, you can provide fallbacks using feature detection or older layout methods:

/* Fallback for older browsers */
.container {
  display: block;
  text-align: center; /* Center inline elements */
}

.item {
  display: inline-block;
  width: 30%;
  margin: 0 1.5%;
  vertical-align: top;
}

/* Modern browsers with Flexbox support */
@supports (display: flex) {
  .container {
    display: flex;
    justify-content: center;
    text-align: left;
  }
  
  .item {
    flex: 0 0 30%;
    margin: 0 1.5%;
    width: auto;
  }
}

Vendor Prefixes

Some older browsers may require vendor prefixes for Flexbox properties:

.container {
  display: -webkit-box;
  display: -ms-flexbox;
  display: flex;
  
  -webkit-box-pack: center;
  -ms-flex-pack: center;
  justify-content: center;
}

/* Using a preprocessor or autoprefixer is recommended instead of 
   manually adding all prefixes */

Autoprefixer

Instead of manually adding vendor prefixes, it's recommended to use tools like Autoprefixer, which automatically adds the necessary prefixes based on browser support data.

With build tools like webpack, Gulp, or Parcel, you can integrate Autoprefixer into your workflow to handle vendor prefixes automatically.

Flexbox Best Practices

Here are some best practices to follow when working with Flexbox:

Use the Shorthand Properties

Use the flex shorthand instead of individual properties when possible:

/* Instead of this */
.item {
  flex-grow: 1;
  flex-shrink: 1;
  flex-basis: 0;
}

/* Use this */
.item {
  flex: 1 1 0; /* Or simply flex: 1; */
}

Be Mindful of Flex Direction

Remember that justify-content and align-items work along different axes depending on the flex direction:

/* In row direction */
.row {
  display: flex;
  flex-direction: row;
  justify-content: center; /* Horizontal centering */
  align-items: center;     /* Vertical centering */
}

/* In column direction */
.column {
  display: flex;
  flex-direction: column;
  justify-content: center; /* Vertical centering */
  align-items: center;     /* Horizontal centering */
}

Use Flexbox for One-Dimensional Layouts

Flexbox is designed for one-dimensional layouts (rows OR columns). For two-dimensional layouts (rows AND columns), consider using CSS Grid:

/* Good use of Flexbox: Navigation menu (one row) */
.nav {
  display: flex;
}

/* Better with Grid: Photo gallery (rows and columns) */
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 20px;
}

Combine with Other Layout Methods

Flexbox works well in combination with other layout methods:

/* Grid for overall layout, Flexbox for components */
.page {
  display: grid;
  grid-template-columns: 1fr 3fr;
  grid-template-rows: auto 1fr auto;
}

.nav {
  display: flex; /* Flexbox for the navigation */
}

/* Absolute positioning within flex items */
.card {
  position: relative;
  display: flex;
  flex-direction: column;
}

.badge {
  position: absolute;
  top: 0;
  right: 0;
}

Be Careful with Flex-Basis and Box-Sizing

Remember that flex-basis is affected by box-sizing:

/* With box-sizing: content-box (default) */
.item {
  flex-basis: 100px;
  padding: 20px;
  /* Total width: 100px + 40px padding = 140px */
}

/* With box-sizing: border-box */
.item {
  box-sizing: border-box;
  flex-basis: 100px;
  padding: 20px;
  /* Total width: 100px (includes content and padding) */
}

Use Gap for Spacing

Modern browsers support the gap property for Flexbox, which is cleaner than using margins:

/* Old approach with margins */
.container {
  display: flex;
}
.item {
  margin-right: 20px;
}
.item:last-child {
  margin-right: 0;
}

/* Modern approach with gap */
.container {
  display: flex;
  gap: 20px; /* Adds space between items, not outside */
}

Consider Accessibility

Be careful when changing the visual order of elements with Flexbox, as it can affect accessibility:

/* This can create a disconnect between visual and tab order */
.container {
  display: flex;
}
.first {
  order: 2; /* Visually appears second */
}
.second {
  order: 1; /* Visually appears first */
}

Screen readers and keyboard navigation follow the source order, not the visual order. Use order judiciously and test with keyboard navigation.

Summary

In this lesson, we've explored CSS Flexbox, a powerful layout system for creating one-dimensional layouts:

  • Flexbox Terminology: Understanding flex containers, flex items, main axis, cross axis, and related concepts
  • Flex Container Properties: Controlling the overall layout with properties like flex-direction, flex-wrap, justify-content, and align-items
  • Flex Item Properties: Controlling individual items with properties like flex-grow, flex-shrink, flex-basis, order, and align-self
  • Common Flexbox Patterns: Implementing navigation menus, card layouts, holy grail layouts, and more
  • Advanced Techniques: Combining Flexbox with other CSS features for complex layouts
  • Browser Support and Fallbacks: Ensuring compatibility with older browsers
  • Best Practices: Following recommended approaches for efficient and maintainable Flexbox layouts

Flexbox has revolutionized CSS layout, making it much easier to create flexible, responsive designs with clean, logical code. It's particularly well-suited for one-dimensional layouts like navigation menus, card layouts, and form controls.
