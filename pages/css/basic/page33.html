CSS Grid

Introduction to CSS Grid

CSS Grid Layout is a two-dimensional layout system designed specifically for creating grid-based user interfaces. Unlike Flexbox, which is primarily one-dimensional, Grid allows you to control both rows and columns simultaneously, making it perfect for overall page layouts and complex component designs.

CSS Grid provides a level of control that was previously impossible without complex hacks or JavaScript. It's particularly well-suited for:

  • Overall page layouts
  • Card-based interfaces
  • Image galleries
  • Form layouts
  • Dashboard interfaces
  • Any design that requires precise alignment in two dimensions

In this lesson, we'll explore the core concepts of CSS Grid and how to use it to create powerful, flexible layouts.

Grid vs. Flexbox

Grid and Flexbox are complementary, not competing technologies:

  • Grid is designed for two-dimensional layouts (rows AND columns)
  • Flexbox is designed for one-dimensional layouts (rows OR columns)

A good rule of thumb:

  • Use Grid for the overall page layout
  • Use Flexbox for UI elements, navigation, and components
  • They can be nested within each other for complex layouts

Grid Terminology

Before diving into Grid properties, it's important to understand the key terminology:

Grid Container and Grid Items

A grid container is an element with display: grid or display: inline-grid. Its direct children automatically become grid items.

.container {
  display: grid; /* Creates a block-level grid container */
}

.inline-container {
  display: inline-grid; /* Creates an inline-level grid container */
}

Grid Lines

Grid lines are the horizontal and vertical dividing lines that make up the structure of the grid. They are numbered starting from 1, and can also be named.

Grid Tracks

Grid tracks are the spaces between adjacent grid lines, creating rows and columns.

Grid Cells

A grid cell is the space between four adjacent grid lines, similar to a cell in a table.

Grid Areas

A grid area is a rectangular area bounded by four grid lines. It can span multiple cells both horizontally and vertically.

Gutters

Gutters (also called gaps) are the spaces between grid tracks.

Grid Visualization

To help visualize the grid structure, you can use the Firefox DevTools Grid Inspector, which shows grid lines, track numbers, and areas. Chrome DevTools also has grid inspection features.

In Firefox, right-click an element with display: grid and select "Inspect Element." Then, in the HTML pane, look for the grid icon next to the element.

Creating a Grid

Let's start by creating a basic grid and defining its structure.

Defining Columns and Rows

The grid-template-columns and grid-template-rows properties define the columns and rows of the grid:

/* Basic grid with fixed-width columns */
.container {
  display: grid;
  grid-template-columns: 200px 200px 200px; /* Three columns, each 200px wide */
  grid-template-rows: 100px 100px; /* Two rows, each 100px tall */
}

/* Grid with mixed units */
.mixed-grid {
  display: grid;
  grid-template-columns: 1fr 200px 2fr; /* Flexible, fixed, and more flexible columns */
  grid-template-rows: auto 100px; /* Auto-sized and fixed-height rows */
}

The fr Unit

The fr unit represents a fraction of the available space in the grid container:

/* Equal-width columns */
.equal-columns {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* Three equal columns */
}

/* Proportional columns */
.proportional-columns {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr; /* Middle column takes twice the space */
}

/* Mixed with fixed widths */
.mixed-columns {
  display: grid;
  grid-template-columns: 200px 1fr 200px; /* Fixed-flexible-fixed */
}

The repeat() Function

The repeat() function makes it easier to create repetitive patterns:

/* Instead of writing 1fr 1fr 1fr 1fr */
.repeated-columns {
  display: grid;
  grid-template-columns: repeat(4, 1fr); /* Four equal columns */
}

/* Repeating patterns */
.pattern-columns {
  display: grid;
  grid-template-columns: repeat(3, 100px 1fr); /* Repeats 100px 1fr three times */
}

minmax() Function

The minmax() function defines a size range, with a minimum and maximum value:

.responsive-columns {
  display: grid;
  grid-template-columns: repeat(3, minmax(200px, 1fr)); /* Columns are at least 200px, but can grow */
}

.flexible-rows {
  display: grid;
  grid-template-rows: minmax(100px, auto) 200px; /* First row is at least 100px, but can grow */
}

Auto-fill and Auto-fit

These keywords create a responsive grid with a dynamic number of columns:

/* Auto-fill: Creates as many columns as possible, even empty ones */
.auto-fill-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}

/* Auto-fit: Creates as many columns as possible and stretches them to fill the space */
.auto-fit-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
}

Grid Gaps

The gap, row-gap, and column-gap properties add space between grid tracks:

.grid-with-gaps {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px; /* 20px gap between all rows and columns */
}

.custom-gaps {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  row-gap: 20px; /* Gap between rows */
  column-gap: 10px; /* Gap between columns */
}

/* Shorthand */
.shorthand-gaps {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px 10px; /* row-gap column-gap */
}

Legacy Gap Properties

Older browsers may require the legacy grid-gap, grid-row-gap, and grid-column-gap properties. Modern browsers support both the legacy and new properties.

.legacy-gaps {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-gap: 20px; /* Legacy property */
  gap: 20px; /* Modern property */
}

Placing Items on the Grid

Once you've defined your grid structure, you can control how items are placed within it.

Grid Lines and Placement

You can place items by specifying which grid lines they should start and end at:

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: repeat(3, 100px);
}

.item1 {
  grid-column-start: 1;
  grid-column-end: 3; /* Spans from line 1 to line 3 (2 columns) */
  grid-row-start: 1;
  grid-row-end: 2; /* Spans from line 1 to line 2 (1 row) */
}

/* Shorthand properties */
.item2 {
  grid-column: 3 / 4; /* From column line 3 to 4 */
  grid-row: 1 / 3; /* From row line 1 to 3 (spans 2 rows) */
}

/* Even shorter using the span keyword */
.item3 {
  grid-column: 1 / span 2; /* Starts at line 1 and spans 2 columns */
  grid-row: 2 / span 2; /* Starts at line 2 and spans 2 rows */
}

Negative Line Numbers

You can use negative line numbers to count from the end of the grid:

.item {
  grid-column: 1 / -1; /* Spans from first to last column line (full width) */
  grid-row: -3 / -1; /* Spans from third-last to last row line */
}

Named Grid Lines

You can name grid lines when defining your grid, then reference them for placement:

.container {
  display: grid;
  grid-template-columns: [sidebar-start] 200px [sidebar-end content-start] 1fr [content-end];
  grid-template-rows: [header-start] 100px [header-end main-start] 1fr [main-end footer-start] 100px [footer-end];
}

.header {
  grid-column: sidebar-start / content-end;
  grid-row: header-start / header-end;
}

.sidebar {
  grid-column: sidebar-start / sidebar-end;
  grid-row: main-start / footer-end;
}

.content {
  grid-column: content-start / content-end;
  grid-row: main-start / main-end;
}

.footer {
  grid-column: content-start / content-end;
  grid-row: footer-start / footer-end;
}

grid-area Property

The grid-area property is a shorthand for grid-row-start, grid-column-start, grid-row-end, and grid-column-end:

.item {
  /* grid-area: row-start / column-start / row-end / column-end */
  grid-area: 1 / 1 / 3 / 4;
}

Named Grid Areas

The grid-template-areas property allows you to create a visual representation of your layout:

.container {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: 100px 1fr 100px;
  grid-template-areas:
    "header header"
    "sidebar content"
    "footer footer";
}

.header {
  grid-area: header;
}

.sidebar {
  grid-area: sidebar;
}

.content {
  grid-area: content;
}

.footer {
  grid-area: footer;
}

You can use a period (.) to represent an empty cell:

.container {
  grid-template-areas:
    "header header header"
    "sidebar content ."
    "sidebar footer footer";
}

Grid Template Areas Rules

When using grid-template-areas, keep these rules in mind:

  • Each row must have the same number of cells
  • To span multiple columns, repeat the name
  • To span multiple rows, use the same name in multiple rows
  • Each area must form a rectangle (no L-shapes)
  • Areas cannot be disconnected (must be contiguous)

Grid Alignment

CSS Grid provides powerful alignment capabilities for both the grid container and individual items.

Aligning Grid Items

These properties control the alignment of items within their grid cells:

justify-items

Aligns items along the row axis (horizontally):

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  
  /* Default: items stretch to fill the cell */
  justify-items: stretch;
  
  /* Items are aligned to the start of the cell */
  justify-items: start;
  
  /* Items are aligned to the end of the cell */
  justify-items: end;
  
  /* Items are centered in the cell */
  justify-items: center;
}

align-items

Aligns items along the column axis (vertically):

.container {
  display: grid;
  grid-template-rows: repeat(3, 100px);
  
  /* Default: items stretch to fill the cell */
  align-items: stretch;
  
  /* Items are aligned to the start of the cell */
  align-items: start;
  
  /* Items are aligned to the end of the cell */
  align-items: end;
  
  /* Items are centered in the cell */
  align-items: center;
}

place-items

A shorthand for align-items and justify-items:

.container {
  display: grid;
  
  /* place-items: align-items justify-items */
  place-items: center; /* Centers items both vertically and horizontally */
  place-items: start end; /* Aligns items to the top-right of each cell */
}

Aligning Individual Items

These properties allow you to override the container's alignment for specific items:

justify-self

Aligns an individual item along the row axis:

.item {
  justify-self: start; /* Aligns this item to the start of its cell */
}

align-self

Aligns an individual item along the column axis:

.item {
  align-self: center; /* Centers this item vertically in its cell */
}

place-self

A shorthand for align-self and justify-self:

.item {
  place-self: center end; /* Aligns this item to the vertical center and right edge */
}

Aligning the Grid Within the Container

These properties control the alignment of the entire grid within the grid container (when the grid is smaller than the container):

justify-content

Aligns the grid along the row axis:

.container {
  display: grid;
  grid-template-columns: repeat(3, 100px); /* Grid is 300px wide */
  width: 500px; /* Container is 500px wide */
  
  /* Default: grid is aligned to the start */
  justify-content: start;
  
  /* Grid is aligned to the end */
  justify-content: end;
  
  /* Grid is centered */
  justify-content: center;
  
  /* Space is distributed between columns */
  justify-content: space-between;
  
  /* Space is distributed around columns */
  justify-content: space-around;
  
  /* Space is distributed evenly */
  justify-content: space-evenly;
}

align-content

Aligns the grid along the column axis:

.container {
  display: grid;
  grid-template-rows: repeat(3, 100px); /* Grid is 300px tall */
  height: 500px; /* Container is 500px tall */
  
  /* Default: grid is aligned to the start */
  align-content: start;
  
  /* Grid is aligned to the end */
  align-content: end;
  
  /* Grid is centered */
  align-content: center;
  
  /* Space is distributed between rows */
  align-content: space-between;
  
  /* Space is distributed around rows */
  align-content: space-around;
  
  /* Space is distributed evenly */
  align-content: space-evenly;
}

place-content

A shorthand for align-content and justify-content:

.container {
  display: grid;
  
  /* place-content: align-content justify-content */
  place-content: center; /* Centers the grid both vertically and horizontally */
  place-content: space-between space-around; /* Different distribution for rows and columns */
}
Property Applies To Aligns Default Value
justify-items Grid Container Items horizontally within cells stretch
align-items Grid Container Items vertically within cells stretch
justify-self Grid Item Item horizontally within its cell auto (inherits from container)
align-self Grid Item Item vertically within its cell auto (inherits from container)
justify-content Grid Container Entire grid horizontally within container start
align-content Grid Container Entire grid vertically within container start

Implicit Grid and Auto Placement

The explicit grid is what you define with grid-template-columns and grid-template-rows. The implicit grid is created automatically when items are placed outside the explicit grid.

Controlling the Implicit Grid

You can control how the implicit grid behaves:

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* Explicit columns */
  grid-template-rows: 100px 100px; /* Explicit rows */
  
  /* Define size for implicit rows */
  grid-auto-rows: 100px;
  
  /* Or use minmax for flexible implicit rows */
  grid-auto-rows: minmax(100px, auto);
  
  /* Define size for implicit columns */
  grid-auto-columns: 1fr;
}

Grid Auto Flow

The grid-auto-flow property controls how auto-placed items are inserted into the grid:

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  
  /* Default: items are placed by filling each row */
  grid-auto-flow: row;
  
  /* Items are placed by filling each column */
  grid-auto-flow: column;
  
  /* Items attempt to fill holes in the grid */
  grid-auto-flow: row dense;
  grid-auto-flow: column dense;
}

Dense Packing

The dense value for grid-auto-flow attempts to fill in holes in the grid by taking items out of order. This can make your layout more compact but may change the visual order of elements, which can affect accessibility.

Use dense with caution, especially when the order of items is important for understanding the content.

Common Grid Patterns

Let's explore some common layout patterns that are easy to achieve with CSS Grid.

Holy Grail Layout

The classic "holy grail" layout with header, footer, and three columns:

<div class="holy-grail">
  <header>Header</header>
  <nav>Navigation</nav>
  <main>Main Content</main>
  <aside>Sidebar</aside>
  <footer>Footer</footer>
</div>
.holy-grail {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "nav main aside"
    "footer footer footer";
  min-height: 100vh;
}

header { grid-area: header; }
nav { grid-area: nav; }
main { grid-area: main; }
aside { grid-area: aside; }
footer { grid-area: footer; }

/* Responsive adjustment */
@media (max-width: 768px) {
  .holy-grail {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "nav"
      "main"
      "aside"
      "footer";
  }
}

Card Grid

A responsive grid of cards that adjusts based on available space:

<div class="card-grid">
  <div class="card">Card 1</div>
  <div class="card">Card 2</div>
  <div class="card">Card 3</div>
  <div class="card">Card 4</div>
  <div class="card">Card 5</div>
  <div class="card">Card 6</div>
</div>
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 20px;
}

.card {
  padding: 20px;
  border: 1px solid #ccc;
  border-radius: 4px;
}

Magazine Layout

A complex magazine-style layout with different-sized items:

<div class="magazine-layout">
  <div class="feature">Feature Article</div>
  <div class="standard">Standard Article 1</div>
  <div class="standard">Standard Article 2</div>
  <div class="standard">Standard Article 3</div>
  <div class="wide">Wide Article</div>
  <div class="tall">Tall Article</div>
</div>
.magazine-layout {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-rows: minmax(100px, auto);
  gap: 20px;
}

.feature {
  grid-column: 1 / 3;
  grid-row: 1 / 3;
}

.standard {
  /* Each standard article takes one cell */
}

.wide {
  grid-column: 3 / 5;
}

.tall {
  grid-row: span 2;
}

Masonry-like Layout

A layout that mimics aspects of a masonry grid:

<div class="masonry-grid">
  <div class="item">Item 1</div>
  <div class="item tall">Item 2 (Tall)</div>
  <div class="item">Item 3</div>
  <div class="item">Item 4</div>
  <div class="item tall">Item 5 (Tall)</div>
  <div class="item">Item 6</div>
</div>
.masonry-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  grid-auto-rows: 200px;
  gap: 20px;
}

.item {
  overflow: hidden;
}

.tall {
  grid-row: span 2; /* Spans two rows */
}

True Masonry Layout

CSS Grid doesn't natively support true masonry layouts (where items are placed based on available vertical space). For a true masonry layout, you can:

  • Use a JavaScript library like Masonry.js
  • Use the experimental CSS masonry value for grid-template-rows (limited browser support)
  • Use CSS columns (with some limitations)

Overlap Layout

A layout with overlapping elements:

<div class="overlap-grid">
  <div class="background">Background</div>
  <div class="foreground">Foreground</div>
  <div class="overlay">Overlay</div>
</div>
.overlap-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: repeat(3, 100px);
}

.background {
  grid-column: 1 / 4;
  grid-row: 1 / 4;
  z-index: 1;
}

.foreground {
  grid-column: 2 / 4;
  grid-row: 2 / 4;
  z-index: 2;
}

.overlay {
  grid-column: 1 / 3;
  grid-row: 1 / 3;
  z-index: 3;
}

Advanced Grid Techniques

Let's explore some more advanced CSS Grid techniques and patterns.

Subgrid

Subgrid allows a grid item that is also a grid container to inherit the grid tracks from its parent:

.parent-grid {
  display: grid;
  grid-template-columns: repeat(9, 1fr);
  grid-template-rows: auto auto;
}

.child-grid {
  grid-column: 2 / 8;
  grid-row: 2;
  
  display: grid;
  grid-template-columns: subgrid; /* Inherits columns from parent */
  grid-template-rows: subgrid; /* Inherits rows from parent */
}

Browser Support for Subgrid

Subgrid is part of the CSS Grid Level 2 specification and has limited browser support. As of 2023, it's supported in Firefox and Safari, with Chrome support coming soon. Check current browser support before using in production.

Grid Template Shorthand

The grid-template property is a shorthand for grid-template-rows, grid-template-columns, and grid-template-areas:

.container {
  display: grid;
  grid-template:
    "header header header" 100px
    "nav main aside" 1fr
    "footer footer footer" 100px
    / 200px 1fr 200px;
}

/* Equivalent to: */
.container {
  display: grid;
  grid-template-rows: 100px 1fr 100px;
  grid-template-columns: 200px 1fr 200px;
  grid-template-areas:
    "header header header"
    "nav main aside"
    "footer footer footer";
}

The grid Shorthand

The grid property is a shorthand for all grid properties:

.container {
  display: grid;
  grid: 100px 1fr 100px / 200px 1fr 200px;
}

/* Equivalent to: */
.container {
  display: grid;
  grid-template-rows: 100px 1fr 100px;
  grid-template-columns: 200px 1fr 200px;
}

You can also use the more complex syntax with named areas:

.container {
  display: grid;
  grid:
    "header header header" 100px
    "nav main aside" 1fr
    "footer footer footer" 100px
    / 200px 1fr 200px;
}

Layering Grid Items

You can create layered layouts by placing multiple items in the same grid area and controlling their stacking with z-index:

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: repeat(3, 100px);
}

.base {
  grid-column: 1 / 4;
  grid-row: 1 / 4;
  z-index: 1;
}

.overlay {
  grid-column: 1 / 4;
  grid-row: 1 / 4;
  z-index: 2;
  /* This item will appear on top of .base */
}

Responsive Layouts Without Media Queries

CSS Grid can create responsive layouts with minimal or no media queries:

.responsive-grid {
  display: grid;
  /* Creates as many columns as will fit, with a minimum width of 200px */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 20px;
}

/* For more complex layouts, you might still need media queries */
.complex-layout {
  display: grid;
  grid-template-columns: 1fr 3fr;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}

@media (max-width: 768px) {
  .complex-layout {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "main"
      "sidebar"
      "footer";
  }
}

Grid Browser Support and Fallbacks

CSS Grid is well-supported in modern browsers, but you may need fallbacks for older browsers.

Browser Support

CSS Grid is supported in all modern browsers, including:

  • Chrome 57+
  • Firefox 52+
  • Safari 10.1+
  • Edge 16+
  • Opera 44+
  • iOS Safari 10.3+
  • Android Browser 67+

Feature Detection

You can use @supports to detect Grid support and provide fallbacks:

/* Fallback for browsers without Grid support */
.container {
  display: flex;
  flex-wrap: wrap;
}

.item {
  flex: 1 1 300px;
  margin: 10px;
}

/* Grid layout for supporting browsers */
@supports (display: grid) {
  .container {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
    gap: 20px;
  }
  
  .item {
    flex: none;
    margin: 0;
  }
}

Progressive Enhancement

Another approach is to use a simple layout that works everywhere, then enhance it with Grid:

/* Base layout that works everywhere */
.container > * {
  padding: 20px;
  margin-bottom: 20px;
}

/* Enhanced layout with Grid */
@supports (display: grid) {
  .container {
    display: grid;
    grid-template-columns: 1fr 3fr;
    grid-template-areas:
      "header header"
      "sidebar main"
      "footer footer";
    gap: 20px;
  }
  
  .container > * {
    margin-bottom: 0; /* Remove margin since we're using gap */
  }
  
  .header { grid-area: header; }
  .sidebar { grid-area: sidebar; }
  .main { grid-area: main; }
  .footer { grid-area: footer; }
}

Autoprefixer and Grid

Autoprefixer can add IE-specific prefixes for CSS Grid, but it doesn't support all Grid features. For IE support, you'll need to enable the Grid option in Autoprefixer and be aware of its limitations.

For most projects, it's better to provide a simpler fallback layout for older browsers rather than trying to make Grid work in IE.

Grid Best Practices

Here are some best practices to follow when working with CSS Grid:

Use Grid for Layout, Flexbox for Components

Use Grid for overall page layout and two-dimensional components, and Flexbox for one-dimensional components:

/* Page layout with Grid */
.page {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}

/* Navigation with Flexbox */
.nav ul {
  display: flex;
  justify-content: space-between;
}

Name Your Grid Areas

Using named grid areas makes your code more readable and maintainable:

/* Using grid-template-areas */
.container {
  grid-template-areas:
    "header header header"
    "sidebar content ads"
    "footer footer footer";
}

/* Instead of line numbers */
.header {
  grid-area: header;
}

/* Rather than this */
.header {
  grid-column: 1 / 4;
  grid-row: 1 / 2;
}

Use Semantic HTML with Grid

Grid works best with semantic HTML elements:

<div class="page">
  <header>Header</header>
  <nav>Navigation</nav>
  <main>Main Content</main>
  <aside>Sidebar</aside>
  <footer>Footer</footer>
</div>
.page {
  display: grid;
  grid-template-areas:
    "header header"
    "nav main"
    "nav aside"
    "footer footer";
}

header { grid-area: header; }
nav { grid-area: nav; }
main { grid-area: main; }
aside { grid-area: aside; }
footer { grid-area: footer; }

Be Mindful of the Visual Order

Grid allows you to change the visual order of elements, which can affect accessibility:

/* This can create a disconnect between source order and visual order */
.item-1 {
  grid-row: 2; /* Visually appears second */
}
.item-2 {
  grid-row: 1; /* Visually appears first */
}

Screen readers and keyboard navigation follow the source order, not the visual order. When possible, keep the visual order aligned with the source order, or use tabindex to ensure proper keyboard navigation.

Use Responsive Techniques

Take advantage of Grid's responsive capabilities:

/* Responsive grid without media queries */
.auto-fit-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 20px;
}

/* Responsive grid with media queries for more control */
.layout {
  display: grid;
  grid-template-columns: 1fr 3fr;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}

@media (max-width: 768px) {
  .layout {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "main"
      "sidebar"
      "footer";
  }
}

Use Grid Gap Instead of Margins

Grid gap provides a cleaner way to add space between items:

/* Instead of this */
.grid-item {
  margin: 10px;
}

/* Use this */
.grid {
  display: grid;
  gap: 20px;
}

Avoid Fixed Heights When Possible

Use auto or minmax() for row heights to accommodate variable content:

/* Instead of fixed heights */
.grid {
  grid-template-rows: 100px 100px 100px;
}

/* Use flexible heights */
.grid {
  grid-template-rows: auto auto auto;
}

/* Or with minimum heights */
.grid {
  grid-template-rows: minmax(100px, auto) minmax(100px, auto) minmax(100px, auto);
}
