Typography

The Importance of Typography

Typography is the art and technique of arranging type to make written language legible, readable, and appealing. In web design, typography plays a crucial role in communication, user experience, and brand identity.

Good typography enhances readability, establishes hierarchy, creates visual interest, and conveys the right tone and mood. Poor typography can make content difficult to read, create visual confusion, and undermine your design's effectiveness.

CSS provides a wide range of properties to control typography, allowing you to create beautiful, readable text that enhances your web pages.

Typography Terminology

Understanding typography terminology helps when working with CSS text properties:

  • Typeface: A set of fonts designed with common characteristics (e.g., Helvetica, Times New Roman)
  • Font: A specific weight, width, and style of a typeface (e.g., Helvetica Bold Italic)
  • Serif: Small lines or strokes at the ends of characters (e.g., Times New Roman)
  • Sans-serif: Fonts without serifs (e.g., Arial, Helvetica)
  • Monospace: Fonts where each character occupies the same width (e.g., Courier, Consolas)
  • Weight: The thickness of character strokes (e.g., light, regular, bold)
  • Style: Variations like italic or oblique
  • Leading: The vertical space between lines of text (line-height in CSS)
  • Kerning: The space between specific character pairs
  • Tracking: The overall spacing between characters (letter-spacing in CSS)

Font Properties

CSS provides several properties to control the appearance of fonts.

Font Family

The font-family property specifies the typeface to use for text. You can provide a list of font families as fallbacks in case the preferred font isn't available:

/* Single font family with generic fallback */
p {
  font-family: Arial, sans-serif;
}

/* Multiple fallback fonts */
h1 {
  font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
}

Generic font families can be used as a last resort:

  • serif: Fonts with small lines at the ends of characters
  • sans-serif: Fonts without serifs
  • monospace: Fonts where all characters have the same width
  • cursive: Fonts that mimic handwriting
  • fantasy: Decorative fonts
  • system-ui: The default user interface font

Font Names with Spaces

If a font name contains spaces, it must be enclosed in quotes:

h1 {
  font-family: "Times New Roman", Times, serif;
}

Font Size

The font-size property sets the size of the text. There are several units you can use:

/* Absolute units */
h1 {
  font-size: 32px;
}

/* Relative to parent element */
p {
  font-size: 1.2em; /* 1.2 times the parent's font size */
}

/* Relative to root element */
h2 {
  font-size: 1.5rem; /* 1.5 times the root element's font size */
}

/* Percentage */
small {
  font-size: 80%; /* 80% of the parent's font size */
}

/* Viewport units */
.hero-text {
  font-size: 5vw; /* 5% of the viewport width */
}
Unit Description Use Case
px Pixels (fixed unit) Precise control, but doesn't scale with user preferences
em Relative to parent element's font size Nested elements, component-based design
rem Relative to root element's font size Consistent scaling across the document
% Percentage of parent element's font size Similar to em, good for nested elements
vw Percentage of viewport width Responsive headlines that scale with screen size
vh Percentage of viewport height Text that scales with viewport height

Best Practice: Use Relative Units

For most text, it's best to use relative units like rem or em rather than fixed units like px. This allows text to scale based on user preferences and device characteristics, improving accessibility and responsiveness.

Font Weight

The font-weight property sets the thickness or boldness of the text:

/* Keyword values */
p {
  font-weight: normal;
}

h1 {
  font-weight: bold;
}

/* Numeric values (100-900) */
.light {
  font-weight: 300;
}

.regular {
  font-weight: 400; /* Same as normal */
}

.medium {
  font-weight: 500;
}

.semi-bold {
  font-weight: 600;
}

.bold {
  font-weight: 700; /* Same as bold */
}

.extra-bold {
  font-weight: 800;
}

.black {
  font-weight: 900;
}

Not all fonts support all weight values. If a specific weight isn't available, the browser will use the closest available weight.

Font Style

The font-style property sets whether text is italic or normal:

p {
  font-style: normal;
}

em {
  font-style: italic;
}

.slanted {
  font-style: oblique; /* Similar to italic but mechanically slanted */
}

Font Variant

The font-variant property can be used to display text in small caps:

.small-caps {
  font-variant: small-caps;
}

Font Stretch

The font-stretch property allows you to make text narrower or wider:

.condensed {
  font-stretch: condensed;
}

.normal-width {
  font-stretch: normal;
}

.expanded {
  font-stretch: expanded;
}

/* Percentage values */
.slightly-condensed {
  font-stretch: 90%;
}

.slightly-expanded {
  font-stretch: 110%;
}

Support for this property depends on the font family. Many fonts don't include condensed or expanded variants.

Font Shorthand

The font shorthand property allows you to set multiple font properties at once:

/* font: font-style font-variant font-weight font-size/line-height font-family */
p {
  font: italic small-caps bold 16px/1.5 Arial, sans-serif;
}

/* Minimum required properties are font-size and font-family */
h1 {
  font: 24px Arial, sans-serif;
}

Font Shorthand Order

When using the font shorthand, the order matters:

  1. font-style
  2. font-variant
  3. font-weight
  4. font-size (required)
  5. line-height (optional, follows font-size with a slash)
  6. font-family (required, must be last)

Any omitted values will be set to their initial values.

Web Fonts

Web fonts allow you to use custom fonts that aren't installed on users' devices. This gives you more design flexibility while ensuring consistent typography across different platforms.

@font-face Rule

The @font-face rule allows you to define custom fonts:

@font-face {
  font-family: 'MyCustomFont';
  src: url('fonts/mycustomfont.woff2') format('woff2'),
       url('fonts/mycustomfont.woff') format('woff');
  font-weight: normal;
  font-style: normal;
}

/* Using the custom font */
body {
  font-family: 'MyCustomFont', Arial, sans-serif;
}

Font Formats

Several font formats are supported by modern browsers:

  • .woff2: Web Open Font Format 2.0 (best compression, modern browsers)
  • .woff: Web Open Font Format (good compression, wide support)
  • .ttf: TrueType Font (older format, larger file size)
  • .eot: Embedded OpenType (for older IE browsers)
  • .svg: SVG Font (rarely used now)

For modern websites, .woff2 and .woff formats are sufficient.

Google Fonts

Google Fonts is a popular service that provides free web fonts. You can include Google Fonts in your project by adding a link in your HTML:

<!-- HTML method -->
<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap" rel="stylesheet">

Or by using the @import rule in your CSS:

/* CSS method */
@import url('https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap');

body {
  font-family: 'Roboto', sans-serif;
}

Variable Fonts

Variable fonts are a newer font technology that allows a single font file to behave like multiple fonts. They can dynamically adjust weight, width, slant, and other attributes:

@font-face {
  font-family: 'MyVariableFont';
  src: url('fonts/myvariablefont.woff2') format('woff2-variations');
  font-weight: 100 900; /* Range of weights supported */
}

.light-text {
  font-family: 'MyVariableFont';
  font-weight: 200;
}

.bold-text {
  font-family: 'MyVariableFont';
  font-weight: 800;
}

/* Using font-variation-settings for custom axes */
.custom-variation {
  font-family: 'MyVariableFont';
  font-variation-settings: 'wght' 375, 'wdth' 80;
}

Font Loading Performance

Web fonts can impact performance if not handled properly. Consider these best practices:

  • Use font-display to control how fonts load and render
  • Limit the number of font weights and styles
  • Use modern formats like WOFF2 for better compression
  • Consider preloading critical fonts
@font-face {
  font-family: 'MyCustomFont';
  src: url('fonts/mycustomfont.woff2') format('woff2');
  font-weight: normal;
  font-style: normal;
  font-display: swap; /* Shows fallback font until custom font loads */
}

Text Properties

CSS provides numerous properties to control the appearance and layout of text.

Text Color

The color property sets the color of text:

/* Named color */
p {
  color: blue;
}

/* Hexadecimal */
h1 {
  color: #ff0000;
}

/* RGB */
.subtitle {
  color: rgb(0, 128, 0);
}

/* RGBA (with alpha/transparency) */
.faded-text {
  color: rgba(0, 0, 0, 0.5);
}

/* HSL (hue, saturation, lightness) */
.accent {
  color: hsl(240, 100%, 50%);
}

Text Alignment

The text-align property controls the horizontal alignment of text:

.left-aligned {
  text-align: left; /* Default for languages that read left-to-right */
}

.centered {
  text-align: center;
}

.right-aligned {
  text-align: right;
}

.justified {
  text-align: justify; /* Spreads text to align with both left and right edges */
}

Text Decoration

The text-decoration property adds lines to text:

a {
  text-decoration: none; /* Removes the default underline from links */
}

.underlined {
  text-decoration: underline;
}

.overlined {
  text-decoration: overline;
}

.line-through {
  text-decoration: line-through;
}

/* Combined decorations */
.fancy {
  text-decoration: underline overline;
}

/* With color and style */
.custom-underline {
  text-decoration: underline dotted red;
}

You can also use individual properties for more control:

.detailed {
  text-decoration-line: underline;
  text-decoration-style: wavy;
  text-decoration-color: blue;
  text-decoration-thickness: 2px;
}

Text Transform

The text-transform property changes the capitalization of text:

.uppercase {
  text-transform: uppercase; /* ALL CAPS */
}

.lowercase {
  text-transform: lowercase; /* all lowercase */
}

.capitalize {
  text-transform: capitalize; /* First Letter Of Each Word */
}

.normal-case {
  text-transform: none; /* No transformation */
}

Text Indent

The text-indent property indents the first line of a block of text:

p {
  text-indent: 2em; /* Indents the first line by 2em */
}

.negative-indent {
  text-indent: -20px; /* Creates a hanging indent */
  padding-left: 20px; /* Ensures text doesn't overflow */
}

Letter Spacing

The letter-spacing property adjusts the space between characters:

.tight {
  letter-spacing: -0.5px;
}

.normal {
  letter-spacing: normal;
}

.spaced {
  letter-spacing: 2px;
}

.very-spaced {
  letter-spacing: 0.5em;
}

Word Spacing

The word-spacing property adjusts the space between words:

.tight-words {
  word-spacing: -2px;
}

.normal-words {
  word-spacing: normal;
}

.spaced-words {
  word-spacing: 5px;
}

White Space

The white-space property controls how white space inside an element is handled:

.normal-wrap {
  white-space: normal; /* Default: collapses white space and wraps text */
}

.no-wrap {
  white-space: nowrap; /* Prevents text from wrapping to a new line */
}

.preserve-space {
  white-space: pre; /* Preserves white space like the HTML <pre> element */
}

.preserve-wrap {
  white-space: pre-wrap; /* Preserves white space but allows wrapping */
}

.collapse-wrap {
  white-space: pre-line; /* Collapses white space but preserves line breaks */
}

Text Overflow

The text-overflow property specifies how overflowed content is signaled to users:

.ellipsis {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis; /* Shows "..." for overflowed text */
}

.clip {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: clip; /* Simply clips the text */
}

Multi-line Ellipsis

The standard text-overflow: ellipsis only works for single lines. For multi-line ellipsis, you can use the following technique:

.multiline-ellipsis {
  display: -webkit-box;
  -webkit-line-clamp: 3; /* Number of lines to show */
  -webkit-box-orient: vertical;
  overflow: hidden;
}

Note that this is not a standard CSS property and may not work in all browsers.

Word Break

The word-break property specifies how words should break when reaching the end of a line:

.normal-break {
  word-break: normal; /* Default behavior */
}

.break-all {
  word-break: break-all; /* May break words at any character */
}

.keep-all {
  word-break: keep-all; /* Prevents breaking words */
}

.break-word {
  overflow-wrap: break-word; /* Breaks words only when necessary */
}

Writing Mode

The writing-mode property sets whether lines of text are laid out horizontally or vertically:

.horizontal {
  writing-mode: horizontal-tb; /* Default: top-to-bottom, left-to-right */
}

.vertical-right-to-left {
  writing-mode: vertical-rl; /* Vertical, right-to-left */
}

.vertical-left-to-right {
  writing-mode: vertical-lr; /* Vertical, left-to-right */
}

This property is particularly useful for languages that can be written vertically, like Japanese or Chinese.

Line and Paragraph Properties

These properties control the spacing and layout of lines and paragraphs.

Line Height

The line-height property sets the height of a line of text, also known as leading:

/* Unitless value (recommended) */
p {
  line-height: 1.5; /* 1.5 times the font size */
}

/* With units */
.specific {
  line-height: 24px;
}

.percentage {
  line-height: 150%;
}

/* Normal is typically around 1.2 */
.normal {
  line-height: normal;
}

Unitless Line Height

It's generally best to use unitless values for line-height. This way, the line height scales proportionally when the font size changes, especially in nested elements.

A line height of 1.5-1.6 is often recommended for body text to improve readability.

Vertical Alignment

The vertical-align property aligns inline or table-cell elements vertically:

/* For inline elements */
img {
  vertical-align: middle;
}

.superscript {
  vertical-align: super;
}

.subscript {
  vertical-align: sub;
}

/* Other values */
.top {
  vertical-align: top;
}

.bottom {
  vertical-align: bottom;
}

.text-top {
  vertical-align: text-top;
}

.text-bottom {
  vertical-align: text-bottom;
}

/* With length */
.custom {
  vertical-align: 5px; /* Shifts the element 5px upward */
}

Note that vertical-align only works on inline or table-cell elements, not on block-level elements.

Text Direction

The direction property sets the text direction:

.left-to-right {
  direction: ltr; /* Left to right (default) */
}

.right-to-left {
  direction: rtl; /* Right to left (for languages like Arabic or Hebrew) */
}

This is often used with the unicode-bidi property for bidirectional text:

.bidi-override {
  direction: rtl;
  unicode-bidi: bidi-override;
}

Columns

CSS provides properties to create multi-column text layouts:

.multi-column {
  column-count: 3; /* Creates 3 columns */
  column-gap: 2em; /* Space between columns */
  column-rule: 1px solid #ccc; /* Line between columns */
}

/* Alternatively, specify column width and let the browser determine the count */
.flexible-columns {
  column-width: 200px; /* Preferred column width */
  column-gap: 2em;
  column-rule: 1px solid #ccc;
}

/* Prevent an element from breaking across columns */
.no-break {
  break-inside: avoid-column;
}

/* Force a column break before an element */
.new-column {
  break-before: column;
}

Hyphens

The hyphens property controls hyphenation of text:

.no-hyphens {
  hyphens: none;
}

.auto-hyphens {
  hyphens: auto; /* Browser determines hyphenation */
}

.manual-hyphens {
  hyphens: manual; /* Only hyphenates at ­ or - characters */
}

For hyphens: auto to work properly, the lang attribute should be set on the HTML element:

<html lang="en">
  <!-- Content -->
</html>

Text Effects

CSS provides several properties for creating special text effects.

Text Shadow

The text-shadow property adds shadow effects to text:

/* Basic shadow */
h1 {
  text-shadow: 2px 2px 4px rgba(0, 0, 0, 0.5);
  /* horizontal offset, vertical offset, blur radius, color */
}

/* Multiple shadows (separated by commas) */
.glow {
  text-shadow: 0 0 5px #fff, 0 0 10px #fff, 0 0 15px #0073e6, 0 0 20px #0073e6;
}

/* Outline effect using multiple shadows */
.outline {
  color: white;
  text-shadow: -1px -1px 0 #000, 1px -1px 0 #000, -1px 1px 0 #000, 1px 1px 0 #000;
}

Text Stroke

The -webkit-text-stroke property (non-standard but widely supported) creates an outline around text:

.outlined-text {
  -webkit-text-stroke: 1px black;
  color: white;
}

/* Can also be specified separately */
.detailed-outline {
  -webkit-text-stroke-width: 2px;
  -webkit-text-stroke-color: red;
}

Background Clip

The background-clip property with text value allows you to fill text with a background image or gradient:

.gradient-text {
  background-image: linear-gradient(45deg, #f3ec78, #af4261);
  background-clip: text;
  -webkit-background-clip: text;
  color: transparent;
}

Mask Image

CSS masks can create interesting text effects by masking an element with text:

.masked-image {
  mask-image: url('text-mask.svg');
  mask-size: cover;
  -webkit-mask-image: url('text-mask.svg');
  -webkit-mask-size: cover;
}

Typography Best Practices

Here are some guidelines for creating effective typography on the web:

Readability

  • Font size: Use at least 16px for body text
  • Line height: Set line-height to 1.5-1.6 for body text
  • Line length: Aim for 45-75 characters per line (about 2-3 alphabets)
  • Contrast: Ensure sufficient contrast between text and background
  • Font choice: Use readable fonts, especially for body text

Hierarchy

  • Create clear visual hierarchy using size, weight, and spacing
  • Use consistent heading styles throughout your site
  • Don't rely solely on color to convey information
  • Use whitespace to group related content and separate sections

Responsive Typography

  • Use relative units like rem or em for font sizes
  • Consider using viewport units for headings
  • Test typography at different screen sizes
  • Adjust line length for different devices

Performance

  • Limit the number of font weights and styles
  • Use modern font formats like WOFF2
  • Consider using system fonts for better performance
  • Use font-display to control font loading behavior

Accessibility

  • Ensure text can be resized without breaking the layout
  • Maintain sufficient color contrast (WCAG recommends at least 4.5:1 for normal text)
  • Don't use justified text, as it can create uneven spacing and make text harder to read
  • Avoid using too many different fonts and styles

System Font Stack

For better performance, consider using a system font stack that uses the default fonts on each operating system:

body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen-Sans, Ubuntu, Cantarell, "Helvetica Neue", sans-serif;
}

This approach provides a native look and feel while avoiding the performance cost of loading web fonts.
