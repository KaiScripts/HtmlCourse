CSS for Print

Introduction to Print Stylesheets

While web design often focuses on screen display, creating print-friendly versions of web content remains an important consideration. Users still print web pages for various reasons: reading offline, archiving information, sharing physical copies, or completing paper forms. A well-designed print stylesheet ensures that your content remains usable and attractive when printed.

In this lesson, we'll explore techniques for creating effective print stylesheets, optimizing web content for printing, and addressing common print-specific challenges.

Why Print Stylesheets Matter

Print stylesheets offer several important benefits:

  • Improved User Experience: Users who print your content will appreciate a version optimized for paper
  • Resource Conservation: Properly formatted print layouts use less ink and paper
  • Accessibility: Some users prefer or need physical copies for various reasons
  • Professional Appearance: Well-designed print versions reflect positively on your brand
  • Content Preservation: Print versions can serve as backups or archives

Print vs. Screen: Key Differences

Print and screen media have fundamental differences that affect design decisions:

Characteristic Screen Print
Dimensions Variable, fluid Fixed, typically standard paper sizes (Letter, A4, etc.)
Color RGB, wide gamut CMYK, limited gamut, potential for grayscale
Resolution Typically 72-300 PPI Typically 300-600 DPI
Interactivity Interactive elements, hover states, animations Static, no interactivity
Navigation Links, menus, infinite scrolling Page numbers, table of contents, physical page turns
Content Flow Single continuous document Discrete pages with breaks
Background Often colored or textured Usually white to save ink
Typography Sans-serif often preferred for readability Serif often preferred for readability

Basic Print Stylesheet Setup

There are several ways to include print-specific CSS:

1. Using Media Queries

The most common approach is to use media queries within your main CSS file:

/* Regular styles for screen */
body {
  background-color: #f5f5f5;
  color: #333;
}

/* Print-specific styles */
@media print {
  body {
    background-color: white;
    color: black;
  }
}

2. Using a Separate Print Stylesheet

You can also create a separate CSS file specifically for print:

<!-- Main stylesheet for screens -->
<link rel="stylesheet" href="styles.css" media="screen">

<!-- Print-specific stylesheet -->
<link rel="stylesheet" href="print.css" media="print">

3. Using the CSS @import Rule

You can import a print stylesheet within your main CSS file:

/* In your main CSS file */
@import url('print.css') print;

Best Practice

Using media queries within your main CSS file is generally preferred for several reasons:

  • Reduces HTTP requests
  • Keeps related styles together
  • Makes maintenance easier
  • Allows for sharing variables and utility classes

However, for very complex print styles or large projects, a separate print stylesheet might be more manageable.

Testing Print Stylesheets

You don't need to waste paper to test print stylesheets. Modern browsers provide tools for previewing print layouts:

Chrome

  • Press Ctrl+P (Windows/Linux) or Cmd+P (Mac)
  • Or go to Menu > Print
  • Use the preview pane to see how your page will print
  • In DevTools, you can also use the Rendering tab to emulate the print media type

Firefox

  • Press Ctrl+P (Windows/Linux) or Cmd+P (Mac)
  • Or go to Menu > Print
  • Click "Preview" to see the print layout
  • In DevTools, you can use the "print" media simulation

Safari

  • Press Cmd+P
  • Or go to File > Print
  • The print dialog shows a preview

Using DevTools

For more detailed testing and debugging:

  1. Open DevTools (F12 or Ctrl+Shift+I)
  2. In Chrome, open the Command Menu (Ctrl+Shift+P) and type "Rendering"
  3. In the Rendering tab, find "Emulate CSS media type" and select "print"
  4. Your page will now render using print styles while allowing you to use DevTools

Essential Print Stylesheet Techniques

Let's explore the fundamental techniques for creating effective print stylesheets.

Hiding Unnecessary Elements

Many elements that are useful on screen become unnecessary or distracting in print:

@media print {
  /* Hide navigation, sidebars, ads, and other non-essential elements */
  nav,
  .sidebar,
  .ads,
  .social-buttons,
  .comments,
  .footer-links,
  .video-player,
  button,
  .search-box,
  #cookie-notice {
    display: none !important;
  }
}

Consider hiding these common elements in print stylesheets:

  • Navigation menus
  • Sidebars
  • Advertisements
  • Social media widgets
  • Comment sections
  • Video players
  • Interactive elements (buttons, forms, etc.)
  • Search boxes
  • Cookie notices and popups

Optimizing Layout for Paper

Adjust your layout to work well on paper:

@media print {
  /* Reset page margins */
  @page {
    margin: 2cm;
  }
  
  /* Make content full-width */
  .container,
  main,
  article {
    width: 100% !important;
    margin: 0 !important;
    padding: 0 !important;
  }
  
  /* Adjust typography for print */
  body {
    font-size: 12pt;
    line-height: 1.5;
    font-family: "Times New Roman", Times, serif;
  }
  
  h1 {
    font-size: 24pt;
    margin-bottom: 0.5cm;
  }
  
  h2 {
    font-size: 18pt;
    margin-bottom: 0.3cm;
  }
  
  /* Ensure dark text on white background */
  body {
    background: white !important;
    color: black !important;
  }
}

Managing Page Breaks

Control where page breaks occur to improve readability:

@media print {
  /* Avoid breaking inside these elements */
  h1, h2, h3, h4, h5, h6,
  img, figure, table {
    break-inside: avoid;
  }
  
  /* Avoid breaking after headings */
  h1, h2, h3, h4, h5, h6 {
    break-after: avoid;
  }
  
  /* Force page breaks before new sections */
  section, .chapter {
    break-before: page;
  }
  
  /* Allow breaking inside paragraphs, but not after the first or before the last line */
  p {
    orphans: 3; /* Minimum lines at bottom of page */
    widows: 3;  /* Minimum lines at top of page */
  }
}

The CSS properties for controlling page breaks include:

  • break-inside: Controls whether a break should occur inside an element
  • break-before: Controls whether a break should occur before an element
  • break-after: Controls whether a break should occur after an element
  • orphans: Minimum number of lines that must be left at the bottom of a page
  • widows: Minimum number of lines that must appear at the top of a page

Legacy Properties

For better browser compatibility, you may want to include the older page break properties alongside the newer ones:

h2 {
  /* Modern syntax */
  break-inside: avoid;
  break-after: avoid;
  
  /* Legacy syntax for older browsers */
  page-break-inside: avoid;
  page-break-after: avoid;
}

Handling Links and URLs

Make links useful in printed format:

@media print {
  /* Make link text black but keep underline for visibility */
  a {
    color: black !important;
    text-decoration: underline;
  }
  
  /* Display the URL after link text */
  a[href^="http"]:after {
    content: " (" attr(href) ")";
    font-size: 90%;
    font-style: italic;
  }
  
  /* Don't display URL for internal page links */
  a[href^="#"]:after {
    content: "";
  }
  
  /* Don't display URL for images that are links */
  a[href^="http"] img:after {
    content: "";
  }
}

Optimizing Images and Media

Ensure images print well and don't waste ink:

@media print {
  /* Ensure images don't get cut off */
  img {
    max-width: 100% !important;
    page-break-inside: avoid;
    break-inside: avoid;
  }
  
  /* Make dark images print better */
  img {
    -webkit-filter: brightness(1.1) contrast(1.3);
    filter: brightness(1.1) contrast(1.3);
  }
  
  /* Hide background images to save ink */
  body, div, section, header, footer, nav, article {
    background-image: none !important;
  }
  
  /* Replace video players with a message */
  .video-container:before {
    content: "Video available at: " attr(data-video-url);
    display: block;
    padding: 1em;
    border: 1px solid #ccc;
    font-style: italic;
  }
}

Adding Print-Only Content

Some content should appear only in the printed version:

/* Hide in normal view */
.print-only {
  display: none;
}

@media print {
  /* Show in print view */
  .print-only {
    display: block;
  }
  
  /* Add URL to the header */
  @top-center {
    content: "Printed from: " attr(data-url);
    font-size: 10pt;
    color: #666;
  }
  
  /* Add page numbers to the footer */
  @bottom-right {
    content: "Page " counter(page) " of " counter(pages);
    font-size: 10pt;
  }
  
  /* Add a print timestamp */
  .article-footer:after {
    content: "Printed on " attr(data-print-date);
    display: block;
    text-align: center;
    font-size: 10pt;
    margin-top: 2em;
  }
}

HTML for print-only elements:

<!-- Print-only header with logo -->
<div class="print-only print-header">
  <img src="logo-print.png" alt="Company Logo">
  <h1>Company Name</h1>
</div>

<!-- Print-only footer with contact info -->
<div class="print-only print-footer">
  <p>For more information, visit www.example.com or call 123-456-7890</p>
</div>

<!-- Print-only QR code -->
<div class="print-only qr-code">
  <img src="qr-code.png" alt="Scan to visit our website">
  <p>Scan to visit our website</p>
</div>

<!-- JavaScript to add current date for printing -->
<script>
  document.querySelector('.article-footer').setAttribute(
    'data-print-date', 
    new Date().toLocaleDateString()
  );
  document.querySelector('html').setAttribute(
    'data-url', 
    window.location.href
  );
</script>

Handling Tables

Ensure tables print clearly and completely:

@media print {
  /* Ensure tables print completely */
  table {
    break-inside: avoid;
    width: 100%;
    border-collapse: collapse;
  }
  
  /* Add borders for clarity */
  th, td {
    border: 1px solid #ddd;
    padding: 8px;
  }
  
  /* Repeat table headers on multi-page tables */
  thead {
    display: table-header-group;
  }
  
  /* Prevent orphaned table footers */
  tfoot {
    display: table-footer-group;
  }
  
  /* For very wide tables, scale them down */
  .wide-table {
    font-size: 9pt;
    transform: scale(0.9);
    transform-origin: top left;
  }
}

Advanced Print Stylesheet Techniques

Beyond the basics, there are several advanced techniques to enhance your print stylesheets.

Using the @page Rule

The @page rule allows you to control page-specific aspects of printing:

@media print {
  /* Basic page setup */
  @page {
    size: letter; /* Can be letter, legal, A4, A5, etc. */
    margin: 2cm; /* Uniform margin */
  }
  
  /* Different margins for different sides */
  @page {
    margin-top: 3cm;
    margin-bottom: 2.5cm;
    margin-left: 2cm;
    margin-right: 2cm;
  }
  
  /* First page special formatting */
  @page :first {
    margin-top: 4cm; /* Extra space at top of first page */
  }
  
  /* Left pages (even-numbered) */
  @page :left {
    margin-left: 3cm;
    margin-right: 2cm;
  }
  
  /* Right pages (odd-numbered) */
  @page :right {
    margin-left: 2cm;
    margin-right: 3cm;
  }
  
  /* Named pages for different sections */
  .chapter {
    page: chapter;
  }
  
  @page chapter {
    margin-bottom: 3cm;
    @bottom-center {
      content: "Chapter " counter(chapter);
    }
  }
}

Page Margin Boxes

The @page rule allows you to add content to the margins of printed pages:

@media print {
  /* Add content to page margins */
  @page {
    @top-left {
      content: "Company Name";
    }
    
    @top-center {
      content: "Confidential Document";
    }
    
    @top-right {
      content: "Page " counter(page);
    }
    
    @bottom-left {
      content: "Printed by: " attr(data-user);
    }
    
    @bottom-center {
      content: "Copyright Â© 2025";
    }
    
    @bottom-right {
      content: "Printed on: " attr(data-print-date);
    }
  }
}

The 16 page margin boxes are:

  • @top-left-corner, @top-left, @top-center, @top-right, @top-right-corner
  • @left-top, @left-middle, @left-bottom
  • @right-top, @right-middle, @right-bottom
  • @bottom-left-corner, @bottom-left, @bottom-center, @bottom-right, @bottom-right-corner

Browser Support Note

Page margin boxes have limited browser support. As of 2025, they work in Firefox but not in Chrome or Safari. For better compatibility, consider using print-only header and footer elements instead.

Generated Content for Print

Use CSS generated content to add useful information to printed pages:

@media print {
  /* Add chapter numbers before headings */
  h1 {
    counter-increment: chapter;
  }
  
  h1::before {
    content: "Chapter " counter(chapter) ": ";
  }
  
  /* Add section numbers before subheadings */
  h2 {
    counter-increment: section;
  }
  
  h2::before {
    content: counter(chapter) "." counter(section) " ";
  }
  
  /* Add "continued" notice for tables that break across pages */
  table::after {
    content: "(Continued on next page)";
    display: block;
    text-align: right;
    font-style: italic;
    font-size: 90%;
    padding-top: 0.5em;
  }
  
  /* Only show "continued" notice if the table actually breaks */
  table:last-child::after {
    content: none;
  }
  
  /* Add footnote references */
  .footnote-ref {
    counter-increment: footnote;
  }
  
  .footnote-ref::after {
    content: "[" counter(footnote) "]";
    vertical-align: super;
    font-size: 75%;
  }
  
  /* Create a footnotes section at the end */
  .footnotes::before {
    content: "Footnotes:";
    font-weight: bold;
    display: block;
    margin-bottom: 1em;
    border-top: 1px solid #ccc;
    padding-top: 1em;
  }
}

Print-Specific Color Adjustments

Optimize colors for printing:

@media print {
  /* Ensure good contrast for printing */
  body {
    color: black !important;
    background: white !important;
  }
  
  /* Darken light text */
  .light-text, .subtle-text {
    color: #333 !important;
  }
  
  /* Convert colored backgrounds to grayscale patterns */
  .primary-bg {
    background-color: white !important;
    background-image: repeating-linear-gradient(
      45deg,
      #000,
      #000 1px,
      #fff 1px,
      #fff 10px
    ) !important;
  }
  
  .secondary-bg {
    background-color: white !important;
    background-image: repeating-linear-gradient(
      -45deg,
      #000,
      #000 1px,
      #fff 1px,
      #fff 10px
    ) !important;
  }
  
  /* Add borders to replace background colors */
  .card, .panel, .box {
    border: 1px solid #999 !important;
    box-shadow: none !important;
  }
  
  /* Improve contrast for links */
  a {
    color: #000 !important;
    font-weight: bold;
    text-decoration: underline;
  }
}

Print-Specific Layout Adjustments

Modify layouts specifically for print:

@media print {
  /* Convert multi-column layout to single column */
  .grid, .columns, .flex-container {
    display: block !important;
  }
  
  /* Expand collapsed sections */
  .accordion-content, .collapsible, .tab-content {
    display: block !important;
    opacity: 1 !important;
    visibility: visible !important;
    height: auto !important;
  }
  
  /* Show all tabs */
  .tab-panel {
    display: block !important;
    margin-bottom: 2em;
  }
  
  .tab-panel::before {
    content: attr(data-tab-title);
    font-weight: bold;
    display: block;
    margin-bottom: 0.5em;
    border-bottom: 1px solid #ccc;
  }
  
  /* Adjust card layouts */
  .card {
    break-inside: avoid;
    page-break-inside: avoid;
    margin-bottom: 1cm;
  }
  
  /* Ensure proper spacing between elements */
  p, h1, h2, h3, h4, h5, h6, ul, ol, dl, table {
    margin-bottom: 0.5cm;
  }
}

Print-Specific Typography

Adjust typography for optimal print readability:

@media print {
  /* Use serif fonts for body text */
  body {
    font-family: "Times New Roman", Times, Georgia, serif;
    font-size: 12pt;
    line-height: 1.5;
  }
  
  /* Keep sans-serif for headings */
  h1, h2, h3, h4, h5, h6 {
    font-family: Arial, Helvetica, sans-serif;
  }
  
  /* Adjust font sizes for print */
  h1 { font-size: 24pt; }
  h2 { font-size: 18pt; }
  h3 { font-size: 14pt; }
  h4 { font-size: 12pt; }
  
  /* Ensure proper line length for readability */
  p, li {
    max-width: 65ch;
  }
  
  /* Adjust letter and word spacing */
  p {
    text-align: justify;
    word-spacing: 0.1em;
    letter-spacing: 0.01em;
  }
  
  /* Use proper quotes */
  blockquote {
    quotes: """ """ "'" "'";
  }
  
  blockquote::before {
    content: open-quote;
  }
  
  blockquote::after {
    content: close-quote;
  }
}

Handling SVG for Print

Ensure SVG graphics print correctly:

@media print {
  /* Ensure SVGs print with correct dimensions */
  svg {
    width: 100% !important;
    height: auto !important;
    max-width: 100% !important;
  }
  
  /* Ensure thin SVG lines are visible when printed */
  svg path, svg line, svg rect, svg circle, svg ellipse {
    stroke-width: 0.5pt !important;
  }
  
  /* Ensure SVG text is readable */
  svg text {
    font-size: 10pt !important;
    fill: black !important;
  }
  
  /* Ensure SVG backgrounds are visible */
  svg[fill="none"] {
    fill: white !important;
  }
}

Printable Certificate Example

Here's an example of CSS for a printable certificate:

/* certificate-print.css */
@media print {
  /* Page setup for landscape certificate */
  @page {
    size: letter landscape;
    margin: 1cm;
  }
  
  body {
    font-family: "Times New Roman", Times, serif;
    font-size: 12pt;
    line-height: 1.4;
    color: #000;
    background: #fff;
    margin: 0;
    padding: 0;
  }
  
  /* Certificate container with border */
  .certificate {
    position: relative;
    width: 25.4cm; /* Letter width in landscape */
    height: 19.05cm; /* Letter height in landscape */
    border: 0.5cm double #000;
    padding: 1cm;
    box-sizing: border-box;
    text-align: center;
    background: #fff;

Print-Friendly Responsive Design

Combining print stylesheets with responsive design requires special considerations.

Responsive Images for Print

Ensure images work well both on screen and in print:

/* Screen styles */
.responsive-image {
  max-width: 100%;
  height: auto;
}

/* Print styles */
@media print {
  .responsive-image {
    max-width: 100% !important;
    height: auto !important;
  }
  
  /* For high-resolution printing */
  .responsive-image[data-print-src] {
    content: attr(data-print-src);
  }
}

HTML for responsive images with print optimization:

<!-- Responsive image with print-specific version -->

Summary

In this lesson, we've explored CSS for print, covering techniques for creating effective print stylesheets and optimizing web content for printing. Here's a summary of what we've learned:

  • Print Stylesheet Basics: Using media queries, separate stylesheets, or @import to define print-specific styles
  • Essential Print Techniques: Hiding unnecessary elements, optimizing layout for paper, managing page breaks, handling links and URLs, optimizing images, and adding print-only content
  • Advanced Print Techniques: Using the @page rule, page margin boxes, generated content, color adjustments, and layout modifications
  • Creating Printable Documents: Designing specialized print layouts for invoices, certificates, and other documents
  • Print-Friendly Responsive Design: Ensuring responsive designs work well when printed

Remember these key principles when creating print stylesheets:

  1. Simplify: Remove unnecessary elements and simplify the layout for print
  2. Optimize: Adjust typography, colors, and images for better print quality
  3. Control: Manage page breaks to ensure content flows logically across pages
  4. Enhance: Add print-specific information like URLs, page numbers, and timestamps
  5. Test: Always test your print stylesheets across different browsers and devices

By implementing these techniques, you can ensure that your web content remains usable and attractive when printed, providing a better experience for users who need physical copies of your content.
