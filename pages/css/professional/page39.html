CSS Preprocessors

Introduction to CSS Preprocessors

CSS preprocessors are scripting languages that extend CSS with programming features, making stylesheets more powerful, maintainable, and efficient. They compile into standard CSS that browsers can understand, while providing developers with advanced capabilities that aren't available in plain CSS.

In this lesson, we'll explore the most popular CSS preprocessors—Sass, Less, and Stylus—and learn how they can enhance your CSS workflow with features like variables, nesting, mixins, functions, and more.

Why Use CSS Preprocessors?

CSS preprocessors offer several advantages over writing plain CSS:

  • Improved Organization: Features like nesting and partials help organize code more logically
  • Enhanced Maintainability: Variables and mixins reduce repetition and make updates easier
  • Increased Productivity: Built-in functions and operations save time and reduce errors
  • Advanced Capabilities: Conditionals, loops, and mathematical operations add programming power
  • Better Collaboration: Modular structure makes it easier for teams to work together

Popular CSS Preprocessors

While there are several CSS preprocessors available, three have emerged as the most widely used:

Preprocessor Syntax Learning Curve Popularity Key Features
Sass SCSS (.scss) or Sass (.sass) Moderate Very High Most mature, two syntax options, extensive libraries
Less Less (.less) Low High JavaScript-based, CSS-like syntax, easy to learn
Stylus Stylus (.styl) High Moderate Flexible syntax, powerful features, highly expressive

In this lesson, we'll focus primarily on Sass (specifically the SCSS syntax) as it's the most widely used preprocessor, but we'll also cover key differences in Less and Stylus.

Getting Started with Sass

Sass (Syntactically Awesome Style Sheets) is the most mature and widely used CSS preprocessor. It offers two syntax options: SCSS (Sassy CSS), which is a superset of CSS, and the original Sass syntax, which uses indentation instead of braces and semicolons.

Installation and Setup

There are several ways to use Sass in your projects:

1. Command Line

# Install Sass globally using npm
npm install -g sass

# Compile a single file
sass input.scss output.css

# Watch for changes
sass --watch input.scss:output.css

# Watch a directory
sass --watch scss:css

2. Build Tools

Sass can be integrated with build tools like Webpack, Gulp, or Grunt:

// Example with Webpack and sass-loader
module.exports = {
  // ...
  module: {
    rules: [
      {
        test: /\.scss$/,
        use: [
          'style-loader',
          'css-loader',
          'sass-loader'
        ]
      }
    ]
  }
};

3. GUI Applications

Several applications provide a graphical interface for working with Sass:

  • Prepros
  • Koala
  • Scout-App

SCSS vs Sass Syntax

Sass offers two syntax options:

SCSS Syntax (.scss) Sass Syntax (.sass) 
.button {
  background-color: #3498db;
  color: white;
  padding: 10px 15px;
  
  &:hover {
    background-color: darken(#3498db, 10%);
  }
}
 
.button
  background-color: #3498db
  color: white
  padding: 10px 15px
  
  &:hover
    background-color: darken(#3498db, 10%)

We'll focus on the SCSS syntax in this lesson as it's more widely used and easier for CSS developers to adopt since it's a superset of CSS (valid CSS is also valid SCSS).

Basic Project Structure

A typical Sass project structure might look like this:

project/
├── scss/
│   ├── main.scss             # Main file that imports all others
│   ├── _variables.scss       # Variables
│   ├── _mixins.scss          # Mixins and functions
│   ├── _base.scss            # Base styles
│   ├── _typography.scss      # Typography styles
│   ├── _buttons.scss         # Button styles
│   ├── _forms.scss           # Form styles
│   └── _layout.scss          # Layout styles
└── css/
    └── main.css              # Compiled CSS output

Note the underscore prefix (_) for partial files. These are files meant to be imported into other Sass files and not compiled directly to CSS.

Sass Features: Variables and Calculations

One of the most powerful features of Sass is the ability to define and use variables, making your stylesheets more maintainable and consistent.

Variables

Variables in Sass are defined with the $ symbol:

// Defining variables
$primary-color: #3498db;
$secondary-color: #2ecc71;
$font-stack: 'Poppins', sans-serif;
$base-spacing: 1rem;
$border-radius: 4px;

// Using variables
body {
  font-family: $font-stack;
  line-height: 1.6;
}

.button {
  background-color: $primary-color;
  border-radius: $border-radius;
  padding: $base-spacing;
}

.alert {
  border: 1px solid $secondary-color;
  margin-bottom: $base-spacing * 2;
}

Variable Scope

Variables in Sass have scope, meaning they're only available within the block they're defined in, unless they're defined at the root level:

$global-color: blue;  // Global variable

.container {
  $local-padding: 20px;  // Local variable
  padding: $local-padding;
  color: $global-color;
}

.other-element {
  // This works
  color: $global-color;
  
  // This would cause an error
  // padding: $local-padding;
}

Default Values

You can set default values for variables using the !default flag, which only assigns the value if the variable isn't already defined:

// _variables.scss
$primary-color: blue !default;
$secondary-color: green !default;

// main.scss
$primary-color: purple;  // Override the default
@import 'variables';

.element {
  // Will be purple, not blue
  color: $primary-color;
  
  // Will be green (the default)
  background-color: $secondary-color;
}

This is particularly useful for creating configurable libraries or frameworks.

Calculations and Operations

Sass allows you to perform mathematical operations directly in your stylesheets:

$base-spacing: 1rem;

.container {
  // Basic arithmetic
  padding: $base-spacing * 2;
  margin-bottom: $base-spacing + 0.5rem;
  width: 100% - 20px;
  
  // More complex calculations
  line-height: (18px / 16px);
  font-size: calc(1rem + 0.5vw);
}

Color Operations

Sass provides powerful functions for manipulating colors:

$primary-color: #3498db;

.button {
  background-color: $primary-color;
  
  &:hover {
    // Darken by 10%
    background-color: darken($primary-color, 10%);
  }
  
  &:active {
    // Darken by 15%
    background-color: darken($primary-color, 15%);
  }
  
  &.light {
    // Lighten by 20%
    background-color: lighten($primary-color, 20%);
    
    // Adjust saturation
    border-color: saturate($primary-color, 20%);
  }
  
  &.disabled {
    // Reduce opacity
    background-color: rgba($primary-color, 0.5);
    
    // Desaturate (make more gray)
    border-color: desaturate($primary-color, 30%);
  }
}

Other Color Functions

$base-color: #3498db;

.color-variations {
  // Adjust hue (shift color while maintaining lightness/saturation)
  --color-1: adjust-hue($base-color, 30deg);
  
  // Mix colors
  --color-2: mix($base-color, #ff0000, 70%);
  
  // Complement (opposite on color wheel)
  --color-3: complement($base-color);
  
  // Invert
  --color-4: invert($base-color);
  
  // Grayscale
  --color-5: grayscale($base-color);
}

CSS Custom Properties vs. Sass Variables

While both serve similar purposes, they have important differences:

  • Sass variables are processed at compile time and can't be changed in the browser
  • CSS Custom Properties (variables) are processed at runtime and can be changed with JavaScript
  • Sass variables support more operations and can be used in more contexts
  • CSS Custom Properties cascade and inherit like other CSS properties

A common approach is to use Sass variables for values that won't change (like a color palette) and CSS Custom Properties for values that might change dynamically.

Sass Features: Nesting and Parent Selectors

Nesting is one of the most popular features of Sass, allowing you to write more concise and readable CSS by nesting selectors within one another.

Basic Nesting

Nesting allows you to structure your CSS to match the HTML hierarchy:

// Without nesting (CSS)
.card {
  border: 1px solid #ddd;
  border-radius: 4px;
}

.card-header {
  background-color: #f5f5f5;
  padding: 10px;
}

.card-body {
  padding: 15px;
}

.card-footer {
  border-top: 1px solid #ddd;
  padding: 10px;
}

// With nesting (SCSS)
.card {
  border: 1px solid #ddd;
  border-radius: 4px;
  
  .card-header {
    background-color: #f5f5f5;
    padding: 10px;
  }
  
  .card-body {
    padding: 15px;
  }
  
  .card-footer {
    border-top: 1px solid #ddd;
    padding: 10px;
  }
}

The nested SCSS compiles to the same CSS as the non-nested version, but it's more organized and easier to read.

Parent Selector (&)

The ampersand (&) symbol represents the parent selector, allowing you to reference the parent within a nested rule:

.button {
  background-color: #3498db;
  color: white;
  padding: 10px 15px;
  
  // Hover state
  &:hover {
    background-color: darken(#3498db, 10%);
  }
  
  // Active state
  &:active {
    transform: translateY(1px);
  }
  
  // Disabled state
  &.disabled {
    opacity: 0.5;
    pointer-events: none;
  }
  
  // Modifiers
  &--large {
    padding: 15px 20px;
    font-size: 1.2em;
  }
  
  &--small {
    padding: 5px 10px;
    font-size: 0.8em;
  }
  
  // Contextual styling
  .dark-theme & {
    background-color: #2c3e50;
    
    &:hover {
      background-color: lighten(#2c3e50, 10%);
    }
  }
}

This compiles to:

.button {
  background-color: #3498db;
  color: white;
  padding: 10px 15px;
}

.button:hover {
  background-color: #217dbb;
}

.button:active {
  transform: translateY(1px);
}

.button.disabled {
  opacity: 0.5;
  pointer-events: none;
}

.button--large {
  padding: 15px 20px;
  font-size: 1.2em;
}

.button--small {
  padding: 5px 10px;
  font-size: 0.8em;
}

.dark-theme .button {
  background-color: #2c3e50;
}

.dark-theme .button:hover {
  background-color: #3e5771;
}

Nesting Properties

Sass also allows you to nest properties that share a namespace:

// Without property nesting
.element {
  margin-top: 10px;
  margin-right: 15px;
  margin-bottom: 10px;
  margin-left: 15px;
  padding-top: 20px;
  padding-right: 10px;
  padding-bottom: 20px;
  padding-left: 10px;
}

// With property nesting
.element {
  margin: {
    top: 10px;
    right: 15px;
    bottom: 10px;
    left: 15px;
  }
  padding: {
    top: 20px;
    right: 10px;
    bottom: 20px;
    left: 10px;
  }
}

Media Query Nesting

One of the most powerful uses of nesting is with media queries, allowing you to keep responsive styles with the selectors they modify:

// Without media query nesting
.container {
  width: 100%;
  padding: 10px;
}

@media (min-width: 768px) {
  .container {
    width: 750px;
    padding: 20px;
  }
}

@media (min-width: 992px) {
  .container {
    width: 970px;
  }
}

// With media query nesting
.container {
  width: 100%;
  padding: 10px;
  
  @media (min-width: 768px) {
    width: 750px;
    padding: 20px;
  }
  
  @media (min-width: 992px) {
    width: 970px;
  }
}

Avoiding Excessive Nesting

While nesting is powerful, excessive nesting can lead to:

  • Overly specific selectors that are hard to override
  • CSS bloat with unnecessarily long selectors
  • Styles that are too tightly coupled to HTML structure

Best practice is to limit nesting to 3-4 levels deep and use nesting primarily for:

  • Parent-child relationships that are unlikely to change
  • Pseudo-classes and pseudo-elements
  • Media queries
  • State variations (using the & operator)

Sass Features: Mixins and Functions

Mixins and functions are powerful features in Sass that allow you to reuse styles and logic throughout your stylesheets.

Mixins

Mixins are reusable blocks of styles that can be included in other selectors. They're defined with @mixin and included with @include:

// Define a mixin
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

// Use the mixin
.container {
  @include center-flex;
  height: 100vh;
}

.card-content {
  @include center-flex;
  height: 200px;
}

Mixins with Parameters

Mixins can accept parameters, making them more flexible:

// Mixin with parameters
@mixin box-shadow($x, $y, $blur, $color) {
  -webkit-box-shadow: $x $y $blur $color;
  -moz-box-shadow: $x $y $blur $color;
  box-shadow: $x $y $blur $color;
}

// Use the mixin with different values
.card {
  @include box-shadow(0, 2px, 5px, rgba(0, 0, 0, 0.1));
}

.modal {
  @include box-shadow(0, 5px, 15px, rgba(0, 0, 0, 0.3));
}

Default Parameter Values

You can set default values for mixin parameters:

@mixin border-radius($radius: 4px) {
  -webkit-border-radius: $radius;
  -moz-border-radius: $radius;
  border-radius: $radius;
}

// Use default value
.button {
  @include border-radius; // Uses 4px
}

// Override default
.button-rounded {
  @include border-radius(20px);
}

Variable Arguments

Mixins can accept variable numbers of arguments using the ellipsis (...) syntax:

@mixin transition($properties...) {
  -webkit-transition: $properties;
  -moz-transition: $properties;
  transition: $properties;
}

// Use with different numbers of arguments
.button {
  @include transition(background-color 0.3s ease);
}

.card {
  @include transition(transform 0.2s ease, box-shadow 0.2s ease);
}

Content Blocks

Mixins can accept content blocks using @content, which is useful for things like media queries:

@mixin respond-to($breakpoint) {
  @if $breakpoint == small {
    @media (max-width: 576px) {
      @content;
    }
  } @else if $breakpoint == medium {
    @media (max-width: 768px) {
      @content;
    }
  } @else if $breakpoint == large {
    @media (max-width: 992px) {
      @content;
    }
  }
}

// Use the mixin with content
.container {
  width: 1200px;
  
  @include respond-to(large) {
    width: 900px;
  }
  
  @include respond-to(medium) {
    width: 700px;
  }
  
  @include respond-to(small) {
    width: 100%;
  }
}

Functions

While mixins output CSS, functions return values that can be used in property values. Functions are defined with @function and return values with @return:

// Define a function to convert pixels to rem
@function rem($pixels, $context: 16) {
  @return ($pixels / $context) * 1rem;
}

// Use the function
body {
  font-size: rem(16);
}

h1 {
  font-size: rem(32);
  margin-bottom: rem(24);
}

More Function Examples

// Calculate contrast color
@function contrast-color($color) {
  $luminance: (0.2126 * red($color) + 0.7152 * green($color) + 0.0722 * blue($color)) / 255;
  
  @return if($luminance > 0.5, #000, #fff);
}

// Calculate grid column width
@function column-width($columns, $total-columns: 12, $gap: 20px) {
  $total-gap: ($total-columns - 1) * $gap;
  $column-width: (100% - $total-gap) / $total-columns;
  
  @return $column-width * $columns + ($columns - 1) * $gap;
}

// Usage
.button {
  background-color: #3498db;
  color: contrast-color(#3498db); // Returns white
}

.column {
  width: column-width(4); // 4 columns out of 12
}

Built-in Functions

Sass provides many built-in functions for working with colors, numbers, strings, lists, and maps:

// Color functions
$primary: #3498db;
$light-primary: lighten($primary, 20%);
$dark-primary: darken($primary, 20%);
$desaturated: desaturate($primary, 30%);
$complementary: complement($primary);

// List functions
$sizes: 10px 20px 30px 40px;
$first-size: nth($sizes, 1); // 10px
$list-length: length($sizes); // 4
$has-size: index($sizes, 30px); // 3
$reversed: list.reverse($sizes); // 40px 30px 20px 10px

// Map functions
$theme: (
  'primary': #3498db,
  'secondary': #2ecc71,
  'text': #333
);

$primary-color: map-get($theme, 'primary');
$has-key: map-has-key($theme, 'text'); // true
$all-keys: map-keys($theme); // 'primary', 'secondary', 'text'

Mixins vs. Functions

Understanding when to use each:

  • Use mixins when you need to generate CSS rules
  • Use functions when you need to compute and return values

For example:

  • A border-radius mixin would output vendor prefixes and the standard property
  • A contrast-color function would calculate and return a color value

Sass Features: Control Directives and Modules

Sass provides programming-like control directives and a module system for organizing your code.

Conditionals (@if, @else)

Sass allows you to use conditional logic in your stylesheets:

$theme: 'dark';

body {
  @if $theme == 'dark' {
    background-color: #121212;
    color: #f5f5f5;
  } @else if $theme == 'light' {
    background-color: #f5f5f5;
    color: #121212;
  } @else {
    background-color: white;
    color: black;
  }
}

@mixin button-variant($color, $background) {
  color: $color;
  background-color: $background;
  
  @if lightness($background) > 50% {
    border: 1px solid darken($background, 10%);
  } @else {
    border: 1px solid lighten($background, 10%);
  }
}

Loops (@for, @each, @while)

Sass provides several types of loops for generating repetitive styles:

@for Loop

// Generate grid columns
@for $i from 1 through 12 {
  .col-#{$i} {
    width: percentage($i / 12);
  }
}

// Generate heading sizes
@for $i from 1 through 6 {
  h#{$i} {
    font-size: 3rem - ($i * 0.4rem);
  }
}

@each Loop

// Loop through a list
$colors: (primary, #3498db), (secondary, #2ecc71), (danger, #e74c3c);

@each $name, $color in $colors {
  .text-#{$name} {
    color: $color;
  }
  
  .bg-#{$name} {
    background-color: $color;
  }
}

// Loop through a map
$breakpoints: (
  small: 576px,
  medium: 768px,
  large: 992px,
  xlarge: 1200px
);

@each $name, $size in $breakpoints {
  .visible-#{$name} {
    display: none;
    
    @media (min-width: $size) {
      display: block;
    }
  }
}

@while Loop

// Generate decreasing opacity variants
$opacity: 1;
$i: 10;

@while $i > 0 {
  .opacity-#{$i * 10} {
    opacity: $opacity;
  }
  
  $opacity: $opacity - 0.1;
  $i: $i - 1;
}

Modules and Partials

Sass allows you to split your code into multiple files (partials) and import them using the @import or @use directives.

Partials

Partials are Sass files that start with an underscore (_) and aren't compiled to CSS on their own. They're meant to be imported into other files.

// _variables.scss
$primary-color: #3498db;
$secondary-color: #2ecc71;
$font-stack: 'Poppins', sans-serif;

// _mixins.scss
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

// main.scss
@import 'variables';
@import 'mixins';

body {
  font-family: $font-stack;
}

.container {
  @include flex-center;
}

The @use Rule (Modern Approach)

The @use rule is the modern replacement for @import in Sass, providing better encapsulation and namespace control:

// _colors.scss
$primary: #3498db;
$secondary: #2ecc71;

@function contrast($color) {
  @return if(lightness($color) > 50%, black, white);
}

// _typography.scss
$font-stack: 'Poppins', sans-serif;
$base-size: 16px;

@mixin heading($level) {
  font-weight: bold;
  font-size: $base-size * (2 - ($level - 1) * 0.2);
}

// main.scss
@use 'colors';
@use 'typography' as type;

body {
  font-family: type.$font-stack;
  color: colors.$primary;
}

h1 {
  @include type.heading(1);
  color: colors.contrast(colors.$primary);
}

Namespaces and Aliases

With @use, you can control namespaces or create aliases:

// Default namespace (filename)
@use 'colors';
.element { color: colors.$primary; }

// Custom namespace
@use 'colors' as c;
.element { color: c.$primary; }

// No namespace
@use 'colors' as *;
.element { color: $primary; } // Not recommended for large projects

The @forward Rule

The @forward rule lets you combine multiple Sass files while maintaining their namespaces:

// _base-variables.scss
$font-stack: 'Poppins', sans-serif;
$base-size: 16px;

// _theme-variables.scss
$primary: #3498db;
$secondary: #2ecc71;

// _variables.scss (combines both)
@forward 'base-variables';
@forward 'theme-variables';

// main.scss
@use 'variables';
body {
  font-family: variables.$font-stack;
  color: variables.$primary;
}

@import vs. @use

While @import is still widely used, @use is recommended for new projects because it:

  • Only imports each file once, regardless of how many times it's used
  • Creates namespaces to avoid variable name collisions
  • Makes dependencies explicit and easier to track
  • Allows private members with a leading underscore (_)

Note that @import is being deprecated and will eventually be removed from Sass.

Other CSS Preprocessors

While Sass is the most popular CSS preprocessor, Less and Stylus are also widely used and offer their own unique features and syntax.

Less

Less (Leaner Style Sheets) is a JavaScript-based preprocessor that's known for its simplicity and CSS-like syntax.

Variables in Less

// Variables use @ symbol
@primary-color: #3498db;
@secondary-color: #2ecc71;
@base-padding: 15px;

.button {
  background-color: @primary-color;
  padding: @base-padding;
  
  &:hover {
    background-color: darken(@primary-color, 10%);
  }
}

Mixins in Less

// Simple mixin
.border-radius(@radius) {
  -webkit-border-radius: @radius;
  -moz-border-radius: @radius;
  border-radius: @radius;
}

// Using the mixin
.button {
  .border-radius(4px);
}

// Mixins with default parameters
.box-shadow(@x: 0, @y: 1px, @blur: 2px, @color: rgba(0, 0, 0, 0.1)) {
  box-shadow: @x @y @blur @color;
}

// Using with defaults
.card {
  .box-shadow();
}

// Overriding defaults
.modal {
  .box-shadow(0, 5px, 15px, rgba(0, 0, 0, 0.3));
}

Operations in Less

@base-size: 16px;
@line-height: 1.5;

body {
  font-size: @base-size;
  line-height: @line-height;
}

h1 {
  font-size: @base-size * 2;
  margin-bottom: @base-size * @line-height;
}

Nesting in Less

.navbar {
  background-color: #333;
  
  .nav-item {
    display: inline-block;
    
    a {
      color: white;
      padding: 10px 15px;
      
      &:hover {
        color: #ddd;
      }
    }
  }
}

Stylus

Stylus is known for its flexible syntax, allowing you to write CSS with or without punctuation.

Variables in Stylus

// Variables don't need $ or @ prefix
primary-color = #3498db
secondary-color = #2ecc71
base-padding = 15px

.button
  background-color primary-color
  padding base-padding
  
  &:hover
    background-color darken(primary-color, 10%)

Mixins in Stylus

// Define a mixin
border-radius(radius)
  -webkit-border-radius radius
  -moz-border-radius radius
  border-radius radius

// Use the mixin
.button
  border-radius(4px)

// Mixin with default values
box-shadow(x = 0, y = 1px, blur = 2px, color = rgba(0, 0, 0, 0.1))
  box-shadow x y blur color

// Use with defaults
.card
  box-shadow()

// Override defaults
.modal
  box-shadow(0, 5px, 15px, rgba(0, 0, 0, 0.3))

Optional Syntax in Stylus

Stylus allows you to write CSS with or without braces, colons, and semicolons:

// Full CSS-style syntax
.element {
  color: #333;
  font-size: 16px;
}

// Without braces
.element
  color: #333
  font-size: 16px

// Without colons
.element
  color #333
  font-size 16px

// Without parentheses in function calls
.element
  background-color darken(#3498db, 10%)
  // or
  background-color darken #3498db, 10%

Iteration in Stylus

// Generate grid columns
for i in (1..12)
  .col-{i}
    width (i / 12 * 100)%

// Loop through a list
sizes = 'small' 'medium' 'large'
for size in sizes
  .btn-{size}
    if size == 'small'
      padding 5px 10px
    else if size == 'medium'
      padding 10px 15px
    else
      padding 15px 20px

Comparison of Preprocessors

Here's a quick comparison of the three main preprocessors:

Choosing a Preprocessor

When deciding which preprocessor to use, consider:

  • Team familiarity: Which one does your team already know?
  • Project requirements: Do you need specific features?
  • Ecosystem: What libraries and frameworks are available?
  • Build process: How does it integrate with your build tools?

Sass is generally the most feature-rich and widely used, Less is the easiest to learn, and Stylus offers the most flexibility in syntax.

Modern Workflow with Sass

In modern web development, Sass is typically integrated into a build process using tools like Webpack, Gulp, or npm scripts.

Integration with Build Tools

Webpack

// webpack.config.js
module.exports = {
  // ...
  module: {
    rules: [
      {
        test: /\.scss$/,
        use: [
          'style-loader', // Injects styles into DOM
          'css-loader',   // Translates CSS into CommonJS
          'sass-loader'   // Compiles Sass to CSS
        ]
      }
    ]
  }
};

Gulp

// gulpfile.js
const gulp = require('gulp');
const sass = require('gulp-sass')(require('sass'));
const autoprefixer = require('gulp-autoprefixer');
const sourcemaps = require('gulp-sourcemaps');

function styles() {
  return gulp.src('./src/scss/**/*.scss')
    .pipe(sourcemaps.init())
    .pipe(sass().on('error', sass.logError))
    .pipe(autoprefixer())
    .pipe(sourcemaps.write('./'))
    .pipe(gulp.dest('./dist/css'));
}

exports.styles = styles;
exports.watch = function() {
  gulp.watch('./src/scss/**/*.scss', styles);
};

Source maps help with debugging by mapping compiled CSS back to the original Sass files:

With npm scripts:

}

Best Practices for Sass Projects

  • Follow a consistent architecture like 7-1 pattern (7 folders, 1 main file)
  • Use partials to organize code into logical chunks
  • Adopt a naming convention like BEM for classes
  • Keep nesting shallow (3-4 levels max)
  • Document your code with comments
  • Create a style guide for your project
  • Use source maps for easier debugging
  • Optimize for production with minification and autoprefixing

Step 3: Create Functions

In _functions.scss:

// Get color from map
@function color($color-name) {
  @return map-get($colors, $color-name);
}

// Calculate contrast color (black or white)
@function contrast-color($color) {
  $luminance: (0.2126 * red($color) + 0.7152 * green($color) + 0.0722 * blue($color)) / 255;
  
  @return if($luminance > 0.5, #000, #fff);
}

// Get size property
@function size($size-name, $property) {
  $size-map: map-get($sizes, $size-name);
  @return map-get($size-map, $property);
}

Step 4: Create Mixins

// Button base styles
@mixin button-base {
  display: inline-block;
  font-weight: 400;
  text-align: center;
  white-space: nowrap;
  vertical-align: middle;
  user-select: none;
  border: 1px solid transparent;
  transition: all $transition-speed ease-in-out;
  cursor: pointer;
  
  &:focus {
    outline: 0;
    box-shadow: 0 0 0 0.2rem rgba(color('primary'), 0.25);
  }
  
  &:disabled {
    opacity: 0.65;
    cursor: not-allowed;
  }
}

// Button variant
@mixin button-variant($color-name) {
  $background: color($color-name);
  $text-color: contrast-color($background);
  $hover-background: darken($background, 10%);
  $active-background: darken($background, 15%);
  
  background-color: $background;
  color: $text-color;
  
  &:hover {
    background-color: $hover-background;
    color: $text-color;
  }
  
  &:active {
    background-color: $active-background;
    transform: translateY(1px);
  }
}

// Button outline variant
@mixin button-outline-variant($color-name) {
  $color: color($color-name);
  
  background-color: transparent;
  color: $color;
  border-color: $color;
  
  &:hover {
    background-color: $color;
    color: contrast-color($color);
  }
}

// Button size
@mixin button-size($size-name) {
  padding: size($size-name, 'padding');
  font-size: size($size-name, 'font-size');
}

Summary

  • Variables and Calculations: Store and reuse values, perform mathematical operations, and manipulate colors
  • Nesting and Parent Selectors: Write more concise and readable CSS that mirrors HTML structure
  • Mixins and Functions: Create reusable blocks of styles and logic to reduce repetition
  • Control Directives and Modules: Use programming constructs like conditionals and loops, and organize code into modular files
  • Different Preprocessors: Compare Sass, Less, and Stylus to understand their unique features and syntax
  • Modern Workflow: Integrate Sass into a build process with tools like npm scripts, Webpack, or Gulp

CSS preprocessors remain a vital part of modern web development, even as native CSS continues to evolve with features like custom properties and calculations. By leveraging preprocessors, you can write more maintainable, scalable, and efficient stylesheets while maintaining compatibility with all browsers through the compiled CSS output.
