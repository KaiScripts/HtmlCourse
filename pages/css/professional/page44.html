CSS-in-JS

Introduction to CSS-in-JS

CSS-in-JS is a styling technique where JavaScript is used to define and generate CSS instead of writing it in separate files. This approach emerged as web applications became more component-based, and developers sought ways to better encapsulate styles with their corresponding components.

In this lesson, we'll explore the various CSS-in-JS libraries, their approaches, benefits, drawbacks, and best practices. By the end, you'll understand when and how to use CSS-in-JS effectively in your projects.

The Evolution of CSS Approaches

To understand CSS-in-JS, it's helpful to look at how CSS approaches have evolved:

1996

Inline Styles

CSS was initially written directly in HTML using the style attribute. This approach lacked reusability and separation of concerns.

Early 2000s

External Stylesheets

CSS moved to separate files, promoting separation of concerns and reusability, but creating challenges with global scope and specificity.

2009-2012

CSS Preprocessors

Tools like Sass and Less introduced variables, nesting, and mixins, making CSS more maintainable but still not solving scoping issues.

2013-2015

BEM and CSS Methodologies

Naming conventions like BEM helped manage CSS at scale by creating explicit relationships between components and their styles.

2014-2016

CSS Modules

CSS Modules introduced local scoping by default, allowing developers to write CSS that was automatically scoped to specific components.

2014-Present

CSS-in-JS

Libraries like Styled Components and Emotion brought CSS directly into JavaScript, enabling component-scoped styles with full access to the JavaScript ecosystem.

Why CSS-in-JS?

CSS-in-JS emerged to solve several challenges in modern web development:

Key Problems CSS-in-JS Addresses

  • Global Namespace: Traditional CSS uses a global namespace, leading to potential naming conflicts
  • Dependencies: CSS lacks a built-in way to manage dependencies between styles
  • Dead Code Elimination: It's difficult to know which CSS is actually being used
  • Minification: Class names in traditional CSS can't be safely minified
  • Sharing Constants: Sharing values between CSS and JavaScript is cumbersome
  • Non-deterministic Resolution: CSS specificity rules can lead to unpredictable results
  • Isolation: Styles for one component can unintentionally affect others

Core Concepts of CSS-in-JS

While different CSS-in-JS libraries have varying APIs, they share several core concepts:

Component-Scoped Styles

CSS-in-JS libraries typically generate unique class names for each component, ensuring styles don't leak between components:

// Using styled-components
import styled from 'styled-components';

const Button = styled.button`
  background-color: #3498db;
  color: white;
  padding: 10px 15px;
  border: none;
  border-radius: 4px;
  
  &:hover {
    background-color: #2980b9;
  }
`;

// Usage
function App() {
  return <Button>Click Me</Button>;
}

Dynamic Styling Based on Props

CSS-in-JS allows styles to change based on component props:

const Button = styled.button`
  background-color: ${props => props.primary ? '#3498db' : '#e74c3c'};
  color: white;
  padding: 10px 15px;
  border: none;
  border-radius: 4px;
  
  &:hover {
    background-color: ${props => props.primary ? '#2980b9' : '#c0392b'};
  }
`;

// Usage
function App() {
  return (
    <div>
      <Button primary>Primary Button</Button>
      <Button>Secondary Button</Button>
    </div>
  );
}

Theming

Most CSS-in-JS libraries provide theming capabilities:

import { ThemeProvider } from 'styled-components';

const theme = {
  colors: {
    primary: '#3498db',
    secondary: '#e74c3c',
    text: '#333',
    background: '#f9f9f9'
  },

    body: 'Roboto, sans-serif',
    heading: 'Montserrat, sans-serif'
  },

    small: '8px',
    medium: '16px',

  }

const Button = styled.button`

  color: white;
  font-family: ${props => props.theme.fonts.body};

function App() {
  return (
    <ThemeProvider theme={theme}>
      <Button>Themed Button</Button>
    </ThemeProvider>
  );
}

Global Styles

CSS-in-JS libraries also provide ways to define global styles:

import { createGlobalStyle } from 'styled-components';

const GlobalStyle = createGlobalStyle`
  * {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
  }
  
  body {
    font-family: 'Roboto', sans-serif;
    background-color: #f9f9f9;
    color: #333;
  }
  
  a {
    color: #3498db;
    text-decoration: none;
  }
`;

function App() {
  return (
    <>
      <GlobalStyle />
      <div>Your app content</div>
    </>
  );
}

Popular CSS-in-JS Libraries

Several CSS-in-JS libraries have emerged, each with its own approach and features. Let's explore the most popular ones:

Styled Components

Styled Components is one of the most popular CSS-in-JS libraries, focusing on component-based styling with tagged template literals.

Key Features

  • Automatic critical CSS injection
  • No class name bugs
  • Easier deletion of CSS
  • Simple dynamic styling
  • Painless maintenance
  • Automatic vendor prefixing

Basic Usage

import styled from 'styled-components';

// Create a styled component
const Title = styled.h1`
  font-size: 1.5rem;
  text-align: center;
  color: #333;
`;

const Wrapper = styled.section`
  padding: 4em;
  background: papayawhip;
`;

// Use the components
function App() {
  return (
    <Wrapper>
      <Title>Hello World!</Title>
    </Wrapper>
  );
}

Extending Styles

const Button = styled.button`
  background: transparent;
  border-radius: 3px;
  border: 2px solid palevioletred;
  color: palevioletred;
  margin: 0 1em;
  padding: 0.25em 1em;
`;

// Extend the Button component
const PrimaryButton = styled(Button)`
  background: palevioletred;
  color: white;
`;

Adapting Based on Props

const Button = styled.button`
  background: ${props => props.primary ? 'palevioletred' : 'white'};
  color: ${props => props.primary ? 'white' : 'palevioletred'};
  border: 2px solid palevioletred;
  border-radius: 3px;
`;

function App() {
  return (
    <div>
      <Button>Normal Button</Button>
      <Button primary>Primary Button</Button>
    </div>
  );
}

Emotion

Emotion is a flexible CSS-in-JS library designed for high performance and developer experience.

Key Features

  • Highly performant
  • Flexible and composable
  • Works with and without React
  • Supports source maps
  • Minimal size

Basic Usage with React

/** @jsx jsx */
import { css, jsx } from '@emotion/react';

const style = css`
  color: hotpink;
`;

function App() {
  return (
    <div css={style}>
      This has a hotpink color
    </div>
  );
}

Styled Components API

import styled from '@emotion/styled';

const Button = styled.button`
  color: ${props => props.primary ? 'white' : 'hotpink'};
  background-color: ${props => props.primary ? 'hotpink' : 'white'};
  border: 2px solid hotpink;
  border-radius: 3px;
`;

function App() {
  return (
    <div>
      <Button>Normal Button</Button>
      <Button primary>Primary Button</Button>
    </div>
  );
}

Object Styles

/** @jsx jsx */
import { jsx } from '@emotion/react';

function App() {
  return (
    <div
      css={{
        color: 'hotpink',
        '&:hover': {
          color: 'darkorchid'
        }
      }}
    >
      This has a hotpink color and turns darkorchid on hover
    </div>
  );
}

JSS

JSS is a more authoring tool that allows you to use JavaScript to describe styles. It's used by Material-UI and other libraries.

Key Features

  • Framework agnostic
  • Supports plugins
  • Small size
  • High performance

Basic Usage

import { createUseStyles } from 'react-jss';

// Create a styles object
const useStyles = createUseStyles({
  myButton: {
    color: 'green',
    margin: {
      top: 5,
      right: 0,
      bottom: 0,
      left: '1rem'
    },
    '& span': {
      fontWeight: 'bold'
    }
  },
  myLabel: {
    fontStyle: 'italic'
  }
});

function Button() {
  const classes = useStyles();
  return (
    <button className={classes.myButton}>
      <span className={classes.myLabel}>Click me!</span>
    </button>
  );
}

Dynamic Values

const useStyles = createUseStyles({
  myButton: {
    padding: props => props.size === 'small' ? '5px 10px' : '10px 20px',
    background: props => props.primary ? 'blue' : 'gray',
    color: 'white'
  }
});

function Button({ size, primary, children }) {
  const classes = useStyles({ size, primary });
  return (
    <button className={classes.myButton}>
      {children}
    </button>
  );
}

CSS Modules

While not strictly CSS-in-JS, CSS Modules is a popular approach that provides local scoping for CSS classes and is often used alongside JavaScript frameworks.

Key Features

  • Local scope by default
  • Explicit dependencies
  • No runtime overhead
  • Works with existing CSS

Basic Usage

/* Button.module.css */
.button {
  background-color: blue;
  color: white;
  padding: 10px 15px;
  border: none;
  border-radius: 4px;
}

.primary {
  background-color: green;
}

.large {
  font-size: 1.2rem;
  padding: 12px 20px;
}
// Button.jsx
import React from 'react';
import styles from './Button.module.css';

function Button({ primary, large, children }) {
  const buttonClass = `${styles.button} ${primary ? styles.primary : ''} ${large ? styles.large : ''}`;
  
  return (
    <button className={buttonClass}>
      {children}
    </button>
  );
}

Advanced CSS-in-JS Techniques

Now that we've covered the basics, let's explore some advanced techniques and patterns used in CSS-in-JS.

Component Composition

CSS-in-JS libraries make it easy to compose styles by extending components:

import styled from 'styled-components';

// Base button component
const Button = styled.button`
  padding: 10px 15px;
  border-radius: 4px;
  font-size: 1rem;
  cursor: pointer;
  transition: all 0.2s ease;
`;

// Primary button extends the base button
const PrimaryButton = styled(Button)`
  background-color: #3498db;
  color: white;
  border: none;
  
  &:hover {
    background-color: #2980b9;
  }
`;

// Secondary button also extends the base button
const SecondaryButton = styled(Button)`
  background-color: white;
  color: #3498db;
  border: 2px solid #3498db;
  
  &:hover {
    background-color: #f8f9fa;
  }
`;

// Danger button extends the primary button but changes colors
const DangerButton = styled(PrimaryButton)`
  background-color: #e74c3c;
  
  &:hover {
    background-color: #c0392b;
  }
`;

Server-Side Rendering

CSS-in-JS libraries provide solutions for server-side rendering (SSR) to avoid the flash of unstyled content:

// Example with styled-components and Next.js
import Document, { Html, Head, Main, NextScript } from 'next/document';
import { ServerStyleSheet } from 'styled-components';

export default class MyDocument extends Document {
  static async getInitialProps(ctx) {
    const sheet = new ServerStyleSheet();
    const originalRenderPage = ctx.renderPage;

    try {
      ctx.renderPage = () =>
        originalRenderPage({
          enhanceApp: (App) => (props) =>
            sheet.collectStyles(<App {...props} />),
        });

      const initialProps = await Document.getInitialProps(ctx);
      return {
        ...initialProps,
        styles: (
          <>
            {initialProps.styles}
            {sheet.getStyleElement()}
          </>
        ),
      };
    } finally {
      sheet.seal();
    }
  }

  render() {
    return (
      <Html>
        <Head />
        <body>
          <Main />
          <NextScript />
        </body>
      </Html>
    );
  }
}

Zero-Runtime CSS-in-JS

A newer trend in CSS-in-JS is zero-runtime solutions that extract CSS at build time:

// Example with Linaria
import { styled } from '@linaria/react';

const Button = styled.button`
  background-color: ${props => props.primary ? 'blue' : 'gray'};
  color: white;
  padding: 10px 15px;
  border: none;
  border-radius: 4px;
  
  &:hover {
    opacity: 0.8;
  }
`;

// This gets compiled to regular CSS at build time
// No runtime overhead!

Benefits of Zero-Runtime CSS-in-JS

  • No runtime performance overhead
  • Smaller bundle size
  • Better performance on mobile devices
  • Still provides the developer experience benefits of CSS-in-JS

Popular zero-runtime CSS-in-JS libraries include Linaria, Astroturf, and vanilla-extract.

CSS-in-JS Best Practices

Let's explore some best practices for using CSS-in-JS effectively in your projects.

Performance Considerations

CSS-in-JS can impact performance if not used carefully:

Avoid Creating Styles in Render

// Bad: Creates new styles on every render
function Button({ color }) {
  // This creates a new component on every render
  const DynamicButton = styled.button`
    background-color: ${color};
    color: white;
    padding: 10px 15px;
  `;
  
  return <DynamicButton>Click Me</DynamicButton>;
}

// Good: Create styles outside the component
const Button = styled.button`
  background-color: ${props => props.color};
  color: white;
  padding: 10px 15px;
`;

function ButtonComponent({ color }) {
  return <Button color={color}>Click Me</Button>;
}

Use Memoization for Dynamic Styles

import { useMemo } from 'react';
import { css } from '@emotion/react';

function Component({ color, fontSize }) {
  // Memoize the styles to prevent recalculation on every render
  const styles = useMemo(() => ({
    container: css`
      background-color: ${color};
      font-size: ${fontSize}px;
    `
  }), [color, fontSize]);
  
  return <div css={styles.container}>Content</div>;
}

Consider Zero-Runtime Solutions

For performance-critical applications, consider using zero-runtime CSS-in-JS libraries like Linaria or vanilla-extract.

Organization and Maintainability

Proper organization is key to maintaining CSS-in-JS codebases:

Separate Styled Components from Logic

import styled from 'styled-components';

Testing CSS-in-JS

Testing components with CSS-in-JS requires some special considerations:

Setting Up Theme Provider for Tests

// test-utils.js
import React from 'react';
import { render } from '@testing-library/react';
import { ThemeProvider } from 'styled-components';
import theme from './theme';

const AllTheProviders = ({ children }) => {
  return (
    <ThemeProvider theme={theme}>
      {children}
    </ThemeProvider>
  );

  render(ui, { wrapper: AllTheProviders, ...options });

// re-export everything

Pros and Cons of CSS-in-JS

Let's evaluate the advantages and disadvantages of CSS-in-JS to help you decide if it's right for your project.

Advantages of CSS-in-JS

  • Scoped Styles: CSS-in-JS provides automatic scoping, eliminating the risk of style conflicts
  • Dynamic Styling: Styles can easily adapt based on props and state
  • Colocated Code: Styles live with the components they style, improving maintainability
  • Dead Code Elimination: Unused styles are automatically removed when components are deleted
  • Critical CSS Extraction: Many libraries automatically extract and inline critical CSS
  • TypeScript Integration: Better type safety for styles and props
  • Full JavaScript Power: Access to JavaScript variables, functions, and logic in styles
  • Theming: Robust theming capabilities with access to theme values in components

Disadvantages of CSS-in-JS

  • Runtime Overhead: Most CSS-in-JS libraries add some JavaScript overhead
  • Bundle Size: Adds additional code to your JavaScript bundle
  • Learning Curve: Requires learning new APIs and patterns
  • Build Complexity: May require additional build configuration
  • Developer Tools: Browser DevTools integration isn't always as seamless as with traditional CSS
  • SSR Complexity: Server-side rendering requires additional setup
  • Legacy Browser Support: Some features may require polyfills for older browsers

When to Use CSS-in-JS

CSS-in-JS is particularly well-suited for:

  • Component-based applications (React, Vue, etc.)
  • Projects where component reusability is important
  • Applications with complex theming requirements
  • Teams familiar with JavaScript who want to minimize context switching
  • Projects where styles need to be dynamically generated based on props or state

When to Consider Alternatives

Traditional CSS approaches might be better for:

  • Performance-critical applications where every kilobyte matters
  • Projects with significant existing CSS that would be costly to migrate
  • Teams with specialized CSS developers who prefer traditional CSS workflows
  • Simple websites or applications where the benefits of CSS-in-JS don't outweigh the costs
  • Projects where browser support for older versions is critical

Finding the Right Balance

Many projects use a hybrid approach:

  • Global styles and resets with traditional CSS
  • Component-specific styles with CSS-in-JS
  • Consider zero-runtime CSS-in-JS for performance-critical applications
  • Use CSS Modules as a middle ground between traditional CSS and full CSS-in-JS

// src/components/Button/index.js
import styled, { css } from 'styled-components';

const getButtonColors = (props) => {
  const { variant, theme, color = 'primary' } = props;
  const colorValue = theme.colors[color];
  
  if (variant === 'outlined') {
    return css`
      color: ${colorValue};
      background-color: transparent;
      border: 1px solid ${colorValue};
      
      &:hover {
        background-color: ${colorValue}20;
      }
    `;
  }
  
  if (variant === 'text') {
    return css`
      color: ${colorValue};
      background-color: transparent;
      border: none;
      
      &:hover {
        background-color: ${colorValue}10;
      }
    `;
  }
  
  // Default is 'contained'
  return css`
    color: ${theme.colors.text.light};
    background-color: ${colorValue};
    border: none;
    
    &:hover {
      background-color: ${colorValue}dd;
    }
  `;
};

const getSizeStyles = (props) => {
  const { size = 'medium', theme } = props;
  
  if (size === 'small') {
    return css`
      padding: ${theme.spacing.xs} ${theme.spacing.md};
      font-size: ${theme.typography.fontSize.small};
    `;
  }
  
  if (size === 'large') {
    return css`
      padding: ${theme.spacing.md} ${theme.spacing.xl};
      font-size: ${theme.typography.fontSize.large};
    `;
  }
  
  // Default is 'medium'
  return css`
    padding: ${theme.spacing.sm} ${theme.spacing.lg};
    font-size: ${theme.typography.fontSize.medium};
  `;
};

const Button = styled.button`
  font-family: ${props => props.theme.typography.fontFamily};
  font-weight: ${props => props.theme.typography.fontWeight.medium};
  border-radius: ${props => props.theme.borderRadius.small};
  cursor: pointer;
  transition: all ${props => props.theme.transitions.short};
  
  /* Apply color styles based on variant */
  ${props => getButtonColors(props)}
  
  /* Apply size styles */
  ${props => getSizeStyles(props)}
  
  /* Full width option */
  ${props => props.fullWidth && css`
    width: 100%;
  `}
  
  /* Disabled state */
  ${props => props.disabled && css`
    opacity: 0.5;
    cursor: not-allowed;
    
    &:hover {
      background-color: ${
        props.variant === 'contained' 
          ? props.theme.colors[props.color || 'primary'] 
          : props.variant === 'outlined' 
            ? 'transparent' 
            : 'transparent'
      };
    }
  `}
`;

export default Button;
