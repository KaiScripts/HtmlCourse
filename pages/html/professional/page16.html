HTML Media & Graphics

Rich Media on the Web

HTML5 introduced powerful native support for multimedia and graphics, eliminating the need for plugins like Flash or Silverlight. In this lesson, we'll explore the various HTML elements and technologies for embedding and manipulating media and graphics on web pages.

These capabilities allow developers to create rich, interactive experiences directly in the browser, from simple audio and video playback to complex animations and visualizations.

Why Native Media Matters

Native HTML media and graphics provide several advantages over plugin-based solutions:

  • Better performance with hardware acceleration
  • Improved security without third-party plugins
  • Better accessibility with built-in controls and features
  • Consistent experience across browsers and devices
  • Mobile compatibility without plugin dependencies
  • Deeper integration with JavaScript and CSS

Audio Element

The <audio> element allows you to embed sound content in your web pages without requiring a plugin.

Basic Audio Implementation

<!-- Basic audio with controls -->
<audio src="audio-file.mp3" controls>
    Your browser does not support the audio element.
</audio>

Audio with Multiple Sources

To ensure compatibility across browsers, you can provide multiple audio formats using the <source> element:

<audio controls>
    <source src="audio-file.mp3" type="audio/mpeg">
    <source src="audio-file.ogg" type="audio/ogg">
    <source src="audio-file.wav" type="audio/wav">
    Your browser does not support the audio element.
</audio>

Audio Attributes

The <audio> element supports several attributes to control its behavior:

<audio 
    controls     <!-- Shows the audio player controls -->
    autoplay     <!-- Starts playing automatically (use with caution) -->
    muted        <!-- Mutes the audio initially -->
    loop         <!-- Repeats the audio when finished -->
    preload="auto" <!-- Preloads the audio file (auto, metadata, or none) -->
>
    <source src="audio-file.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Autoplay Considerations

Modern browsers restrict autoplay of media with sound to improve user experience and reduce annoyance. To use autoplay:

  • Always combine autoplay with muted for reliable autoplay
  • Consider using JavaScript to play audio after user interaction
  • Provide clear controls for users to start playback manually

Controlling Audio with JavaScript

The <audio> element provides a rich JavaScript API for controlling playback:

<audio id="myAudio" src="audio-file.mp3"></audio>

<button onclick="playAudio()">Play</button>
<button onclick="pauseAudio()">Pause</button>
<button onclick="setHalfVolume()">Volume 50%</button>
<button onclick="setFullVolume()">Volume 100%</button>

<script>
    const audio = document.getElementById('myAudio');
    
    function playAudio() {
        audio.play();
    }
    
    function pauseAudio() {
        audio.pause();
    }
    
    function setHalfVolume() {
        audio.volume = 0.5;
    }
    
    function setFullVolume() {
        audio.volume = 1.0;
    }
    
    // Listen for events
    audio.addEventListener('play', function() {
        console.log('Audio started playing');
    });
    
    audio.addEventListener('ended', function() {
        console.log('Audio playback finished');
    });
</script>

Common Audio Properties and Methods

Here are some of the most useful properties and methods for the audio element:

Properties:

  • currentTime: Gets or sets the current playback position in seconds
  • duration: Returns the length of the audio in seconds
  • paused: Returns whether the audio is paused
  • volume: Gets or sets the volume (0.0 to 1.0)
  • muted: Gets or sets whether the audio is muted
  • playbackRate: Gets or sets the playback speed (1.0 is normal)
  • ended: Returns whether the audio has ended

Methods:

  • play(): Starts playing the audio
  • pause(): Pauses the audio
  • load(): Reloads the audio element

Events:

  • play: Fired when playback has begun
  • pause: Fired when playback is paused
  • ended: Fired when playback has completed
  • timeupdate: Fired when the current playback position changes
  • volumechange: Fired when the volume changes
  • loadedmetadata: Fired when metadata (like duration) has loaded

Creating a Custom Audio Player

You can create a custom audio player using the JavaScript API:

<div class="custom-audio-player">
    <audio id="customAudio" src="audio-file.mp3" preload="metadata"></audio>
    
    <button id="playPauseBtn" class="player-btn">Play</button>
    
    <div class="progress-container">
        <div id="progressBar" class="progress-bar"></div>
    </div>
    
    <span id="currentTime">0:00</span> / <span id="duration">0:00</span>
    
    <div class="volume-container">
        <button id="muteBtn" class="player-btn">Mute</button>
        <input type="range" id="volumeSlider" min="0" max="1" step="0.1" value="1">
    </div>
</div>

<style>
    .custom-audio-player {
        background: #222;
        border-radius: 8px;
        padding: 15px;
        color: white;
        display: flex;
        align-items: center;
        gap: 10px;
    }
    
    .player-btn {
        background: #3366ff;
        color: white;
        border: none;
        border-radius: 4px;
        padding: 5px 10px;
        cursor: pointer;
    }
    
    .progress-container {
        flex-grow: 1;
        background: #444;
        height: 8px;
        border-radius: 4px;
        cursor: pointer;
        position: relative;
    }
    
    .progress-bar {
        background: #3366ff;
        height: 100%;
        border-radius: 4px;
        width: 0%;
    }
    
    .volume-container {
        display: flex;
        align-items: center;
        gap: 5px;
    }
</style>

<script>
    document.addEventListener('DOMContentLoaded', function() {
        const audio = document.getElementById('customAudio');
        const playPauseBtn = document.getElementById('playPauseBtn');
        const muteBtn = document.getElementById('muteBtn');
        const volumeSlider = document.getElementById('volumeSlider');
        const progressBar = document.getElementById('progressBar');
        const progressContainer = document.querySelector('.progress-container');
        const currentTimeDisplay = document.getElementById('currentTime');
        const durationDisplay = document.getElementById('duration');
        
        // Format time in MM:SS format
        function formatTime(seconds) {
            const minutes = Math.floor(seconds / 60);
            const remainingSeconds = Math.floor(seconds % 60);
            return `${minutes}:${remainingSeconds < 10 ? '0' : ''}${remainingSeconds}`;
        }
        
        // Play/Pause button
        playPauseBtn.addEventListener('click', function() {
            if (audio.paused) {
                audio.play();
                playPauseBtn.textContent = 'Pause';
            } else {
                audio.pause();
                playPauseBtn.textContent = 'Play';
            }
        });
        
        // Mute button
        muteBtn.addEventListener('click', function() {
            audio.muted = !audio.muted;
            muteBtn.textContent = audio.muted ? 'Unmute' : 'Mute';
        });
        
        // Volume slider
        volumeSlider.addEventListener('input', function() {
            audio.volume = this.value;
            if (audio.volume === 0) {
                audio.muted = true;
                muteBtn.textContent = 'Unmute';
            } else {
                audio.muted = false;
                muteBtn.textContent = 'Mute';
            }
        });
        
        // Progress bar
        audio.addEventListener('timeupdate', function() {
            const progress = (audio.currentTime / audio.duration) * 100;
            progressBar.style.width = `${progress}%`;
            currentTimeDisplay.textContent = formatTime(audio.currentTime);
        });
        
        // Click on progress bar to seek
        progressContainer.addEventListener('click', function(e) {
            const rect = this.getBoundingClientRect();
            const pos = (e.clientX - rect.left) / rect.width;
            audio.currentTime = pos * audio.duration;
        });
        
        // Update duration display when metadata is loaded
        audio.addEventListener('loadedmetadata', function() {
            durationDisplay.textContent = formatTime(audio.duration);
        });
        
        // Update button text when playback ends
        audio.addEventListener('ended', function() {
            playPauseBtn.textContent = 'Play';
        });
    });
</script>

Audio Accessibility

To make audio content accessible:

  • Always provide transcripts for audio content
  • Ensure custom controls have proper keyboard support
  • Use ARIA attributes for custom players
  • Don't autoplay audio with sound without user consent
  • Provide visible feedback for audio state changes

Video Element

The <video> element allows you to embed video content in your web pages without requiring a plugin.

Basic Video Implementation

<!-- Basic video with controls -->
<video src="video-file.mp4" controls width="640" height="360">
    Your browser does not support the video element.
</video>

Video with Multiple Sources

To ensure compatibility across browsers, you can provide multiple video formats:

<video controls width="640" height="360" poster="video-thumbnail.jpg">
    <source src="video-file.mp4" type="video/mp4">
    <source src="video-file.webm" type="video/webm">
    <source src="video-file.ogg" type="video/ogg">
    Your browser does not support the video element.
</video>

Video Attributes

The <video> element supports several attributes to control its behavior:

<video 
    controls     <!-- Shows the video player controls -->
    width="640"  <!-- Sets the width of the video player -->
    height="360" <!-- Sets the height of the video player -->
    poster="thumbnail.jpg" <!-- Image to show before video plays -->
    autoplay     <!-- Starts playing automatically (use with caution) -->
    muted        <!-- Mutes the video initially -->
    loop         <!-- Repeats the video when finished -->
    preload="auto" <!-- Preloads the video file (auto, metadata, or none) -->
    playsinline  <!-- Plays inline on iOS instead of fullscreen -->
>
    <source src="video-file.mp4" type="video/mp4">
    Your browser does not support the video element.
</video>

Video Format Considerations

Different browsers support different video formats:

  • MP4 (H.264): Widely supported across all modern browsers
  • WebM (VP8/VP9): Better quality at smaller file sizes, supported in most modern browsers
  • Ogg (Theora): Open format, less widely supported

For maximum compatibility, provide at least MP4 and WebM formats.

Adding Captions and Subtitles

The <track> element allows you to add text tracks like subtitles, captions, descriptions, chapters, or metadata to videos:

<video controls width="640" height="360">
    <source src="video-file.mp4" type="video/mp4">
    
    <!-- Subtitles in English -->
    <track 
        src="subtitles-en.vtt" 
        kind="subtitles" 
        srclang="en" 
        label="English"
        default
    >
    
    <!-- Subtitles in Spanish -->
    <track 
        src="subtitles-es.vtt" 
        kind="subtitles" 
        srclang="es" 
        label="Español"
    >
    
    <!-- Closed captions (includes non-speech sounds) -->
    <track 
        src="captions.vtt" 
        kind="captions" 
        srclang="en" 
        label="Captions"
    >
    
    Your browser does not support the video element.
</video>

The kind attribute can have the following values:

  • subtitles: Translation of foreign language dialogue
  • captions: Transcription of dialogue and non-speech sounds for the deaf or hard-of-hearing
  • descriptions: Text descriptions of the visual content for the blind or visually impaired
  • chapters: Chapter titles for navigation
  • metadata: Content used by scripts but not visible to users

WebVTT Format

Text tracks use the WebVTT (Web Video Text Tracks) format. Here's an example of a simple WebVTT file:

WEBVTT

00:00:05.000 --> 00:00:08.000
Today we'll learn about HTML5 video.

00:00:10.000 --> 00:00:15.000
The video element allows you to embed videos directly in your web pages.

Controlling Video with JavaScript

Like the audio element, the video element provides a rich JavaScript API for controlling playback:

<video id="myVideo" src="video-file.mp4" width="640" height="360"></video>

<div class="video-controls">
    <button onclick="playVideo()">Play</button>
    <button onclick="pauseVideo()">Pause</button>
    <button onclick="skipAhead()">Skip 10s</button>
    <button onclick="toggleFullscreen()">Fullscreen</button>
</div>

<script>
    const video = document.getElementById('myVideo');
    
    function playVideo() {
        video.play();
    }
    
    function pauseVideo() {
        video.pause();
    }
    
    function skipAhead() {
        video.currentTime += 10;
    }
    
    function toggleFullscreen() {
        if (video.requestFullscreen) {
            video.requestFullscreen();
        } else if (video.webkitRequestFullscreen) { /* Safari */
            video.webkitRequestFullscreen();
        } else if (video.msRequestFullscreen) { /* IE11 */
            video.msRequestFullscreen();
        }
    }
    
    // Listen for events
    video.addEventListener('play', function() {
        console.log('Video started playing');
    });
    
    video.addEventListener('ended', function() {
        console.log('Video playback finished');
    });
</script>

Picture-in-Picture

Modern browsers support picture-in-picture mode for videos, allowing users to continue watching while browsing other content:

<video id="pipVideo" src="video-file.mp4" controls></video>
<button id="pipButton">Picture-in-Picture</button>

<script>
    const video = document.getElementById('pipVideo');
    const pipButton = document.getElementById('pipButton');
    
    // Check if Picture-in-Picture is supported
    if (document.pictureInPictureEnabled) {
        pipButton.addEventListener('click', function() {
            if (document.pictureInPictureElement) {
                document.exitPictureInPicture();
            } else {
                video.requestPictureInPicture();
            }
        });
    } else {
        pipButton.disabled = true;
        pipButton.textContent = 'Picture-in-Picture Not Supported';
    }
</script>

Video Performance Tips

  • Use the poster attribute to show an image while the video loads
  • Set preload="none" or preload="metadata" for videos that aren't immediately needed
  • Compress videos appropriately for web delivery
  • Consider using adaptive streaming for longer videos
  • Provide multiple resolutions for different devices and connections
  • Use lazy loading techniques for videos further down the page

Canvas Element

The <canvas> element provides a drawing surface that you can use to create dynamic graphics, animations, and interactive content using JavaScript.

Basic Canvas Setup

<canvas id="myCanvas" width="600" height="400">
    Your browser does not support the canvas element.
</canvas>

<script>
    // Get the canvas element
    const canvas = document.getElementById('myCanvas');
    
    // Get the drawing context
    const ctx = canvas.getContext('2d');
    
    // Now you can draw on the canvas using the context
</script>

Canvas Dimensions

It's important to set the width and height attributes directly on the canvas element, not through CSS. Using CSS to resize a canvas will scale the rendered output, potentially causing blurriness.

Drawing Basic Shapes

The canvas API provides methods for drawing various shapes:

// Get the canvas context
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Set fill and stroke styles
ctx.fillStyle = '#3366ff';
ctx.strokeStyle = '#ff3366';
ctx.lineWidth = 3;

// Draw a rectangle
ctx.fillRect(50, 50, 100, 80); // x, y, width, height

// Draw a stroked rectangle
ctx.strokeRect(200, 50, 100, 80);

// Draw a circle
ctx.beginPath();
ctx.arc(100, 200, 50, 0, Math.PI * 2); // x, y, radius, startAngle, endAngle
ctx.fill();

// Draw a line
ctx.beginPath();
ctx.moveTo(200, 150);
ctx.lineTo(300, 250);
ctx.stroke();

// Draw a path
ctx.beginPath();
ctx.moveTo(350, 50);
ctx.lineTo(450, 50);
ctx.lineTo(400, 150);
ctx.closePath(); // Connects back to the starting point
ctx.fill();
ctx.stroke();

Adding Text

You can add text to a canvas using the text drawing methods:

// Set text properties
ctx.font = 'bold 24px Arial';
ctx.fillStyle = '#333';
ctx.textAlign = 'center';
ctx.textBaseline = 'middle';

// Draw filled text
ctx.fillText('Hello Canvas!', canvas.width / 2, 50);

// Draw outlined text
ctx.strokeText('Outlined Text', canvas.width / 2, 100);

Working with Images

You can draw images onto a canvas:

// Create an image object
const img = new Image();
img.src = 'image.jpg';

// Wait for the image to load
img.onload = function() {
    // Draw the image
    ctx.drawImage(img, 50, 50); // Draw at position x=50, y=50
    
    // Draw with specific dimensions
    ctx.drawImage(img, 200, 50, 150, 100); // x, y, width, height
    
    // Draw a portion of the image (cropping)
    // Parameters: image, sourceX, sourceY, sourceWidth, sourceHeight, 
    //             destX, destY, destWidth, destHeight
    ctx.drawImage(img, 100, 100, 200, 200, 400, 50, 150, 150);
};

Transformations

Canvas supports various transformations to manipulate the drawing context:

// Save the current state
ctx.save();

// Translate (move) the origin
ctx.translate(100, 100);

// Rotate (in radians)
ctx.rotate(Math.PI / 4); // 45 degrees

// Scale
ctx.scale(1.5, 0.5); // x scale, y scale

// Draw a rectangle at the transformed position
ctx.fillStyle = 'green';
ctx.fillRect(-25, -25, 50, 50); // This will be rotated and scaled

// Restore the original state
ctx.restore();

// The next drawing operations will use the original untransformed context
ctx.fillStyle = 'purple';
ctx.fillRect(200, 200, 50, 50);

Animation

You can create animations by repeatedly clearing and redrawing the canvas:

// Animation variables
let x = 0;
let y = canvas.height / 2;
const radius = 20;
let dx = 2; // x velocity

function animate() {
    // Clear the canvas
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    
    // Draw the circle
    ctx.beginPath();
    ctx.arc(x, y, radius, 0, Math.PI * 2);
    ctx.fillStyle = '#3366ff';
    ctx.fill();
    
    // Update position
    x += dx;
    
    // Bounce off walls
    if (x + radius > canvas.width || x - radius < 0) {
        dx = -dx;
    }
    
    // Request next frame
    requestAnimationFrame(animate);
}

// Start the animation
animate();

Gradients and Patterns

Canvas supports gradients and patterns for filling shapes:

// Linear gradient
const linearGradient = ctx.createLinearGradient(0, 0, 200, 0);
linearGradient.addColorStop(0, 'blue');
linearGradient.addColorStop(0.5, 'purple');
linearGradient.addColorStop(1, 'red');

ctx.fillStyle = linearGradient;
ctx.fillRect(50, 50, 200, 100);

// Radial gradient
const radialGradient = ctx.createRadialGradient(
    400, 100, 10,  // Inner circle (x, y, radius)
    400, 100, 80   // Outer circle (x, y, radius)
);
radialGradient.addColorStop(0, 'white');
radialGradient.addColorStop(0.5, 'yellow');
radialGradient.addColorStop(1, 'orange');

ctx.fillStyle = radialGradient;
ctx.beginPath();
ctx.arc(400, 100, 80, 0, Math.PI * 2);
ctx.fill();

// Pattern
const patternImg = new Image();
patternImg.src = 'pattern.png';
patternImg.onload = function() {
    const pattern = ctx.createPattern(patternImg, 'repeat');
    ctx.fillStyle = pattern;
    ctx.fillRect(50, 200, 200, 150);
};

Compositing

Canvas allows you to control how new shapes are composited with existing content:

// Draw a blue rectangle
ctx.fillStyle = 'blue';
ctx.fillRect(50, 50, 100, 100);

// Set compositing operation
ctx.globalCompositeOperation = 'source-over'; // Default - draw on top
// Other values: source-in, source-out, source-atop, 
//               destination-over, destination-in, destination-out, destination-atop,
//               lighter, copy, xor, multiply, screen, overlay, darken, lighten, etc.

// Draw a partially transparent red circle
ctx.fillStyle = 'rgba(255, 0, 0, 0.5)';
ctx.beginPath();
ctx.arc(100, 100, 50, 0, Math.PI * 2);
ctx.fill();

Saving Canvas Content

You can save the content of a canvas as an image:

// Convert canvas to data URL
const dataURL = canvas.toDataURL('image/png'); // Default is PNG
// For JPEG: canvas.toDataURL('image/jpeg', 0.8); // 0.8 is quality (0-1)

// Create a link to download the image
const link = document.createElement('a');
link.download = 'canvas-image.png';
link.href = dataURL;
link.click();

Canvas Performance Tips

  • Minimize the number of state changes (fillStyle, strokeStyle, etc.)
  • Group similar drawing operations together
  • Use requestAnimationFrame instead of setTimeout for animations
  • Avoid unnecessary canvas resizing
  • For complex scenes, consider using multiple canvas layers
  • Use canvas size appropriate for your needs (larger canvas = more memory)
  • For static content, consider drawing once and using the result as an image

Canvas Accessibility

Canvas content is not inherently accessible to screen readers. To improve accessibility:

<canvas id="pieChart" width="400" height="400" 
        aria-label="Pie chart showing sales distribution" 
        role="img">
    <p>Sales Distribution: Electronics 45%, Clothing 30%, Books 15%, Other 10%</p>
</canvas>

For interactive canvas applications, you should:

  • Provide keyboard navigation alternatives
  • Use ARIA live regions to announce changes
  • Consider providing an alternative text-based view
  • Add focus indicators for interactive elements

SVG (Scalable Vector Graphics)

SVG is an XML-based vector image format for two-dimensional graphics. Unlike canvas, SVG is resolution-independent and maintains its quality at any size.

Basic SVG Structure

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <!-- SVG content goes here -->
</svg>

Drawing Basic Shapes

SVG provides elements for various shapes:

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <!-- Rectangle -->
    <rect x="50" y="50" width="100" height="80" fill="#3366ff" stroke="#000" stroke-width="2" />
    
    <!-- Circle -->
    <circle cx="250" cy="90" r="40" fill="#ff3366" />
    
    <!-- Ellipse -->
    <ellipse cx="400" cy="90" rx="60" ry="30" fill="#33ff66" />
    
    <!-- Line -->
    <line x1="50" y1="150" x2="150" y2="200" stroke="#000" stroke-width="3" />
    
    <!-- Polyline (open path) -->
    <polyline points="200,150 250,180 300,150 350,200" 
              fill="none" stroke="#000" stroke-width="2" />
    
    <!-- Polygon (closed path) -->
    <polygon points="400,150 450,150 450,200 400,200" 
             fill="#ffcc00" stroke="#000" stroke-width="2" />
    
    <!-- Path (complex shapes) -->
    <path d="M50,250 L150,250 L100,300 Z" fill="#9933ff" />
    
    <!-- Text -->
    <text x="300" y="250" font-family="Arial" font-size="24" text-anchor="middle">
        SVG Text
    </text>
</svg>

Path Element

The <path> element is the most powerful SVG element, allowing you to create any shape using a series of commands:

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <path d="M100,100 L200,100 L200,200 L100,200 Z" 
          fill="#3366ff" stroke="#000" stroke-width="2" />
          
    <path d="M300,100 
             C350,50 450,50 500,100 
             C550,150 550,250 500,300 
             C450,350 350,350 300,300 
             C250,250 250,150 300,100 Z" 
          fill="#ff3366" />
</svg>

Path commands:

  • M x,y: Move to (absolute)
  • m dx,dy: Move to (relative)
  • L x,y: Line to (absolute)
  • l dx,dy: Line to (relative)
  • H x: Horizontal line to (absolute)
  • h dx: Horizontal line to (relative)
  • V y: Vertical line to (absolute)
  • v dy: Vertical line to (relative)
  • C x1,y1 x2,y2 x,y: Cubic Bézier curve (absolute)
  • c dx1,dy1 dx2,dy2 dx,dy: Cubic Bézier curve (relative)
  • Q x1,y1 x,y: Quadratic Bézier curve (absolute)
  • q dx1,dy1 dx,dy: Quadratic Bézier curve (relative)
  • A rx,ry rotation large-arc-flag sweep-flag x,y: Arc (absolute)
  • a rx,ry rotation large-arc-flag sweep-flag dx,dy: Arc (relative)
  • Z or z: Close path

Styling SVG Elements

SVG elements can be styled with attributes or CSS:

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <!-- Styling with attributes -->
    <rect x="50" y="50" width="100" height="80" 
          fill="#3366ff" 
          stroke="#000" 
          stroke-width="2" 
          opacity="0.8" 
          rx="10" 
          ry="10" />
    
    <!-- Styling with CSS -->
    <style>
        .styled-circle {
            fill: #ff3366;
            stroke: #000;
            stroke-width: 2;
            stroke-dasharray: 10 5; /* Dashed line pattern */
        }
        
        .styled-circle:hover {
            fill: #ffcc00;
            transform: scale(1.1);
            transition: all 0.3s ease;
        }
    </style>
    
    <circle class="styled-circle" cx="250" cy="90" r="40" />
</svg>

Gradients and Patterns

SVG supports gradients and patterns for filling shapes:

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <!-- Define a linear gradient -->
    <defs>
        <linearGradient id="myGradient" x1="0%" y1="0%" x2="100%" y2="0%">
            <stop offset="0%" stop-color="blue" />
            <stop offset="50%" stop-color="purple" />
            <stop offset="100%" stop-color="red" />
        </linearGradient>
        
        <!-- Define a radial gradient -->
        <radialGradient id="myRadialGradient" cx="50%" cy="50%" r="50%" fx="50%" fy="50%">
            <stop offset="0%" stop-color="white" />
            <stop offset="50%" stop-color="yellow" />
            <stop offset="100%" stop-color="orange" />
        </radialGradient>
        
        <!-- Define a pattern -->
        <pattern id="myPattern" x="0" y="0" width="20" height="20" patternUnits="userSpaceOnUse">
            <circle cx="10" cy="10" r="5" fill="#3366ff" />
        </pattern>
    </defs>
    
    <!-- Use the linear gradient -->
    <rect x="50" y="50" width="200" height="100" fill="url(#myGradient)" />
    
    <!-- Use the radial gradient -->
    <circle cx="400" cy="100" r="80" fill="url(#myRadialGradient)" />
    
    <!-- Use the pattern -->
    <rect x="50" y="200" width="200" height="150" fill="url(#myPattern)" />
</svg>

Animation

SVG supports declarative animations with SMIL (Synchronized Multimedia Integration Language):

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <!-- Animated circle -->
    <circle cx="100" cy="100" r="40" fill="#3366ff">
        <!-- Animate position -->
        <animate 
            attributeName="cx" 
            from="100" 
            to="500" 
            dur="3s" 
            repeatCount="indefinite" 
            fill="freeze" />
            
        <!-- Animate color -->
        <animate 
            attributeName="fill" 
            from="#3366ff" 
            to="#ff3366" 
            dur="3s" 
            repeatCount="indefinite" 
            fill="freeze" />
    </circle>
    
    <!-- Animated rectangle -->
    <rect x="100" y="200" width="100" height="50" fill="#33ff66">
        <!-- Animate transformation -->
        <animateTransform 
            attributeName="transform" 
            type="rotate" 
            from="0 150 225" 
            to="360 150 225" 
            dur="5s" 
            repeatCount="indefinite" />
    </rect>
</svg>

You can also animate SVG with CSS or JavaScript:

<!-- CSS Animation -->
<style>
    .animated-circle {
        animation: pulse 2s infinite alternate;
    }
    
    @keyframes pulse {
        from { r: 40; fill: #3366ff; }
        to { r: 60; fill: #ff3366; }
    }
</style>

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <circle class="animated-circle" cx="300" cy="200" r="40" />
</svg>

<!-- JavaScript Animation -->
<svg id="jsSvg" width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <rect id="animatedRect" x="100" y="100" width="100" height="100" fill="#3366ff" />
</svg>

<script>
    const rect = document.getElementById('animatedRect');
    let x = 100;
    let direction = 1;
    
    function animate() {
        x += direction * 2;
        
        if (x > 400 || x < 100) {
            direction *= -1;
        }
        
        rect.setAttribute('x', x);
        requestAnimationFrame(animate);
    }
    
    animate();
</script>

Interactivity

SVG elements can respond to events:

<svg width="600" height="400" viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
    <!-- Click event -->
    <circle id="clickableCircle" cx="100" cy="100" r="40" fill="#3366ff" 
            onclick="alert('Circle clicked!')" />
    
    <!-- Hover styles with CSS -->
    <style>
        #hoverRect {
            fill: #33ff66;
            transition: fill 0.3s ease;
        }
        
        #hoverRect:hover {
            fill: #ff3366;
            cursor: pointer;
        }
    </style>
    
    <rect id="hoverRect" x="200" y="60" width="100" height="80" />
    
    <!-- JavaScript events -->
    <polygon id="interactivePolygon" 
             points="400,60 450,60 475,100 450,140 400,140 375,100" 
             fill="#ffcc00" />
</svg>

<script>
    const polygon = document.getElementById('interactivePolygon');
    
    polygon.addEventListener('mouseenter', function() {
        this.setAttribute('fill', '#ff3366');
    });
    
    polygon.addEventListener('mouseleave', function() {
        this.setAttribute('fill', '#ffcc00');
    });
    
    polygon.addEventListener('click', function() {
        const currentFill = this.getAttribute('fill');
        const newFill = currentFill === '#ff3366' ? '#3366ff' : '#ff3366';
        this.setAttribute('fill', newFill);
    });
</script>

SVG vs. Canvas

Feature SVG Canvas
Resolution Vector-based, resolution-independent Pixel-based, resolution-dependent
DOM Integration Part of the DOM, each element is accessible Single HTML element, content not in DOM
Performance Better for fewer, larger objects Better for many small objects or pixel manipulation
Event Handling Native event handling for each element Manual event handling required
Accessibility Better accessibility with native text and ARIA support Limited accessibility, requires additional work
Animation CSS, SMIL, or JavaScript JavaScript only
Best Use Cases Icons, illustrations, infographics, interactive diagrams Games, complex animations, image manipulation, data visualization

When to Use SVG vs. Canvas

Use SVG when:

  • You need resolution independence
  • You have a smaller number of larger objects
  • You need accessibility
  • You need to interact with individual elements
  • You're creating icons, logos, or illustrations

Use Canvas when:

  • You need to render many objects
  • You need pixel-level manipulation
  • You're creating games or complex animations
  • You need to process or generate images
  • Performance is critical

Responsive Images

HTML5 provides several features for implementing responsive images that adapt to different screen sizes and device capabilities.

The picture Element

The <picture> element allows you to specify multiple sources for an image based on media queries or image format support:

<!-- Different images based on viewport width -->
<picture>
    <source srcset="small.jpg" media="(max-width: 600px)">
    <source srcset="medium.jpg" media="(max-width: 1200px)">
    <source srcset="large.jpg">
    <img src="fallback.jpg" alt="Responsive image example">
</picture>

<!-- Different image formats based on browser support -->
<picture>
    <source srcset="image.webp" type="image/webp">
    <source srcset="image.avif" type="image/avif">
    <img src="image.jpg" alt="Image with format fallbacks">
</picture>

The srcset and sizes Attributes

The srcset and sizes attributes allow the browser to choose the most appropriate image based on the device's characteristics:

<!-- Different resolutions based on device pixel ratio -->
<img src="image-1x.jpg" 
     srcset="image-1x.jpg 1x, image-2x.jpg 2x, image-3x.jpg 3x" 
     alt="Image with different resolutions">

<!-- Different image sizes based on viewport width -->
<img src="small.jpg" 
     srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1600w" 
     sizes="(max-width: 600px) 400px, (max-width: 1200px) 800px, 1600px" 
     alt="Image with different sizes">

In the second example:

  • srcset defines the available images and their widths (400w, 800w, 1600w)
  • sizes tells the browser what size the image will be displayed at in different scenarios
  • The browser uses this information to download the most appropriate image

Loading Attribute

The loading attribute allows you to defer loading of off-screen images:

<!-- Lazy loading -->
<img src="image.jpg" alt="Lazy loaded image" loading="lazy">

<!-- Eager loading (default) -->
<img src="critical-image.jpg" alt="Critical image" loading="eager">

Decoding Attribute

The decoding attribute provides a hint to the browser about how to decode the image:

<!-- Asynchronous decoding -->
<img src="image.jpg" alt="Async decoded image" decoding="async">

<!-- Synchronous decoding -->
<img src="critical-image.jpg" alt="Sync decoded image" decoding="sync">

<!-- Auto decoding (browser decides) -->
<img src="another-image.jpg" alt="Auto decoded image" decoding="auto">

CSS Background Images

CSS also provides options for responsive background images:

<style>
    .hero {
        width: 100%;
        height: 50vh;
        background-image: url('large.jpg');
        background-size: cover;
        background-position: center;
    }
    
    /* Responsive background images */
    @media (max-width: 1200px) {
        .hero {
            background-image: url('medium.jpg');
        }
    }
    
    @media (max-width: 600px) {
        .hero {
            background-image: url('small.jpg');
            height: 30vh;
        }
    }
    
    /* Image-set for background images */
    .high-res-bg {
        background-image: url('image-1x.jpg');
        background-image: -webkit-image-set(
            url('image-1x.jpg') 1x,
            url('image-2x.jpg') 2x,
            url('image-3x.jpg') 3x
        );
        background-image: image-set(
            url('image-1x.jpg') 1x,
            url('image-2x.jpg') 2x,
            url('image-3x.jpg') 3x
        );
    }
</style>

<div class="hero"></div>
<div class="high-res-bg"></div>

Responsive Images Best Practices

  • Always include width and height attributes on images to prevent layout shifts
  • Use modern image formats like WebP or AVIF with appropriate fallbacks
  • Optimize images for each size to reduce file size
  • Use lazy loading for images below the fold
  • Consider art direction (different crops or compositions) for different screen sizes
  • Test on various devices and screen sizes

Embedding External Content

HTML provides several ways to embed external content from other sources.

iframe Element

The <iframe> element allows you to embed another HTML document within your page:

<!-- Basic iframe -->
<iframe src="https://example.com" width="600" height="400">
    Your browser does not support iframes.
</iframe>

<!-- YouTube video embed -->
<iframe 
    width="560" 
    height="315" 
    src="https://www.youtube.com/embed/VIDEO_ID" 
    title="YouTube video player" 
    frameborder="0" 
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
    allowfullscreen>
</iframe>

<!-- Google Maps embed -->
<iframe 
    width="600" 
    height="450" 
    src="https://www.google.com/maps/embed?pb=!1m18!1m12!..." 
    style="border:0;" 
    allowfullscreen="" 
    loading="lazy" 
    referrerpolicy="no-referrer-when-downgrade">
</iframe>

iframe Security

iframes can pose security risks, so it's important to use security attributes:

<iframe 
    src="https://example.com" 
    width="600" 
    height="400"
    sandbox="allow-scripts allow-same-origin allow-forms"
    referrerpolicy="no-referrer"
    loading="lazy"
></iframe>

The sandbox attribute restricts the capabilities of the iframe. Values include:

  • allow-scripts: Allows JavaScript execution
  • allow-same-origin: Allows the content to be treated as same-origin
  • allow-forms: Allows form submission
  • allow-popups: Allows popups
  • allow-top-navigation: Allows navigation of the top-level browsing context

embed Element

The <embed> element is used to embed external content that requires a plugin:

<!-- PDF embed -->
<embed 
    src="document.pdf" 
    type="application/pdf"
    width="600" 
    height="400">

<!-- Flash embed (legacy) -->
<embed 
    src="animation.swf" 
    type="application/x-shockwave-flash"
    width="600" 
    height="400">

object Element

The <object> element is similar to <embed> but allows for fallback content:

<!-- PDF object -->
<object 
    data="document.pdf" 
    type="application/pdf"
    width="600" 
    height="400">
    <p>Your browser doesn't support PDF viewing. <a href="document.pdf">Download the PDF</a> instead.</p>
</object>

<!-- SVG object -->
<object 
    data="image.svg" 
    type="image/svg+xml"
    width="300" 
    height="200">
    <img src="fallback.png" alt="Fallback image">
</object>

Embedding Best Practices

  • Always use HTTPS URLs for embedded content
  • Apply appropriate sandbox restrictions to iframes
  • Use lazy loading for embedded content below the fold
  • Provide fallback content for browsers that don't support the embedded content
  • Consider the performance impact of embedded content
  • Be aware of privacy implications when embedding third-party content

Summary

In this lesson, we've explored HTML media and graphics capabilities that allow you to create rich, interactive web content:

  • The <audio> element for embedding sound content with controls and JavaScript API

These technologies provide powerful tools for creating engaging web experiences without relying on third-party plugins. By understanding when and how to use each technology, you can create optimized, accessible, and interactive media experiences for your users.

Remember these key points:

  • Choose the right technology for your specific needs (SVG vs. Canvas, native controls vs. custom)
  • Optimize for performance with responsive techniques and appropriate file formats
  • Ensure accessibility with captions, transcripts, and ARIA attributes
  • Provide fallbacks for browsers that don't support certain features
  • Consider security implications when embedding external content
