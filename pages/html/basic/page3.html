HTML Elements & Tags

Understanding HTML Elements

HTML elements are the building blocks of web pages. They define the structure and content that users see in their browsers. An HTML element typically consists of a start tag, content, and an end tag.

Element Structure

<tagname>Content goes here...</tagname>

Where:

  • <tagname> is the opening tag
  • "Content goes here..." is the content
  • </tagname> is the closing tag

For example, a paragraph element looks like this:

<p>This is a paragraph of text.</p>

The browser uses these tags to determine how to display the content. The tags themselves are not displayed.

Types of HTML Elements

HTML elements can be categorized in several ways. Let's explore some of the most important distinctions:

Block-Level vs. Inline Elements

Block-Level Elements

Block-level elements start on a new line and take up the full width available. They create a "block" in the flow of the document.

Examples: <div>, <h1> to <h6>, <p>, <ul>, <li>

Inline Elements

Inline elements do not start on a new line and only take up as much width as necessary. They flow within the text.

Examples: <span>, <a>, <strong>, <em>, <img>

Here's a visual example of how block and inline elements behave:

<p>This is a block-level paragraph element.</p>
<p>This is another paragraph with <strong>inline</strong> elements inside it.</p>

In this example, each <p> element starts on a new line (block-level), while the <strong> element flows within the text of the paragraph (inline).

Empty Elements vs. Container Elements

Container Elements

Most HTML elements are container elements, meaning they have both opening and closing tags with content in between.

Examples: <p></p>, <div></div>, <span></span>

Empty Elements

Empty elements (also called self-closing elements) don't have closing tags because they don't contain content.

Examples: <img>, <br>, <hr>, <input>

Empty elements are written in one of two ways:

<!-- HTML5 style -->
<img src="image.jpg" alt="Description">

<!-- XHTML style (also valid in HTML5) -->
<img src="image.jpg" alt="Description" />

Both styles are valid in HTML5, but the first style is more common.

Essential HTML Elements

Let's explore some of the most commonly used HTML elements that every web developer should know:

Headings

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important):

<h1>This is a Level 1 Heading</h1>
<h2>This is a Level 2 Heading</h2>
<h3>This is a Level 3 Heading</h3>
<h4>This is a Level 4 Heading</h4>
<h5>This is a Level 5 Heading</h5>
<h6>This is a Level 6 Heading</h6>

Heading Best Practices

  • Use only one <h1> per page (typically for the main title)
  • Don't skip heading levels (e.g., don't go from <h2> to <h4>)
  • Use headings to create a logical document outline

Paragraphs

The <p> element represents a paragraph of text:

<p>This is a paragraph of text. Paragraphs are block-level elements that automatically add space before and after the content.</p>
<p>This is another paragraph. Notice how it starts on a new line.</p>

Divisions

The <div> element is a generic container used to group content for styling and layout purposes:

<div class="container">
    <h2>Section Title</h2>
    <p>This content is grouped together in a div.</p>
</div>

Divs have no semantic meaning by themselves but are extremely useful for organizing and styling content.

Spans

The <span> element is an inline container used to style or manipulate a specific portion of text:

<p>This paragraph has <span style="color: blue;">some blue text</span> in it.</p>

Line Breaks

The <br> element creates a line break within text:

<p>This text will be on one line.<br>This text will be on a new line.</p>

Unlike paragraphs, line breaks don't add extra spacing.

Horizontal Rules

The <hr> element creates a horizontal line, often used to separate content sections:

<h2>Section 1</h2>
<p>Content for section 1...</p>
<hr>
<h2>Section 2</h2>
<p>Content for section 2...</p>

Text Formatting Elements

HTML provides several elements for formatting text:

<p><strong>Bold text</strong> indicates importance.</p>
<p><em>Italic text</em> adds emphasis.</p>
<p><u>Underlined text</u> (use sparingly, as it can be confused with links).</p>
<p><mark>Highlighted text</mark> draws attention.</p>
<p><small>Smaller text</small> for fine print.</p>
<p><del>Deleted text</del> shows removed content.</p>
<p><ins>Inserted text</ins> shows added content.</p>
<p>H<sub>2</sub>O (subscript) and 10<sup>2</sup> (superscript).</p>

Semantic vs. Presentational

Modern HTML emphasizes semantic elements (which describe their meaning) over presentational ones (which only describe appearance).

For example, use <strong> (semantic: important text) instead of <b> (presentational: bold text).

Nested Elements

HTML elements can be nested inside other elements, creating a hierarchical structure. This is a fundamental concept in HTML.

<div class="card">
    <h2>Card Title</h2>
    <p>This paragraph is inside the card. It contains <strong>bold text</strong> and a <a href="https://example.com">link</a>.</p>
</div>

In this example:

  • The <h2> and <p> elements are nested inside the <div>
  • The <strong> and <a> elements are nested inside the <p>

This nesting creates a parent-child relationship between elements. In the example above:

  • The <div> is the parent of <h2> and <p>
  • The <p> is the parent of <strong> and <a>
  • The <h2>, <p>, <strong>, and <a> are all descendants of <div>

Proper Nesting

Elements must be properly nested. This means that if element B starts inside element A, then element B must end before element A ends.

Correct: <a><strong>Text</strong></a>

Incorrect: <a><strong>Text</a></strong>

Comments in HTML

HTML comments allow you to add notes to your code that are not displayed in the browser. They're useful for documenting your code or temporarily disabling parts of it.

<!-- This is an HTML comment -->

<!-- 
    This is a multi-line comment.
    It can span multiple lines.
-->

<!-- This section needs to be updated
<div class="outdated">
    <p>This content is temporarily disabled.</p>
</div>
-->

Comments are visible in the source code but not on the rendered page. They're invaluable for collaboration and maintenance.

HTML Element Reference

There are over 100 HTML elements available, each with its specific purpose. Here's a quick reference of some commonly used elements grouped by category:

Document Structure

  • <html> - Root element
  • <head> - Container for metadata
  • <body> - Container for visible content
  • <header> - Page or section header
  • <footer> - Page or section footer
  • <main> - Main content area
  • <section> - Thematic grouping of content
  • <article> - Self-contained composition
  • <aside> - Related but tangential content
  • <nav> - Navigation links

Text Content

  • <h1> to <h6> - Headings
  • <p> - Paragraph
  • <ul> - Unordered list
  • <ol> - Ordered list
  • <li> - List item
  • <dl> - Description list
  • <dt> - Term in a description list
  • <dd> - Description in a description list
  • <blockquote> - Block quotation
  • <figure> - Self-contained content with optional caption
  • <figcaption> - Caption for a figure

Inline Text

  • <a> - Hyperlink
  • <strong> - Important text
  • <em> - Emphasized text
  • <span> - Generic inline container
  • <code> - Computer code
  • <q> - Inline quotation
  • <abbr> - Abbreviation
  • <cite> - Citation
  • <time> - Time

Multimedia

  • <img> - Image
  • <audio> - Audio content
  • <video> - Video content
  • <source> - Media resource
  • <track> - Text track for media
  • <canvas> - Drawing area
  • <svg> - Scalable Vector Graphics

Forms

  • <form> - Form container
  • <input> - Input control
  • <textarea> - Multiline text input
  • <button> - Clickable button
  • <select> - Dropdown list
  • <option> - Option in a select element
  • <label> - Label for a form control
  • <fieldset> - Group of form controls
  • <legend> - Caption for a fieldset

We'll explore many of these elements in more detail in the upcoming lessons.

Practice Exercise

Let's put what we've learned into practice by creating a simple HTML document that uses various elements:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>HTML Elements Practice</title>
</head>
<body>
    <h1>My HTML Elements Practice</h1>
    
    <h2>About Me</h2>
    <p>Hello! I'm learning <strong>HTML</strong> and finding it <em>very</em> interesting.</p>
    
    <h2>My Favorite Foods</h2>
    <ul>
        <li>Pizza</li>
        <li>Ice Cream</li>
        <li>Sushi</li>
    </ul>
    
    <h2>My Daily Schedule</h2>
    <ol>
        <li>Wake up</li>
        <li>Eat breakfast</li>
        <li>Study HTML</li>
        <li>Practice coding</li>
        <li>Sleep</li>
    </ol>
    
    <hr>
    
    <h2>Contact Me</h2>
    <p>Email me at: <a href="mailto:example@example.com">example@example.com</a></p>
</body>
</html>

Try It Yourself

Create the HTML document above and view it in your browser. Then try to:

  1. Add another section with a heading
  2. Include a blockquote with your favorite quote
  3. Add an image (you can use a placeholder like <img src="https://via.placeholder.com/150" alt="Placeholder">)
  4. Create a table with at least 2 rows and 2 columns

Summary

In this lesson, we've explored HTML elements and tags, the building blocks of web pages:

  • HTML elements consist of opening tags, content, and closing tags
  • Elements can be block-level (start on a new line) or inline (flow within text)
  • Some elements are empty (self-closing) and don't have closing tags
  • Elements can be nested to create a hierarchical structure
  • HTML provides elements for headings, paragraphs, text formatting, lists, and more
  • Comments allow you to add notes to your code that aren't displayed in the browser

Understanding HTML elements is fundamental to web development. In the next lesson, we'll dive deeper into text formatting elements and how to structure content effectively.
