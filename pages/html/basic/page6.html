Images & Multimedia in HTML

Enhancing Web Pages with Visual Content

While text forms the foundation of web content, visual elements like images, videos, and audio bring web pages to life. Multimedia content can convey information more effectively, create emotional connections, and enhance the overall user experience.

HTML5 provides robust support for multimedia, making it easier than ever to incorporate various types of media into your web pages. In this lesson, we'll explore how to add images, videos, audio, and other multimedia elements to your websites.

Evolution of Multimedia on the Web

In the early days of the web, adding multimedia was complex and often required plugins like Flash. HTML5 introduced native support for audio and video, eliminating the need for third-party plugins and making multimedia more accessible and secure.

Adding Images to Web Pages

Images are the most common type of multimedia content on the web. They can be used for photos, illustrations, icons, logos, backgrounds, and more.

The Image Element

Images are added to web pages using the <img> element. This is an empty (self-closing) element that doesn't have a closing tag.

<img src="image.jpg" alt="Description of the image">

The <img> element has two essential attributes:

  • src (source): Specifies the path to the image file
  • alt (alternative text): Provides a text description of the image

Accessibility Tip

The alt attribute is crucial for accessibility. It provides a text alternative for users who cannot see the image, such as those using screen readers or those with images disabled. It also displays if the image fails to load.

Good alt text should be concise but descriptive, conveying the purpose and content of the image. For decorative images that don't add content, use an empty alt attribute (alt="") to indicate that the image can be safely ignored by screen readers.

Image File Paths

The src attribute can contain different types of file paths:

<!-- Absolute URL (external image) -->
<img src="https://example.com/images/photo.jpg" alt="Example photo">

<!-- Relative URL (image in the same directory) -->
<img src="logo.png" alt="Company logo">

<!-- Relative URL (image in a subdirectory) -->
<img src="images/banner.jpg" alt="Banner image">

<!-- Relative URL (image in a parent directory) -->
<img src="../icons/icon.svg" alt="Icon">

Image Formats

Several image formats are supported by web browsers, each with its own advantages:

JPEG (.jpg, .jpeg)

Best for photographs and complex images with many colors. Supports lossy compression but doesn't support transparency.

PNG (.png)

Best for images with transparency, simple graphics, and text. Lossless compression but larger file sizes than JPEG for photographs.

GIF (.gif)

Supports simple animations and transparency. Limited to 256 colors, so not suitable for photographs.

SVG (.svg)

Vector format that scales without losing quality. Ideal for logos, icons, and simple illustrations. Can be styled with CSS and manipulated with JavaScript.

WebP (.webp)

Modern format developed by Google that offers better compression than JPEG and PNG with support for transparency and animation. Not supported by all browsers.

AVIF (.avif)

Newest format with excellent compression and quality. Limited browser support but growing.

Choosing the Right Format

As a general rule:

  • Use JPEG for photographs
  • Use PNG for images with transparency or simple graphics
  • Use SVG for icons, logos, and illustrations
  • Use WebP when you can provide fallbacks for older browsers

Additional Image Attributes

Besides src and alt, the <img> element supports several other attributes:

<!-- Setting width and height -->
<img src="image.jpg" alt="Description" width="500" height="300">

<!-- Adding a title (tooltip) -->
<img src="image.jpg" alt="Description" title="Additional information about the image">

<!-- Specifying loading behavior -->
<img src="image.jpg" alt="Description" loading="lazy">

<!-- Controlling image decoding -->
<img src="image.jpg" alt="Description" decoding="async">

Performance Tip

Always specify the width and height attributes for images. This helps the browser allocate space for the image before it loads, reducing layout shifts as the page loads.

The loading="lazy" attribute defers loading images until they're near the viewport, improving page load performance.

Responsive Images

Responsive images adapt to different screen sizes and device capabilities, ensuring optimal display and performance across devices.

Basic Responsive Images with CSS

The simplest way to make images responsive is with CSS:

img {
    max-width: 100%;
    height: auto;
}

This ensures that images never exceed their container's width and maintain their aspect ratio.

The Picture Element

For more advanced responsive image solutions, HTML5 introduced the <picture> element. It allows you to specify different image sources for different screen sizes, pixel densities, or image formats.

<picture>
    <!-- For small screens -->
    <source media="(max-width: 600px)" srcset="small-image.jpg">
    
    <!-- For medium screens -->
    <source media="(max-width: 1200px)" srcset="medium-image.jpg">
    
    <!-- For browsers that don't support WebP -->
    <source srcset="image.jpg">
    
    <!-- Fallback for browsers that don't support the picture element -->
    <img src="image.jpg" alt="Description">
</picture>

The browser will use the first <source> element that matches the current conditions. If none match, or if the browser doesn't support the <picture> element, it will use the <img> element as a fallback.

The Srcset Attribute

The srcset attribute allows you to provide multiple image sources with different resolutions, letting the browser choose the most appropriate one based on the device's pixel density and viewport size.

<!-- Different resolutions for different pixel densities -->
<img src="image.jpg" alt="Description" 
     srcset="image.jpg 1x, image@2x.jpg 2x, image@3x.jpg 3x">

<!-- Different sizes for different viewport widths -->
<img src="image-large.jpg" alt="Description" 
     srcset="image-small.jpg 500w, image-medium.jpg 1000w, image-large.jpg 1500w"
     sizes="(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 33vw">

In the second example, the sizes attribute tells the browser what size the image will be displayed at different viewport widths, helping it choose the most appropriate source from the srcset.

Image Maps

Image maps allow you to create clickable areas on an image, each linking to a different destination. They're useful for creating interactive diagrams, maps, or complex navigation elements.

<img src="map.jpg" alt="Office floor plan" usemap="#office-map">

<map name="office-map">
    <!-- Rectangular area -->
    <area shape="rect" coords="34,44,270,350" alt="Conference Room" href="conference.html">
    
    <!-- Circular area -->
    <area shape="circle" coords="337,300,44" alt="Lobby" href="lobby.html">
    
    <!-- Polygonal area -->
    <area shape="poly" coords="140,121,181,116,204,160,204,222,191,270,140,329,85,271,78,234,78,182" 
          alt="Office Space" href="office.html">
</map>

The usemap attribute of the <img> element references the name attribute of the <map> element. Each <area> element defines a clickable region with these attributes:

  • shape: The shape of the area (rect, circle, poly)
  • coords: The coordinates that define the area
  • alt: Alternative text for accessibility
  • href: The URL to navigate to when the area is clicked

Figure and Figcaption

The <figure> and <figcaption> elements provide a semantic way to associate captions with images or other content.

<figure>
    <img src="chart.jpg" alt="Sales chart showing growth in Q3">
    <figcaption>Figure 1: Sales increased by 25% in Q3 2025.</figcaption>
</figure>

The <figure> element represents self-contained content that is referenced from the main content, such as illustrations, diagrams, photos, or code listings. The <figcaption> element provides a caption or legend for the figure.

Using these elements improves the semantic structure of your document and helps screen readers understand the relationship between the image and its caption.

Adding Video to Web Pages

HTML5 introduced the <video> element for embedding videos without requiring plugins like Flash.

Basic Video Element

<video src="video.mp4" controls width="640" height="360">
    Your browser does not support the video element.
</video>

The content between the opening and closing tags is displayed if the browser doesn't support the <video> element.

Video Attributes

The <video> element supports several attributes to control its behavior:

<video src="video.mp4" controls width="640" height="360" 
       autoplay muted loop poster="video-thumbnail.jpg">
    Your browser does not support the video element.
</video>
  • controls: Displays video controls (play, pause, volume, etc.)
  • autoplay: Starts playing the video automatically (many browsers require muted for autoplay)
  • muted: Mutes the audio
  • loop: Plays the video repeatedly
  • poster: Specifies an image to display before the video plays
  • preload: Hints how much of the video to preload (none, metadata, auto)
  • playsinline: Plays the video inline on iOS devices instead of fullscreen

Multiple Sources

Different browsers support different video formats. To ensure maximum compatibility, you can provide multiple sources:

<video controls width="640" height="360">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    <p>Your browser does not support the video element. <a href="video.mp4">Download the video</a>.</p>
</video>

The browser will use the first source it supports. If none are supported, it will display the fallback content.

Video Formats

Common video formats for the web include:

  • MP4 (H.264): Widely supported across all browsers
  • WebM (VP8/VP9): Open format with better compression than MP4
  • Ogg (Theora): Open format, less common

Adding Audio to Web Pages

Similar to video, HTML5 introduced the <audio> element for embedding audio content.

Basic Audio Element

<audio src="audio.mp3" controls>
    Your browser does not support the audio element.
</audio>

Audio Attributes

The <audio> element supports many of the same attributes as the <video> element:

<audio src="audio.mp3" controls autoplay muted loop preload="auto">
    Your browser does not support the audio element.
</audio>

Multiple Sources

As with video, you can provide multiple audio sources for better compatibility:

<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    <p>Your browser does not support the audio element. <a href="audio.mp3">Download the audio</a>.</p>
</audio>

Audio Formats

Common audio formats for the web include:

  • MP3: Widely supported, good compression
  • AAC: Better quality than MP3 at similar bit rates
  • Ogg Vorbis: Open format, good quality
  • WAV: Uncompressed, high quality but large file size

Embedding Content

The <iframe> element allows you to embed content from other sources, such as maps, videos from platforms like YouTube, or other web pages.

Embedding YouTube Videos

<iframe width="560" height="315" 
        src="https://www.youtube.com/embed/VIDEO_ID" 
        title="YouTube video player" 
        frameborder="0" 
        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
        allowfullscreen></iframe>

Embedding Google Maps

<iframe 
        src="https://www.google.com/maps/embed?pb=!1m18!1m12!..." 
        width="600" height="450" 
        style="border:0;" 
        allowfullscreen="" 
        loading="lazy"></iframe>

Embedding Other Web Pages

<iframe src="https://example.com" width="800" height="600">
    <p>Your browser does not support iframes.</p>
</iframe>

Security Considerations

Embedding content from other sources can pose security risks. Always use the sandbox attribute to restrict the capabilities of the embedded content, and only embed content from trusted sources.

<iframe src="https://example.com" sandbox="allow-scripts allow-same-origin"></iframe>

SVG Graphics

Scalable Vector Graphics (SVG) is an XML-based format for vector graphics. Unlike raster formats (JPEG, PNG), SVG images can scale to any size without losing quality, making them ideal for responsive design.

Embedding SVG

There are several ways to include SVG in your web pages:

<!-- As an image -->
<img src="icon.svg" alt="Icon">

<!-- Inline SVG -->
<svg width="100" height="100" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="40" stroke="black" stroke-width="2" fill="red" />
</svg>

<!-- Using an object -->
<object data="graphic.svg" type="image/svg+xml">
    <img src="graphic.png" alt="Fallback image">
</object>

Inline SVG offers the most flexibility, allowing you to style the SVG with CSS and manipulate it with JavaScript.

Basic SVG Shapes

SVG provides several basic shapes:

<!-- Rectangle -->
<svg width="200" height="100">
    <rect width="150" height="75" fill="blue" />
</svg>

<!-- Circle -->
<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" fill="green" />
</svg>

<!-- Ellipse -->
<svg width="200" height="100">
    <ellipse cx="100" cy="50" rx="80" ry="40" fill="yellow" />
</svg>

<!-- Line -->
<svg width="200" height="100">
    <line x1="0" y1="0" x2="200" y2="100" stroke="red" stroke-width="2" />
</svg>

<!-- Polygon -->
<svg width="200" height="200">
    <polygon points="100,10 40,180 190,60 10,60 160,180" fill="purple" />
</svg>

SVG also supports more complex shapes, text, gradients, filters, and animations.

Multimedia Accessibility

Making multimedia content accessible is essential for users with disabilities. Here are some key considerations:

Alternative Text for Images

Always provide descriptive alt text for images that convey information:

<img src="chart.jpg" alt="Bar chart showing sales increasing by 25% in Q3 2025">

Captions and Transcripts

Provide captions for videos and transcripts for audio content:

<video controls>
    <source src="video.mp4" type="video/mp4">
    <track src="captions.vtt" kind="subtitles" srclang="en" label="English">
</video>

The <track> element is used to specify text tracks (like subtitles or captions) for media elements.

Audio Descriptions

For videos with important visual information, provide audio descriptions that describe what's happening on screen for users who can't see the video.

Controls

Always include controls for audio and video content so users can pause, adjust volume, or stop playback as needed.

No Auto-Play

Avoid auto-playing media with sound, as it can be disruptive and confusing, especially for screen reader users.

Practice Exercise: Adding Multimedia

Let's put what we've learned into practice by creating a web page with various multimedia elements:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Multimedia Example</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            line-height: 1.6;
            max-width: 800px;
            margin: 0 auto;
            padding: 20px;
        }
        
        img {
            max-width: 100%;
            height: auto;
        }
        
        figure {
            margin: 0;
            padding: 10px;
            border: 1px solid #ddd;
            border-radius: 5px;
        }
        
        figcaption {
            font-style: italic;
            text-align: center;
            margin-top: 10px;
        }
        
        .gallery {
            display: grid;
            grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
            gap: 20px;
            margin: 20px 0;
        }
        
        video, audio {
            width: 100%;
            margin: 20px 0;
        }
    </style>
</head>
<body>
    <h1>Multimedia Example</h1>
    
    <h2>Featured Image</h2>
    <figure>
        <img src="https://via.placeholder.com/800x400" alt="A placeholder image for demonstration">
        <figcaption>Figure 1: This is a placeholder image used for demonstration purposes.</figcaption>
    </figure>
    
    <h2>Image Gallery</h2>
    <div class="gallery">
        <img src="https://via.placeholder.com/300x200?text=Image+1" alt="Gallery image 1">
        <img src="https://via.placeholder.com/300x200?text=Image+2" alt="Gallery image 2">
        <img src="https://via.placeholder.com/300x200?text=Image+3" alt="Gallery image 3">
        <img src="https://via.placeholder.com/300x200?text=Image+4" alt="Gallery image 4">
    </div>
    
    <h2>Responsive Image Example</h2>
    <picture>
        <source media="(max-width: 600px)" srcset="https://via.placeholder.com/400x200?text=Small">
        <source media="(max-width: 1000px)" srcset="https://via.placeholder.com/600x300?text=Medium">
        <img src="https://via.placeholder.com/800x400?text=Large" alt="Responsive image example">
    </picture>
    
    <h2>Video Example</h2>
    <video controls poster="https://via.placeholder.com/800x450?text=Video+Thumbnail">
        <source src="https://www.w3schools.com/html/mov_bbb.mp4" type="video/mp4">
        Your browser does not support the video element.
    </video>
    
    <h2>Audio Example</h2>
    <audio controls>
        <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
        Your browser does not support the audio element.
    </audio>
    
    <h2>SVG Example</h2>
    <svg width="200" height="200" viewBox="0 0 200 200">
        <rect x="50" y="50" width="100" height="100" fill="#3a86ff" />
        <circle cx="100" cy="100" r="40" fill="#ff006e" />
        <text x="100" y="180" text-anchor="middle" fill="#000">SVG Example</text>

        width="100%"

</body>
</html>

Summary

In this lesson, we've explored images and multimedia in HTML:

  • Images are added using the <img> element with src and alt attributes
  • Responsive images can be created using CSS, the <picture> element, or the srcset attribute
  • The <figure> and <figcaption> elements provide semantic structure for images with captions
  • Videos are embedded using the <video> element with various attributes to control playback
  • Audio content is added with the <audio> element
  • The <iframe> element allows embedding content from other sources
  • SVG provides scalable vector graphics that can be embedded directly or as external files
  • Accessibility considerations include alternative text, captions, and transcripts

Multimedia content enhances the user experience and can convey information more effectively than text alone. By following best practices for performance and accessibility, you can create engaging, inclusive web pages that work well for all users.
