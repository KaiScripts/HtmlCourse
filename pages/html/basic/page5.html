Links & Navigation in HTML

The Power of Hyperlinks

Hyperlinks are what make the web a "web." They connect pages and resources together, allowing users to navigate between them. Without links, each web page would be an isolated island of information.

The ability to link documents together is what transformed the internet from a collection of separate files into the interconnected World Wide Web we know today. In fact, HTML stands for HyperText Markup Language, and the "HyperText" part refers specifically to text that contains links to other content.

Historical Note

The concept of hypertext was first proposed by Ted Nelson in the 1960s, but it was Tim Berners-Lee who implemented it in the early 1990s when he created the World Wide Web. The ability to link documents together was revolutionary and fundamentally changed how we access and share information.

The Anchor Element

In HTML, links are created using the anchor element <a>. The most important attribute of the anchor element is href (hypertext reference), which specifies the destination of the link.

<a href="https://www.example.com">Visit Example.com</a>

This creates a link with the text "Visit Example.com" that, when clicked, takes the user to the URL specified in the href attribute.

Link Text

The content between the opening and closing <a> tags is what users see and click on. This can be text, images, or even other HTML elements (except for other anchor elements).

<!-- Text link -->
<a href="about.html">About Us</a>

<!-- Image link -->
<a href="product.html"><img src="product-image.jpg" alt="Our Product"></a>

<!-- Button-like link -->
<a href="contact.html"><div class="button">Contact Us</div></a>

Accessibility Tip

Always make sure your link text is descriptive and makes sense out of context. Screen reader users often navigate by jumping from link to link, so link text like "click here" or "read more" doesn't provide enough information about where the link leads.

Good: <a href="pricing.html">View our pricing plans</a>

Poor: <a href="pricing.html">Click here</a>

Types of Links

HTML supports several types of links, each with its own purpose and syntax:

Absolute URLs

Absolute URLs provide the complete path to a resource, including the protocol (http, https), domain name, and path.

<a href="https://www.example.com/about/team.html">Our Team</a>

Use absolute URLs when linking to pages on different websites.

Relative URLs

Relative URLs specify the path to a resource relative to the current document. They don't include the protocol or domain name.

<!-- Link to a page in the same directory -->
<a href="contact.html">Contact Us</a>

<!-- Link to a page in a subdirectory -->
<a href="about/team.html">Our Team</a>

<!-- Link to a page in a parent directory -->
<a href="../products.html">Products</a>

<!-- Link to a page two directories up -->
<a href="../../index.html">Home</a>

Use relative URLs when linking to pages within your own website. They make your site more portable and easier to maintain.

Root-Relative URLs

Root-relative URLs specify the path from the root directory of the website. They start with a forward slash.

<a href="/about/team.html">Our Team</a>

This link points to the "team.html" page in the "about" directory from the root of the website, regardless of where the current page is located.

Fragment Identifiers (Anchor Links)

Fragment identifiers link to a specific section within a page. They use the # symbol followed by the ID of the target element.

<!-- Link to a section on the same page -->
<a href="#section2">Go to Section 2</a>

<!-- The target section -->
<h2 id="section2">Section 2</h2>

You can also combine fragment identifiers with URLs to link to a specific section of another page:

<a href="about.html#team">Our Team</a>

Email Links

Email links open the user's default email client with a new message addressed to the specified email address. They use the mailto: protocol.

<a href="mailto:contact@example.com">Email Us</a>

You can also include subject, body text, and other email fields:

<a href="mailto:contact@example.com?subject=Website%20Inquiry&body=Hello%2C%20I%20have%20a%20question%20about...">Email Us</a>

Note that spaces and special characters in the subject and body must be URL-encoded (e.g., spaces become %20).

Phone Links

Phone links initiate a call when clicked on mobile devices. They use the tel: protocol.

<a href="tel:+1234567890">Call Us</a>

Download Links

You can use the download attribute to specify that a link should download a file rather than navigate to it.

<a href="files/brochure.pdf" download>Download Brochure</a>

You can also specify a different filename for the downloaded file:

<a href="files/brochure.pdf" download="company-brochure.pdf">Download Brochure</a>

Link Attributes

The anchor element supports several attributes that modify its behavior:

Target Attribute

The target attribute specifies where to open the linked document. The most common values are:

<!-- Open in the same tab/window (default) -->
<a href="page.html" target="_self">Same Tab</a>

<!-- Open in a new tab/window -->
<a href="page.html" target="_blank">New Tab</a>

<!-- Open in the parent frame (used with framesets) -->
<a href="page.html" target="_parent">Parent Frame</a>

<!-- Open in the full body of the window (used with framesets) -->
<a href="page.html" target="_top">Full Window</a>

Security Tip

When using target="_blank", it's recommended to also include rel="noopener noreferrer" to prevent potential security vulnerabilities:

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Example</a>

Rel Attribute

The rel attribute specifies the relationship between the current document and the linked document. Common values include:

<!-- Indicates that the link is to an alternate version of the page -->
<a href="mobile-version.html" rel="alternate">Mobile Version</a>

<!-- Indicates that the link is to the author of the document -->
<a href="author.html" rel="author">About the Author</a>

<!-- Indicates that the link is to an external resource -->
<a href="https://example.com" rel="external">External Site</a>

<!-- Indicates that the link should not be followed by search engines -->
<a href="private.html" rel="nofollow">Private Page</a>

<!-- Prevents the new page from accessing the window.opener property -->
<a href="https://example.com" target="_blank" rel="noopener">Safe External Link</a>

Title Attribute

The title attribute provides additional information about the link. It typically appears as a tooltip when the user hovers over the link.

<a href="products.html" title="View our complete product catalog">Products</a>

Styling Links

Links have different states that can be styled using CSS:

/* Unvisited link */
a:link {
    color: #3a86ff;
    text-decoration: none;
}

/* Visited link */
a:visited {
    color: #8338ec;
}

/* Mouse over link */
a:hover {
    color: #ff006e;
    text-decoration: underline;
}

/* Selected link */
a:active {
    color: #ffbe0b;
}

These styles should be defined in this order (link, visited, hover, active) for proper behavior.

Accessibility Tip

Always ensure there's sufficient contrast between link text and regular text, and provide visual cues beyond just color (like underlines) to indicate links. This helps users with color vision deficiencies identify links.

Creating Navigation Menus

Navigation menus are collections of links that help users move around a website. They're typically implemented using lists:

Basic Navigation Menu

<nav>
    <ul>
        <li><a href="index.html">Home</a></li>
        <li><a href="about.html">About</a></li>
        <li><a href="services.html">Services</a></li>
        <li><a href="contact.html">Contact</a></li>
    </ul>
</nav>

The <nav> element is a semantic container for navigation links. Using an unordered list (<ul>) for navigation is a common practice that provides good structure and accessibility.

Dropdown Menu

Dropdown menus can be created using nested lists:

<nav>
    <ul>
        <li><a href="index.html">Home</a></li>
        <li>
            <a href="products.html">Products</a>
            <ul class="dropdown">
                <li><a href="products/category1.html">Category 1</a></li>
                <li><a href="products/category2.html">Category 2</a></li>
                <li><a href="products/category3.html">Category 3</a></li>
            </ul>
        </li>
        <li><a href="about.html">About</a></li>
        <li><a href="contact.html">Contact</a></li>
    </ul>
</nav>

The dropdown functionality would be implemented with CSS and possibly JavaScript.

Breadcrumb Navigation

Breadcrumb navigation shows the user's location in the website hierarchy:

<nav aria-label="Breadcrumb">
    <ol class="breadcrumb">
        <li><a href="index.html">Home</a></li>
        <li><a href="products.html">Products</a></li>
        <li aria-current="page">Product Name</li>
    </ol>
</nav>

Breadcrumbs typically use ordered lists (<ol>) since they represent a sequence of steps.

Skip Navigation Links

Skip navigation links allow keyboard users and screen reader users to bypass navigation menus and jump directly to the main content. They're an important accessibility feature.

<a href="#main-content" class="skip-link">Skip to main content</a>

<!-- Navigation menu here -->

<main id="main-content">
    <!-- Main content here -->
</main>

The skip link is typically the first focusable element on the page and is often visually hidden until it receives focus.

.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    background: #000;
    color: white;
    padding: 8px;
    z-index: 100;
}

.skip-link:focus {
    top: 0;
}

Link Best Practices

Follow these best practices to create effective and accessible links:

Use Descriptive Link Text

Link text should clearly indicate where the link will take the user.

  • Good: <a href="pricing.html">View our pricing plans</a>
  • Poor: <a href="pricing.html">Click here</a>

Keep Link Text Concise

Link text should be brief but descriptive. Avoid using entire sentences as link text.

Indicate External Links

Let users know when a link will take them to an external site. This can be done with an icon or explicit text.

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Example Website (External Link)</a>

Indicate File Downloads

When linking to a file that will be downloaded, indicate the file type and size if possible.

<a href="files/report.pdf" download>Download Annual Report (PDF, 2.5MB)</a>

Avoid "Click Here"

Don't use generic phrases like "click here" or "read more" as link text. These phrases provide no context about the link destination and are problematic for screen reader users.

Don't Open Too Many New Windows

Use target="_blank" sparingly. Opening too many new windows or tabs can be disorienting for users, especially those with cognitive disabilities.

Check for Broken Links

Regularly check your website for broken links. Broken links frustrate users and harm your site's credibility and search engine ranking.

Practice Exercise: Creating a Navigation System

Let's put what we've learned into practice by creating a simple website with a navigation system:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Website</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            line-height: 1.6;
            margin: 0;
            padding: 0;
        }
        
        .skip-link {
            position: absolute;
            top: -40px;
            left: 0;
            background: #000;
            color: white;
            padding: 8px;
            z-index: 100;
        }
        
        .skip-link:focus {
            top: 0;
        }
        
        header {
            background: #333;
            color: white;
            padding: 1rem;
        }
        
        nav ul {
            list-style: none;
            padding: 0;
            display: flex;
        }
        
        nav li {
            margin-right: 1rem;
        }
        
        nav a {
            color: white;
            text-decoration: none;
        }
        
        nav a:hover {
            text-decoration: underline;
        }
        
        main {
            padding: 1rem;
        }
        
        footer {
            background: #333;
            color: white;
            padding: 1rem;
            text-align: center;
        }
    </style>
</head>
<body>
    <a href="#main-content" class="skip-link">Skip to main content</a>
    
    <header>
        <h1>My Website</h1>
        <nav>
            <ul>
                <li><a href="index.html">Home</a></li>
                <li><a href="about.html">About</a></li>
                <li><a href="services.html">Services</a></li>
                <li><a href="contact.html">Contact</a></li>
            </ul>
        </nav>
    </header>
    
    <main id="main-content">
        <section>
            <h2>Welcome to My Website</h2>
            <p>This is the home page of my website. Here you'll find information about:</p>
            <ul>
                <li><a href="#section1">Section 1</a></li>
                <li><a href="#section2">Section 2</a></li>
                <li><a href="#section3">Section 3</a></li>
            </ul>
        </section>
        
        <section id="section1">
            <h2>Section 1</h2>
            <p>This is the content for section 1.</p>
            <p><a href="#main-content">Back to top</a></p>
        </section>
        
        <section id="section2">
            <h2>Section 2</h2>
            <p>This is the content for section 2.</p>
            <p><a href="#main-content">Back to top</a></p>
        </section>
        
        <section id="section3">
            <h2>Section 3</h2>
            <p>This is the content for section 3.</p>
            <p><a href="#main-content">Back to top</a></p>
        </section>
        
        <section>
            <h2>External Resources</h2>
            <ul>
                <li><a href="https://example.com" target="_blank" rel="noopener noreferrer">Example Website (External Link)</a></li>
                <li><a href="files/sample.pdf" download>Download Sample PDF</a></li>
                <li><a href="mailto:contact@example.com">Email Us</a></li>
            </ul>
        </section>
    </main>
    
    <footer>
        <p>© 2025 My Website. All rights reserved.</p>
        <nav>
            <ul>
                <li><a href="privacy.html">Privacy Policy</a></li>
                <li><a href="terms.html">Terms of Service</a></li>
                <li><a href="sitemap.html">Sitemap</a></li>
            </ul>
        </nav>
    </footer>
</body>
</html>

Try It Yourself

Create the HTML document above and view it in your browser. Then try to:

  1. Add a dropdown menu to one of the navigation items
  2. Create a breadcrumb navigation at the top of the main content
  3. Add a phone link to the external resources section
  4. Create a second page (e.g., about.html) and link to it from the navigation

Summary

In this lesson, we've explored links and navigation in HTML:

  • Links are created using the anchor element <a> with the href attribute
  • Different types of links include absolute URLs, relative URLs, fragment identifiers, email links, and phone links
  • Link attributes like target, rel, and title modify link behavior
  • Navigation menus are typically created using lists within a <nav> element
  • Skip navigation links improve accessibility by allowing users to bypass navigation menus
  • Best practices include using descriptive link text, indicating external links and file downloads, and avoiding generic phrases like "click here"

Links are what make the web a web, connecting pages and resources together. Well-designed navigation systems help users find what they're looking for and move around your website efficiently.
