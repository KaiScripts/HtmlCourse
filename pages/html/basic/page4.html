Text Formatting in HTML

The Importance of Text Formatting

Text is the primary way we communicate information on the web. Well-formatted text improves readability, emphasizes important points, and creates visual hierarchy that guides users through your content.

HTML provides a variety of elements specifically designed for text formatting. These elements help you structure your content in a way that's both visually appealing and semantically meaningful.

Semantic vs. Presentational

Modern HTML emphasizes semantic formatting (which describes the meaning or purpose of content) over presentational formatting (which only describes appearance).

For example, use <strong> (semantic: important text) instead of <b> (presentational: bold text).

Headings: Creating Structure

Headings are one of the most important text formatting elements. They create a hierarchical structure for your content and help users and search engines understand the organization of your page.

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important):

<h1>Main Page Title (H1)</h1>
<h2>Major Section Heading (H2)</h2>
<h3>Subsection Heading (H3)</h3>
<h4>Minor Heading (H4)</h4>
<h5>Small Heading (H5)</h5>
<h6>Smallest Heading (H6)</h6>

Heading Best Practices

  • Use only one <h1> per page (typically for the main title)
  • Don't skip heading levels (e.g., don't go from <h2> to <h4>)
  • Use headings to create a logical document outline
  • Keep headings concise and descriptive
  • Use headings for structure, not for styling (don't use an <h3> just because you like its size)

Headings are not just about appearance—they create an outline of your document that helps users (especially those using screen readers) navigate your content.

Paragraphs: The Building Blocks of Content

The paragraph element <p> is the workhorse of web content. It's used for blocks of text and automatically adds space before and after the content.

<p>This is a paragraph of text. Paragraphs are block-level elements that automatically add space before and after the content.</p>
<p>This is another paragraph. Notice how it starts on a new line with space between paragraphs.</p>

Paragraphs help break up your content into digestible chunks, making it easier for users to read and understand your message.

Line Breaks

Sometimes you need a line break without starting a new paragraph. The <br> element creates a line break within text:

<p>This text will be on one line.<br>This text will be on a new line without the extra spacing a new paragraph would create.</p>

When to Use Line Breaks

Use line breaks sparingly and only when the content requires it, such as in:

  • Poetry or song lyrics
  • Addresses
  • When you need a visual break without semantic separation

Don't use multiple <br> elements to create spacing between paragraphs—use CSS for that purpose.

Horizontal Rules

The <hr> element creates a horizontal line, often used to separate content sections:

<h2>Section 1</h2>
<p>Content for section 1...</p>
<hr>
<h2>Section 2</h2>
<p>Content for section 2...</p>

Horizontal rules provide a visual separation between content sections, helping users understand when one topic ends and another begins.

Text Emphasis and Importance

HTML provides several elements for emphasizing text and indicating importance:

Strong Importance

The <strong> element indicates that text has strong importance, seriousness, or urgency. Browsers typically render it in bold:

<p><strong>Warning:</strong> This action cannot be undone.</p>

Emphasis

The <em> element indicates emphasis that subtly changes the meaning of a sentence. Browsers typically render it in italics:

<p>I <em>really</em> want to learn HTML.</p>

Bold and Italic (Presentational)

While semantic elements are preferred, HTML also provides presentational elements for bold and italic text:

<!-- Bold text (presentational) -->
<p>This is <b>bold text</b> without semantic importance.</p>

<!-- Italic text (presentational) -->
<p>This is <i>italic text</i> without semantic emphasis.</p>

When to Use Presentational Elements

Use <b> and <i> only when there's no semantic alternative, such as:

  • Using <b> for keywords or product names
  • Using <i> for technical terms, foreign phrases, or thoughts

In most cases, <strong> and <em> are better choices.

Additional Text Formatting Elements

HTML provides several other elements for specific text formatting needs:

Marked Text

The <mark> element represents text that is highlighted for reference or notation purposes:

<p>The most important part is <mark>highlighted</mark> for emphasis.</p>

Subscript and Superscript

The <sub> and <sup> elements represent subscript and superscript text:

<!-- Subscript for chemical formulas -->
<p>Water is H<sub>2</sub>O</p>

<!-- Superscript for mathematical expressions -->
<p>The area of a square is side<sup>2</sup></p>

<!-- Superscript for footnotes -->
<p>This statement needs a citation<sup>1</sup>.</p>

Inserted and Deleted Text

The <ins> and <del> elements represent text that has been added to or removed from a document:

<p>The meeting is scheduled for <del>Monday</del> <ins>Tuesday</ins> at 3 PM.</p>

Small Text

The <small> element represents side comments or small print:

<p>This is a regular paragraph.</p>
<p><small>Copyright 2025. All rights reserved.</small></p>

Abbreviations

The <abbr> element represents an abbreviation or acronym:

<p><abbr title="HyperText Markup Language">HTML</abbr> is the standard markup language for web pages.</p>

When users hover over the abbreviation, they'll see the full title as a tooltip.

Quotations and Citations

HTML provides specific elements for quotations and citations:

Blockquotes

The <blockquote> element represents a section quoted from another source:

<blockquote>
    <p>The only way to learn a new programming language is by writing programs in it.</p>
    <cite>Dennis Ritchie</cite>
</blockquote>

Inline Quotes

The <q> element represents a short inline quotation:

<p>As Einstein said, <q>Imagination is more important than knowledge.</q></p>

Browsers typically add quotation marks around the content of a <q> element.

Citations

The <cite> element represents the title of a work:

<p>My favorite book is <cite>The Hitchhiker's Guide to the Galaxy</cite> by Douglas Adams.</p>

Browsers typically render the content of a <cite> element in italics.

Code and Preformatted Text

HTML provides elements specifically for displaying code and preserving formatting:

Code

The <code> element represents computer code:

<p>The <code>console.log()</code> function outputs a message to the console.</p>

Preformatted Text

The <pre> element represents preformatted text, preserving both spaces and line breaks:

<pre>
function greet() {
    console.log("Hello, world!");
}
</pre>

The <pre> element is often used together with <code> for displaying code blocks:

<pre><code>
function greet() {
    console.log("Hello, world!");
}
</code></pre>

Keyboard Input

The <kbd> element represents keyboard input:

<p>To save the file, press <kbd>Ctrl</kbd> + <kbd>S</kbd>.</p>

Sample Output

The <samp> element represents sample output from a computer program:

<p>The program returned: <samp>Error: File not found</samp></p>

Variables

The <var> element represents a variable in a mathematical expression or programming context:

<p>The area of a circle is π × <var>r</var><sup>2</sup>, where <var>r</var> is the radius.</p>

Generic Text Containers

HTML provides generic container elements for when no specific semantic element is appropriate:

Span

The <span> element is an inline container used to style or manipulate a specific portion of text:

<p>This paragraph has <span style="color: blue;">some blue text</span> in it.</p>

Division

The <div> element is a block-level container used to group content for styling and layout purposes:

<div class="highlight-box">
    <h3>Important Note</h3>
    <p>This content is grouped together in a div for styling purposes.</p>
</div>

When to Use Generic Containers

Use <span> and <div> when:

  • No other semantic element is appropriate
  • You need a container for styling purposes
  • You need to group elements for JavaScript manipulation

Always prefer semantic elements when they're available.

Character Entities

Sometimes you need to display characters that have special meaning in HTML or aren't easily typed. Character entities allow you to display these special characters:

<!-- Less than sign -->
<p>Use &lt; for the less than symbol.</p>

<!-- Greater than sign -->
<p>Use &gt; for the greater than symbol.</p>

<!-- Ampersand -->
<p>Use &amp; for the ampersand symbol.</p>

<!-- Non-breaking space -->
<p>This text has a non-breaking&nbsp;space.</p>

<!-- Copyright symbol -->
<p>&copy; 2025 My Company</p>

<!-- Registered trademark -->
<p>Product Name&reg;</p>

<!-- Trademark -->
<p>Product Name&trade;</p>

<!-- Euro sign -->
<p>Price: &euro;99.99</p>

<!-- Pound sign -->
<p>Price: &pound;79.99</p>

<!-- Yen sign -->
<p>Price: &yen;9999</p>

Common Character Entities

Character Entity Description
< &lt; Less than
> &gt; Greater than
& &amp; Ampersand
" &quot; Double quotation mark
' &apos; Single quotation mark (apostrophe)
&nbsp; Non-breaking space
© &copy; Copyright symbol
® &reg; Registered trademark

Practice Exercise: Text Formatting

Let's put what we've learned into practice by creating an HTML document that uses various text formatting elements:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Text Formatting Practice</title>
</head>
<body>
    <h1>Text Formatting in HTML</h1>
    
    <h2>Headings and Paragraphs</h2>
    <p>This is a regular paragraph with <strong>strong importance</strong> and <em>emphasized</em> text.</p>
    <p>This paragraph contains <mark>highlighted text</mark> that stands out.</p>
    
    <h2>Quotations</h2>
    <blockquote>
        <p>The greatest glory in living lies not in never falling, but in rising every time we fall.</p>
        <cite>Nelson Mandela</cite>
    </blockquote>
    <p>As Shakespeare wrote, <q>All the world's a stage, and all the men and women merely players.</q></p>
    
    <h2>Scientific Notation</h2>
    <p>Water is H<sub>2</sub>O and the formula for the area of a circle is πr<sup>2</sup>.</p>
    
    <h2>Code Examples</h2>
    <p>The <code>alert()</code> function displays a message to the user.</p>
    <pre><code>
function greet(name) {
    return "Hello, " + name + "!";
}
    </code></pre>
    
    <h2>Keyboard Instructions</h2>
    <p>To save your work, press <kbd>Ctrl</kbd> + <kbd>S</kbd>.</p>
    
    <hr>
    
    <p><small>&copy; 2025 Text Formatting Examples. All rights reserved.</small></p>
</body>
</html>

Try It Yourself

Create the HTML document above and view it in your browser. Then try to:

  1. Add a section with deleted and inserted text
  2. Create a paragraph with an abbreviation and its full form
  3. Add a sample output using the <samp> element
  4. Use a <span> to style a portion of text differently

Summary

In this lesson, we've explored text formatting in HTML:

  • Headings (<h1> to <h6>) create structure and hierarchy
  • Paragraphs (<p>) are the building blocks of content
  • Emphasis elements (<strong>, <em>) highlight important text
  • Specialized elements exist for subscript, superscript, marked text, and more
  • Quotation elements (<blockquote>, <q>, <cite>) properly format quoted content
  • Code elements (<code>, <pre>) display computer code
  • Generic containers (<span>, <div>) group content for styling
  • Character entities display special characters

Proper text formatting is essential for creating readable, accessible, and well-structured web content. In the next lesson, we'll explore links and navigation, which allow users to move between pages and sections of your website.
