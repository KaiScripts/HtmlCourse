Tables in HTML

Understanding HTML Tables

Tables are used to organize and display data in rows and columns. They're perfect for presenting tabular information such as financial data, schedules, comparison charts, and other structured content that needs to be aligned in a grid format.

While tables were once misused for page layout, modern web development uses CSS for layout and reserves tables for their intended purpose: displaying tabular data. In this lesson, we'll learn how to create well-structured, accessible tables that effectively present your data.

Historical Note

In the early days of the web, tables were commonly used for page layout because CSS support was limited. This practice is now discouraged as it creates accessibility issues and makes responsive design difficult. Always use CSS for layout and tables only for tabular data.

Basic Table Structure

A basic HTML table consists of the <table> element, which contains rows defined by the <tr> (table row) element. Each row contains cells defined by the <td> (table data) element for regular cells or the <th> (table header) element for header cells.

Simple Table Example

<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>Country</th>
    </tr>
    <tr>
        <td>John Smith</td>
        <td>28</td>
        <td>USA</td>
    </tr>
    <tr>
        <td>Maria Garcia</td>
        <td>32</td>
        <td>Spain</td>
    </tr>
</table>

This creates a simple 3Ã—3 table with a header row and two data rows. The <th> elements are used for the column headers, which are typically displayed in bold and centered by default.

Table Elements Hierarchy

Here's the hierarchy of the basic table elements:

  • <table>: The container for the entire table
  • <tr>: Table row, contains a single row of cells
  • <th>: Table header cell, used for column or row headers
  • <td>: Table data cell, used for regular data

Accessibility Tip

Always use <th> elements for headers rather than styling <td> elements to look like headers. Screen readers use <th> elements to help users navigate tables and understand the relationship between headers and data cells.

Table Sections

HTML tables can be divided into three sections: header, body, and footer. These sections are defined by the <thead>, <tbody>, and <tfoot> elements, respectively.

<table>
    <thead>
        <tr>
            <th>Product</th>
            <th>Price</th>
            <th>Quantity</th>
            <th>Total</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Widget A</td>
            <td>$10.00</td>
            <td>2</td>
            <td>$20.00</td>
        </tr>
        <tr>
            <td>Widget B</td>
            <td>$15.00</td>
            <td>3</td>
            <td>$45.00</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <th colspan="3">Total</th>
            <td>$65.00</td>
        </tr>
    </tfoot>
</table>

Using these section elements provides several benefits:

  • Improved semantics and accessibility
  • Ability to style each section differently with CSS
  • When printing long tables that span multiple pages, browsers can repeat the header and footer on each page

Note that the <tfoot> element can appear before or after the <tbody> in the HTML code, but it will always be rendered at the bottom of the table.

Table Captions

The <caption> element provides a title or description for a table. It should be the first child of the <table> element.

<table>
    <caption>Monthly Sales Report (2025)</caption>
    <tr>
        <th>Month</th>
        <th>Sales</th>
    </tr>
    <tr>
        <td>January</td>
        <td>$10,000</td>
    </tr>
    <tr>
        <td>February</td>
        <td>$12,500</td>
    </tr>
</table>

The caption is typically displayed above the table, but its position can be adjusted with CSS. Using captions improves accessibility and provides context for the table's content.

Accessibility Tip

Always include a caption that clearly describes the table's content. This helps all users, especially those using screen readers, understand the purpose and context of the table before navigating through it.

Row and Column Spans

Sometimes you need cells that span multiple rows or columns. HTML provides the colspan and rowspan attributes for this purpose.

Column Span

The colspan attribute specifies how many columns a cell should span:

<table>
    <tr>
        <th colspan="2">Name</th>
        <th>Age</th>
    </tr>
    <tr>
        <td>First Name</td>
        <td>Last Name</td>
        <td>Years</td>
    </tr>
    <tr>
        <td>John</td>
        <td>Smith</td>
        <td>28</td>
    </tr>
</table>

In this example, the "Name" header spans two columns.

Row Span

The rowspan attribute specifies how many rows a cell should span:

<table>
    <tr>
        <th>Category</th>
        <th>Product</th>
        <th>Price</th>
    </tr>
    <tr>
        <td rowspan="2">Electronics</td>
        <td>Smartphone</td>
        <td>$699</td>
    </tr>
    <tr>
        <td>Laptop</td>
        <td>$999</td>
    </tr>
</table>

In this example, the "Electronics" cell spans two rows.

Complex Table with Row and Column Spans

Here's a more complex example that uses both row and column spans:

<table>
    <caption>Product Comparison</caption>
    <tr>
        <th rowspan="2">Category</th>
        <th colspan="2">Basic Model</th>
        <th colspan="2">Premium Model</th>
    </tr>
    <tr>
        <th>Price</th>
        <th>Features</th>
        <th>Price</th>
        <th>Features</th>
    </tr>
    <tr>
        <td>Smartphone</td>
        <td>$399</td>
        <td>4GB RAM, 64GB Storage</td>
        <td>$799</td>
        <td>8GB RAM, 256GB Storage</td>
    </tr>
    <tr>
        <td>Laptop</td>
        <td>$699</td>
        <td>i5, 8GB RAM, 256GB SSD</td>
        <td>$1299</td>
        <td>i7, 16GB RAM, 512GB SSD</td>
    </tr>
</table>

When using row and column spans, it's important to ensure that the total number of cells in each row remains consistent with the table structure. Cells that are spanned by other cells are simply omitted from the HTML.

Table Headers and Associations

For complex tables, it's important to explicitly associate header cells with data cells to improve accessibility. This is done using the scope attribute for simple tables and the id and headers attributes for more complex tables.

The Scope Attribute

The scope attribute specifies whether a header cell is a header for a column, row, or group of columns or rows:

<table>
    <tr>
        <th scope="col">Name</th>
        <th scope="col">Age</th>
        <th scope="col">Country</th>
    </tr>
    <tr>
        <th scope="row">John Smith</th>
        <td>28</td>
        <td>USA</td>
    </tr>
    <tr>
        <th scope="row">Maria Garcia</th>
        <td>32</td>
        <td>Spain</td>
    </tr>
</table>

The scope attribute can have the following values:

  • col: The header cell is a header for a column
  • row: The header cell is a header for a row
  • colgroup: The header cell is a header for a group of columns
  • rowgroup: The header cell is a header for a group of rows

The ID and Headers Attributes

For more complex tables, you can use the id attribute on header cells and the headers attribute on data cells to create explicit associations:

<table>
    <caption>Quarterly Sales by Department</caption>
    <tr>
        <th id="empty"></th>
        <th id="q1">Q1</th>
        <th id="q2">Q2</th>
        <th id="q3">Q3</th>
        <th id="q4">Q4</th>
    </tr>
    <tr>
        <th id="electronics">Electronics</th>
        <td headers="electronics q1">$10,000</td>
        <td headers="electronics q2">$12,500</td>
        <td headers="electronics q3">$15,000</td>
        <td headers="electronics q4">$18,000</td>
    </tr>
    <tr>
        <th id="clothing">Clothing</th>
        <td headers="clothing q1">$8,000</td>
        <td headers="clothing q2">$9,500</td>
        <td headers="clothing q3">$11,000</td>
        <td headers="clothing q4">$15,000</td>
    </tr>
</table>

In this example, each data cell has a headers attribute that references the IDs of its associated header cells. This creates an explicit association between headers and data cells, which is especially helpful for screen readers.

Accessibility Tip

Always use the scope attribute for simple tables and the id/headers attributes for complex tables. This helps screen readers announce the appropriate headers when a user navigates to a data cell, making the table more understandable for users with visual impairments.

Column Groups

The <colgroup> and <col> elements allow you to group and style columns without having to specify styles for each cell individually.

<table>
    <colgroup>
        <col style="background-color: #f0f0f0;">
        <col span="2" style="background-color: #e0e0e0;">
        <col style="background-color: #d0d0d0;">
    </colgroup>
    <tr>
        <th>Product</th>
        <th>Q1</th>
        <th>Q2</th>
        <th>Total</th>
    </tr>
    <tr>
        <td>Widget A</td>
        <td>100</td>
        <td>150</td>
        <td>250</td>
    </tr>
    <tr>
        <td>Widget B</td>
        <td>80</td>
        <td>120</td>
        <td>200</td>
    </tr>
</table>

In this example, the first column has a light gray background, the second and third columns have a slightly darker background, and the fourth column has an even darker background. The span attribute on the second <col> element indicates that it applies to two columns.

The <colgroup> and <col> elements should appear after the <caption> element (if present) and before any other table content.

Styling Limitation

Note that only a limited set of CSS properties can be applied to <col> elements: background, border, width, visibility, and a few others. Properties like padding, height, text-align, and vertical-align must be applied to the cells themselves.

Styling Tables with CSS

HTML tables have minimal styling by default. CSS can be used to enhance their appearance and readability.

Basic Table Styling

table {
    border-collapse: collapse;  /* Removes space between cells */
    width: 100%;                /* Makes table full width of container */
    margin-bottom: 20px;        /* Adds space below the table */
}

th, td {
    border: 1px solid #ddd;     /* Adds borders to cells */
    padding: 8px;               /* Adds space inside cells */
    text-align: left;           /* Aligns text to the left */
}

th {
    background-color: #f2f2f2;  /* Light gray background for headers */
    font-weight: bold;          /* Makes header text bold */
}

tr:nth-child(even) {
    background-color: #f9f9f9;  /* Zebra striping for rows */
}

tr:hover {
    background-color: #f5f5f5;  /* Highlight on row hover */
}

Responsive Tables

Tables can be challenging on small screens. Here are some approaches to make tables responsive:

1. Horizontal Scrolling

.table-container {
    overflow-x: auto;  /* Enables horizontal scrolling */
    max-width: 100%;   /* Ensures container doesn't overflow */
}

/* Then wrap your table in a div with this class */

2. Stacked Layout for Small Screens

@media (max-width: 600px) {
    table, thead, tbody, th, td, tr {
        display: block;  /* Makes all elements block-level */
    }
    
    thead tr {
        position: absolute;
        top: -9999px;
        left: -9999px;  /* Hides the header row */
    }
    
    tr {
        margin-bottom: 15px;  /* Adds space between rows */
        border: 1px solid #ccc;
    }
    
    td {
        border: none;
        position: relative;
        padding-left: 50%;  /* Makes room for labels */
    }
    
    td:before {
        position: absolute;
        left: 6px;
        width: 45%;
        padding-right: 10px;
        white-space: nowrap;
        content: attr(data-label);  /* Uses data-label attribute for headers */
        font-weight: bold;
    }
}

For this approach, you'll need to add data-label attributes to your <td> elements:

<td data-label="Name">John Smith</td>

3. Using CSS Grid or Flexbox

For more complex responsive tables, you might consider using CSS Grid or Flexbox instead of the <table> element. This gives you more control over the layout at different screen sizes.

Table Accessibility

Creating accessible tables is crucial for users with disabilities, especially those using screen readers. Here are some key practices:

1. Use Proper Table Structure

  • Use <caption> to provide a title
  • Use <thead>, <tbody>, and <tfoot> to organize content
  • Use <th> for header cells and <td> for data cells

2. Associate Headers with Data Cells

  • Use the scope attribute for simple tables
  • Use id and headers attributes for complex tables

3. Provide a Summary

While the summary attribute is deprecated in HTML5, you can provide a summary in the <caption> element or in the surrounding text.

4. Keep Tables Simple

When possible, avoid complex tables with multiple levels of headers or cells that span many rows or columns. Consider breaking complex tables into simpler ones.

5. Test with Screen Readers

Test your tables with screen readers to ensure they're properly announced and navigable.

ARIA Roles

For complex tables or when using non-table elements for tabular data, you can use ARIA roles to improve accessibility:

<div role="table" aria-label="Sales Data">
    <div role="rowgroup">
        <div role="row">
            <div role="columnheader">Product</div>
            <div role="columnheader">Price</div>
        </div>
    </div>
    <div role="rowgroup">
        <div role="row">
            <div role="cell">Widget A</div>
            <div role="cell">$10.00</div>
        </div>
    </div>
</div>

When to Use Tables

Tables should be used for tabular data, not for layout. Here are some appropriate uses for tables:

Appropriate Uses

  • Financial data (e.g., budgets, sales reports)
  • Schedules and timetables
  • Product comparison charts
  • Statistical data
  • Specifications and technical data

Inappropriate Uses

  • Page layout
  • Forms layout (use CSS instead)
  • Navigation menus (use lists instead)
  • Decorative elements

Alternatives to Tables

For non-tabular data, consider these alternatives:

  • CSS Grid or Flexbox for layout
  • Lists for collections of items
  • Description lists for name-value pairs
  • Cards for product displays

Summary

In this lesson, we've explored tables in HTML:

  • Tables are used to display tabular data in rows and columns
  • Basic table structure includes <table>, <tr>, <th>, and <td> elements
  • Tables can be divided into sections using <thead>, <tbody>, and <tfoot>
  • The <caption> element provides a title for the table
  • Cells can span multiple rows or columns using the rowspan and colspan attributes
  • Header cells should be associated with data cells using the scope attribute or id/headers attributes
  • CSS can be used to style tables and make them responsive
  • Accessibility is crucial for tables, especially for users with screen readers

Tables are powerful tools for organizing and presenting data, but they should be used appropriately and with accessibility in mind. In the next lesson, we'll explore forms, which allow users to input data and interact with web pages.
